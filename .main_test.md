Integration test file that validates the main CLI application by building and executing it as a subprocess to test command-line behavior.

## Key Components

- **TestMainIntegration**: Main test function that builds the CLI binary and runs integration tests
- **Test cases**: Covers help flag, version flag, and invalid flag scenarios
- **Binary execution**: Uses `os/exec` to run the compiled binary with different arguments
- **Output validation**: Captures and validates stdout/stderr output

## Usage Example

```go
// Run the integration tests
go test -v ./main_test.go

// The test automatically:
// 1. Builds a test binary
// 2. Executes it with various flags
// 3. Validates exit codes and output
// 4. Cleans up the test binary

// Example test case structure:
{
    name:     "help",
    args:     []string{"--help"},
    wantErr:  false,
    contains: "OpenFrame CLI",
}
```

The test ensures the CLI properly handles common flags like `--help`, `--version`, and gracefully handles invalid arguments. It validates both successful executions and error conditions by checking exit codes and output content.
