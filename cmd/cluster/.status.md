This file implements a CLI command for displaying detailed Kubernetes cluster status information, providing health, node status, resource usage, and connectivity details.

## Key Components

- **`getStatusCmd()`** - Creates and configures the `cluster status` cobra command with flags and validation
- **`runClusterStatus()`** - Executes the status operation, handling cluster selection and delegating to service layer
- **Command structure** - Supports optional cluster name argument with interactive selection fallback
- **Flag integration** - Uses global flags system with status-specific options like `--detailed` and `--no-apps`

## Usage Example

```go
// Create the status command
statusCmd := getStatusCmd()

// Command supports multiple usage patterns:
// openframe cluster status my-cluster
// openframe cluster status  // interactive selection
// openframe cluster status my-cluster --detailed

// The command execution flow:
// 1. Lists available clusters
// 2. Handles cluster selection (from args or interactive UI)
// 3. Calls service layer to display status information
err := statusCmd.Execute()
if err != nil {
    log.Fatal(err)
}
```

The command integrates with the cluster service layer and operations UI to provide a user-friendly interface for checking cluster health and status information.
