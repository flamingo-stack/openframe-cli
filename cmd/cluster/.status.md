A Cobra command implementation that displays detailed status and health information for Kubernetes clusters managed by OpenFrame.

## Key Components

**Main Functions:**
- `getStatusCmd()` - Creates and configures the Cobra command with flags and validation
- `runClusterStatus()` - Executes the status operation with cluster selection and service calls

**Key Features:**
- Interactive cluster selection when no name provided
- Comprehensive status reporting including health, nodes, apps, and resources
- Configurable detail levels and filtering options
- Pre-run validation of flags and global configuration

## Usage Example

```go
// Create the status command
statusCmd := getStatusCmd()

// Command supports multiple usage patterns:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status my-cluster --no-apps

// The command will:
// 1. Validate global and status-specific flags
// 2. List available clusters
// 3. Select target cluster (interactive if needed)
// 4. Display comprehensive status information
```

The command integrates with the OpenFrame cluster service layer to provide detailed insights into cluster health, node status, installed applications, resource usage, and connectivity - essential for MSP technicians monitoring Kubernetes infrastructure.
