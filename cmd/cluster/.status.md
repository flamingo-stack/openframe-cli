Implements the cluster status command for the OpenFrame CLI, providing detailed information about Kubernetes cluster health and status.

## Key Components

- **getStatusCmd()** - Creates and configures the `cluster status` cobra command with flags and validation
- **runClusterStatus()** - Main execution function that handles cluster selection and displays status information
- **Command flags** - Supports `--detailed` and `--no-apps` options for customizing output
- **Interactive selection** - Provides UI for selecting clusters when no name is specified

## Usage Example

```go
// Create the status command
statusCmd := getStatusCmd()

// Command usage patterns:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status my-cluster --no-apps
```

The command integrates with the cluster service layer to retrieve and display comprehensive cluster information including health status, node details, installed applications, and resource usage. It supports both direct cluster naming and interactive selection when multiple clusters are available.
