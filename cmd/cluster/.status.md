A Cobra CLI command implementation that displays detailed Kubernetes cluster status information including health, nodes, applications, and resource usage.

## Key Components

- **getStatusCmd()** - Creates and configures the cluster status command with flags and validation
- **runClusterStatus()** - Executes the status operation, handling cluster selection and displaying comprehensive status information
- **Command configuration** - Supports optional cluster name argument with interactive selection fallback
- **Flag integration** - Incorporates global and status-specific flags for detailed output control

## Usage Example

```go
// Create status command
statusCmd := getStatusCmd()

// Command supports multiple usage patterns:
// openframe cluster status my-cluster
// openframe cluster status  # interactive selection
// openframe cluster status my-cluster --detailed

// The command will:
// 1. List available clusters
// 2. Allow interactive selection if no name provided
// 3. Display comprehensive status including:
//    - Cluster health and connectivity
//    - Node status and resource usage
//    - Installed applications (unless --no-apps flag)
//    - Detailed information (with --detailed flag)

// Integration with service layer
service := utils.GetCommandService()
err := service.ShowClusterStatus(clusterName, detailed, noApps, verbose)
```

The command provides flexible cluster status inspection with both direct naming and interactive selection modes, making it suitable for both scripted and manual operations.
