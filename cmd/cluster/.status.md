Defines the CLI command for displaying Kubernetes cluster status and health information. This file implements the `status` subcommand that allows users to view detailed cluster information including health, nodes, applications, and resource usage.

## Key Components

- **`getStatusCmd()`** - Creates and configures the cobra command for cluster status operations
- **`runClusterStatus()`** - Executes the status check logic with cluster selection and service interaction
- **Command structure** - Supports both direct cluster naming and interactive selection
- **Flag integration** - Incorporates global flags and status-specific options like `--detailed` and `--no-apps`

## Usage Example

```go
// Command usage examples:
// openframe cluster status my-cluster
// openframe cluster status  # interactive selection
// openframe cluster status my-cluster --detailed

// The command integrates with the service layer:
service := utils.GetCommandService()
operationsUI := ui.NewOperationsUI()

// Lists available clusters and handles selection
clusters, err := service.ListClusters()
clusterName, err := operationsUI.SelectClusterForOperation(clusters, args, "check status")

// Displays cluster status with configurable detail level
err = service.ShowClusterStatus(clusterName, detailed, noApps, verbose)
```
