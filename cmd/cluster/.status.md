This file implements the `status` command for displaying detailed information about Kubernetes clusters, including health, node status, applications, and resource usage.

## Key Components

- **`getStatusCmd()`** - Creates and configures the cobra command with flags and validation
- **`runClusterStatus()`** - Main execution function that handles cluster selection and status display
- **Command configuration** - Sets up argument parsing, flag validation, and help text
- **Service integration** - Uses cluster service and operations UI for business logic

## Usage Example

```go
// Command usage from CLI
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status  // interactive selection

// Internal usage in the command structure
statusCmd := getStatusCmd()
rootCmd.AddCommand(statusCmd)

// The command will:
// 1. List available clusters
// 2. Allow interactive selection if no name provided
// 3. Display detailed cluster status including:
//    - Cluster health and connectivity
//    - Node status and resource usage
//    - Installed applications (unless --no-apps flag)
//    - Additional details with --detailed flag
```

The command supports both direct cluster specification and interactive selection, with optional flags for controlling output verbosity and filtering application information.
