Implements the `cluster status` command for displaying detailed Kubernetes cluster information including health, nodes, applications, and resource usage.

## Key Components

- **getStatusCmd()**: Creates and configures the cobra command with flags, validation, and examples
- **runClusterStatus()**: Executes the status operation with interactive cluster selection and service delegation
- **Command validation**: Validates global and status-specific flags before execution
- **Interactive UI**: Provides cluster selection interface when no name is provided

## Usage Example

```go
// Basic status check
cmd := getStatusCmd()
cmd.SetArgs([]string{"my-cluster"})
cmd.Execute()

// Interactive selection with detailed output
cmd.SetArgs([]string{"--detailed"})
cmd.Execute()

// Status without applications info
cmd.SetArgs([]string{"my-cluster", "--no-apps"})
cmd.Execute()
```

The command supports both direct cluster specification and interactive selection, with optional detailed output and application filtering flags. It integrates with the service layer for actual cluster status retrieval and uses the operations UI for user interaction.
