This file implements a Cobra CLI command for displaying Kubernetes cluster status information with interactive cluster selection and detailed reporting capabilities.

## Key Components

- **`getStatusCmd()`** - Creates and configures the `status` subcommand with flags, validation, and usage examples
- **`runClusterStatus()`** - Executes the status operation with cluster selection UI and delegates to service layer
- **Command structure** - Supports optional cluster name argument with interactive fallback selection
- **Flag integration** - Incorporates global flags and status-specific flags (detailed, no-apps, verbose)

## Usage Example

```go
// Command usage patterns:
// openframe cluster status my-cluster
// openframe cluster status  # interactive selection
// openframe cluster status my-cluster --detailed

// Internal function flow:
statusCmd := getStatusCmd()
err := statusCmd.Execute() // triggers runClusterStatus

// The command will:
// 1. List available clusters
// 2. Show interactive selection if no cluster specified
// 3. Display comprehensive status including health, nodes, apps, and resources
```

The command provides a user-friendly interface for monitoring cluster health and status, with options for detailed output and the ability to exclude application information when needed.
