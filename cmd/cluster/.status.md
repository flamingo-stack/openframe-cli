Provides the `status` subcommand for the OpenFrame CLI cluster management, allowing users to view detailed cluster information including health, nodes, applications, and resource usage.

## Key Components

- **`getStatusCmd()`** - Creates and configures the cobra.Command for cluster status operations
- **`runClusterStatus()`** - Main execution function that handles cluster selection and status retrieval
- **Interactive selection** - Uses UI components to allow users to select clusters when none specified
- **Flag validation** - Validates status-specific flags and global configuration before execution

## Usage Example

```go
// Command usage patterns:
// openframe cluster status my-cluster          // Show status for specific cluster
// openframe cluster status                     // Interactive cluster selection
// openframe cluster status my-cluster --detailed  // Detailed status information

// The command supports these flags:
// --detailed    Show comprehensive cluster details
// --no-apps     Skip application status information
// --verbose     Enable verbose output

// Integration example:
cmd := getStatusCmd()
rootCmd.AddCommand(cmd)
```

The command integrates with the cluster service layer to retrieve status information and presents it through a user-friendly interface. It includes validation for both global and status-specific flags, and gracefully handles cases where no clusters are available or selected.
