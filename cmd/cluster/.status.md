Provides a Cobra command for displaying detailed Kubernetes cluster status information, including health, node status, applications, and resource usage.

## Key Components

- **`getStatusCmd()`** - Creates and configures the `status` command with flags and validation
- **`runClusterStatus()`** - Executes the status operation with cluster selection and service delegation
- **Command Configuration** - Supports optional cluster name argument with interactive selection fallback
- **Flag Integration** - Includes `--detailed`, `--no-apps`, and global `--verbose` options

## Usage Example

```go
// Register the status command in cluster command group
clusterCmd.AddCommand(getStatusCmd())

// Command usage patterns:
// openframe cluster status                    # Interactive cluster selection
// openframe cluster status my-cluster         # Direct cluster specification  
// openframe cluster status my-cluster --detailed --verbose

// The command integrates with the service layer:
func (s *Service) ShowClusterStatus(name string, detailed, noApps, verbose bool) error {
    // Implementation handles actual status retrieval and display
}
```

The command follows the CLI's common patterns with pre-run validation, global flag synchronization, and wrapped execution through the service layer for consistent error handling and logging.
