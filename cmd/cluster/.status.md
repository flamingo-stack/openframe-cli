Provides cluster status inspection functionality for Kubernetes clusters with interactive selection and detailed reporting capabilities.

## Key Components

- **`getStatusCmd()`** - Creates and configures the Cobra command for cluster status operations with flags and validation
- **`runClusterStatus()`** - Executes the status check workflow including cluster selection and status display

## Usage Example

```go
// Create status command
statusCmd := getStatusCmd()

// Command supports multiple usage patterns:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status  // interactive selection

// Run status check
err := runClusterStatus(cmd, []string{"production-cluster"})
if err != nil {
    log.Fatal(err)
}
```

The command displays comprehensive cluster information including health status, node details, installed applications, resource usage, and connectivity data. It supports both direct cluster name specification and interactive selection from available clusters, with optional detailed output and application filtering flags.
