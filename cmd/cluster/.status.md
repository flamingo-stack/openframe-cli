Implements cluster status command functionality for the openframe CLI, providing detailed information about Kubernetes cluster health, nodes, and applications.

## Key Components

- **`getStatusCmd()`** - Creates and configures the `cluster status` cobra command with flags and validation
- **`runClusterStatus()`** - Core execution logic that handles cluster selection and displays status information
- **Command flags** - Supports `--detailed` and `--no-apps` options for customizing output
- **Interactive selection** - Provides UI for selecting clusters when no name is specified

## Usage Example

```go
// Command usage examples:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status my-cluster --no-apps

// The command integrates with the CLI framework:
cmd := getStatusCmd()
rootCmd.AddCommand(cmd)

// Internal execution flow:
service := utils.GetCommandService()
clusters, err := service.ListClusters()
if err != nil {
    return fmt.Errorf("failed to list clusters: %w", err)
}

// Shows interactive cluster selection if no args provided
clusterName, err := operationsUI.SelectClusterForOperation(clusters, args, "check status")
```

The command supports both direct cluster specification and interactive selection, with configurable detail levels for status output.
