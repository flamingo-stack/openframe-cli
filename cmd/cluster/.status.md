Implements the `cluster status` command for displaying detailed Kubernetes cluster information including health, node status, applications, and resource usage.

## Key Components

- **`getStatusCmd()`** - Creates and configures the cobra command with flags and validation
- **`runClusterStatus()`** - Main execution logic that handles cluster selection and status display
- **Status flags** - Supports `--detailed` and `--no-apps` options for customizing output
- **Interactive selection** - Provides UI for selecting clusters when name not specified

## Usage Example

```go
// Command line usage examples:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status my-cluster --no-apps

// In code, the command is registered as part of the cluster command group
cmd := getStatusCmd()
rootCmd.AddCommand(cmd)

// The command integrates with the service layer:
service := utils.GetCommandService()
clusters, err := service.ListClusters()
if err != nil {
    return fmt.Errorf("failed to list clusters: %w", err)
}

// Shows status with configurable detail level
err = service.ShowClusterStatus(clusterName, detailed, noApps, verbose)
```
