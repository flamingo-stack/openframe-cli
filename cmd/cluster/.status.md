Implements the cluster status command functionality for the OpenFrame CLI, providing detailed information about Kubernetes cluster health, nodes, applications, and resource usage.

## Key Components

- **getStatusCmd()**: Creates and configures the `cluster status` Cobra command with flags and validation
- **runClusterStatus()**: Main execution logic that lists clusters, handles selection, and displays status information

## Usage Example

```go
// Get the status command for registration
statusCmd := getStatusCmd()

// The command supports various usage patterns:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status  # interactive selection

// Command execution flow
func runClusterStatus(cmd *cobra.Command, args []string) error {
    service := utils.GetCommandService()
    operationsUI := ui.NewOperationsUI()
    
    // List and select cluster
    clusters, err := service.ListClusters()
    clusterName, err := operationsUI.SelectClusterForOperation(clusters, args, "check status")
    
    // Show detailed status
    globalFlags := utils.GetGlobalFlags()
    return service.ShowClusterStatus(clusterName, globalFlags.Status.Detailed, 
                                   globalFlags.Status.NoApps, globalFlags.Global.Verbose)
}
```

The command integrates with the cluster service layer to provide comprehensive status information including cluster health, node status, installed applications, and resource usage metrics. It supports both direct cluster name specification and interactive cluster selection.
