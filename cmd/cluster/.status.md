Implements the cluster status command functionality for displaying comprehensive Kubernetes cluster information and health metrics.

## Key Components

- **`getStatusCmd()`** - Creates and configures the Cobra command for cluster status operations with flag initialization and validation
- **`runClusterStatus()`** - Core execution function that handles cluster selection and delegates status display to the service layer
- **Command Structure** - Supports optional cluster name argument with interactive selection fallback
- **Flag Integration** - Integrates with global and status-specific flags (detailed output, app filtering, verbose mode)

## Usage Example

```go
// Register the status command
clusterCmd.AddCommand(getStatusCmd())

// Command usage patterns:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status my-cluster --no-apps --verbose
```

The command provides flexible cluster status checking with interactive cluster selection when no name is provided, detailed output options, and comprehensive error handling through the service layer abstraction.
