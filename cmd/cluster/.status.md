Provides a Cobra command for displaying detailed Kubernetes cluster status information. The command supports interactive cluster selection and offers various display options for cluster health, node status, applications, and resource usage.

## Key Components

- **`getStatusCmd()`** - Creates and configures the Cobra command for cluster status operations
- **`runClusterStatus()`** - Executes the status operation with cluster selection and service delegation
- **Status flags** - Supports `--detailed` and `--no-apps` options for customizing output
- **Interactive UI** - Provides user-friendly cluster selection when no cluster name is specified

## Usage Example

```go
// Create the status command
statusCmd := getStatusCmd()

// Add to parent command
parentCmd.AddCommand(statusCmd)

// Command supports multiple usage patterns:
// openframe cluster status my-cluster           # Direct cluster name
// openframe cluster status                      # Interactive selection  
// openframe cluster status my-cluster --detailed # Detailed output
// openframe cluster status my-cluster --no-apps  # Skip application info
```

The command integrates with the cluster service layer to retrieve and display comprehensive cluster information, with validation and error handling for robust operation. It follows the standard CLI pattern of optional positional arguments with interactive fallbacks.
