This file implements the `status` command for the OpenFrame CLI cluster management functionality, providing detailed information about Kubernetes cluster health and status.

## Key Components

- **`getStatusCmd()`** - Creates and configures the cobra command for cluster status operations
- **`runClusterStatus()`** - Core execution logic that handles cluster selection and displays status information
- **Command flags** - Supports `--detailed` and `--no-apps` flags for customizing output detail level
- **Interactive selection** - Provides friendly UI for cluster selection when no name is specified

## Usage Example

```go
// Command usage examples:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status  // interactive selection

// The command integrates with the service layer:
service := utils.GetCommandService()
clusters, err := service.ListClusters()
if err != nil {
    return fmt.Errorf("failed to list clusters: %w", err)
}

// Shows comprehensive cluster information including:
// - Cluster health and connectivity
// - Node status and resource usage  
// - Installed applications (unless --no-apps flag used)
err = service.ShowClusterStatus(clusterName, detailed, noApps, verbose)
```

The command follows standard CLI patterns with proper flag validation, error handling, and user-friendly interactive cluster selection when no cluster name is provided.
