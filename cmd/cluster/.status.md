Implements the `cluster status` command for displaying detailed information about Kubernetes clusters, including health status, node information, and installed applications.

## Key Components

- **`getStatusCmd()`** - Creates and configures the Cobra command with flags and validation
- **`runClusterStatus()`** - Main execution function that handles cluster selection and displays status information
- **Interactive cluster selection** - Uses UI components to prompt users when cluster name is not provided
- **Status flags** - Supports `--detailed` and `--no-apps` options for customizing output

## Usage Example

```go
// Basic status check with interactive selection
cmd := getStatusCmd()
cmd.Execute() // Prompts user to select cluster

// Check status for specific cluster
os.Args = []string{"openframe", "cluster", "status", "my-cluster"}
cmd.Execute()

// Detailed status without applications
os.Args = []string{"openframe", "cluster", "status", "my-cluster", "--detailed", "--no-apps"}
cmd.Execute()
```

The command integrates with the cluster service layer to retrieve and display comprehensive cluster information, with support for both interactive and non-interactive usage patterns.
