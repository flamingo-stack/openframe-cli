Implements the `cluster status` command functionality for displaying detailed Kubernetes cluster information and health status.

## Key Components

**Functions:**
- `getStatusCmd()` - Creates and configures the cobra command with flags and validation
- `runClusterStatus()` - Executes the status operation with cluster selection and service delegation

**Dependencies:**
- Uses `models` package for flag management and validation
- Integrates with `ui` package for interactive cluster selection
- Leverages `utils` package for common command setup and service access

## Usage Example

```go
// Create status command
statusCmd := getStatusCmd()

// Command supports multiple usage patterns:
// openframe cluster status my-cluster          // Direct cluster name
// openframe cluster status                     // Interactive selection  
// openframe cluster status my-cluster --detailed  // With detailed output

// The command validates flags and handles cluster selection
// through an interactive UI when no cluster name is provided
```

The command provides comprehensive cluster information including health status, node details, installed applications, resource usage, and connectivity data. It supports both direct cluster specification and interactive selection from available clusters.
