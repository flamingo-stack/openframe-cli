Implements the "status" subcommand for the cluster CLI, providing detailed cluster health and information display functionality.

## Key Components

**Main Command Functions:**
- `getStatusCmd()` - Creates and configures the status subcommand with flags and validation
- `runClusterStatus()` - Executes the status operation with cluster selection and service delegation

**Command Features:**
- Interactive cluster selection when no name provided
- Detailed status information display
- Support for filtering applications and verbose output
- Validation of status-specific flags

## Usage Example

```go
// Register the status command
clusterCmd.AddCommand(getStatusCmd())

// Command usage examples:
// openframe cluster status my-cluster
// openframe cluster status --detailed
// openframe cluster status my-cluster --no-apps --verbose
```

The command integrates with the cluster service layer to retrieve and display comprehensive cluster information including health status, node details, installed applications, and resource usage metrics. It provides both targeted status checks for specific clusters and interactive selection for operational convenience.
