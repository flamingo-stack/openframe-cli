Implements the `status` command for displaying detailed Kubernetes cluster information and health status. This command provides comprehensive cluster diagnostics including node status, installed applications, resource usage, and connectivity information.

## Key Components

- **`getStatusCmd()`** - Creates and configures the Cobra command with flags, validation, and help text
- **`runClusterStatus()`** - Executes the status operation with cluster selection and service delegation
- **Status flags integration** - Supports detailed output, app filtering, and verbose modes
- **Interactive cluster selection** - Provides user-friendly cluster selection when no name is specified

## Usage Example

```go
// Create the status command
statusCmd := getStatusCmd()

// Execute with explicit cluster name
// openframe cluster status my-cluster

// Execute with interactive selection
// openframe cluster status

// Execute with detailed output
// openframe cluster status my-cluster --detailed

// The command supports these flags:
// --detailed: Show comprehensive cluster information
// --no-apps: Skip application status information
// --verbose: Enable verbose output
```

The command integrates with the cluster service layer to fetch and display cluster information, supports both named and interactive cluster selection, and includes comprehensive flag validation and error handling.
