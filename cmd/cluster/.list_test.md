Test file for the cluster list command functionality, ensuring the command executes properly in a controlled test environment.

## Key Components

- **TestListCommand**: Main test function that validates the cluster list command behavior
- **setupFunc**: Configures test environment with mock executor
- **teardownFunc**: Cleans up global state after test execution
- **init()**: Initializes test mode for the package

## Usage Example

```go
// Run the test
go test ./internal/cluster -run TestListCommand

// The test automatically:
// 1. Sets up a mock executor via setupFunc
// 2. Tests the list command using testutil.TestClusterCommand
// 3. Cleans up global flags via teardownFunc
```

The test follows a standard setup/teardown pattern, using the `testutil.TestClusterCommand` helper to validate that the `getListCmd` function returns a properly configured command that can execute without errors in the test environment.
