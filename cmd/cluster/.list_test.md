Test file for the cluster list command functionality, ensuring proper command execution and validation through mocked environments.

## Key Components

- **TestListCommand**: Main test function that validates the cluster list command behavior
- **setupFunc**: Initializes test environment with mock executor for command simulation
- **teardownFunc**: Cleans up global state after test execution
- **testutil.TestClusterCommand**: Utility function that orchestrates the test lifecycle

## Usage Example

```go
// Run the list command test
func TestListCommand(t *testing.T) {
    setupFunc := func() {
        utils.SetTestExecutor(testutil.NewTestMockExecutor())
    }
    teardownFunc := func() {
        utils.ResetGlobalFlags()
    }
    
    testutil.TestClusterCommand(t, "list", getListCmd, setupFunc, teardownFunc)
}
```

The test leverages dependency injection through mock executors to simulate command execution without requiring actual cluster infrastructure, enabling reliable unit testing of the list command's behavior and output formatting.
