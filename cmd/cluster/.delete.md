Provides a command-line interface for deleting Kubernetes clusters with interactive selection and confirmation. Handles cluster deletion with proper resource cleanup and error handling.

## Key Components

**Functions:**
- `getDeleteCmd()` - Creates and configures the delete subcommand with flags and validation
- `runDeleteCluster()` - Main execution logic for cluster deletion process

**Command Features:**
- Interactive cluster selection when no name provided
- Force deletion flag support
- Pre-run validation and global flag synchronization
- Comprehensive error handling with user-friendly messages

## Usage Example

```go
// Create delete command
deleteCmd := getDeleteCmd()

// Command supports multiple usage patterns:
// openframe cluster delete my-cluster
// openframe cluster delete my-cluster --force  
// openframe cluster delete  # interactive selection

// The command will:
// 1. List available clusters
// 2. Prompt for selection/confirmation if needed
// 3. Detect cluster type automatically
// 4. Execute deletion with cleanup
// 5. Display progress and results
```

The command integrates with the service layer for actual cluster operations and uses the UI layer for user interaction, providing a complete cluster deletion workflow with safety checks and confirmation prompts.
