Provides functionality to delete Kubernetes clusters with comprehensive cleanup and user-friendly interactive selection. Handles stopping intercepts, removing cluster resources, cleaning up Docker resources, and removing configuration files.

## Key Components

- **`getDeleteCmd()`** - Creates the cobra command with delete-specific flags and validation
- **`runDeleteCluster()`** - Main execution function that handles cluster selection, type detection, and deletion
- **Interactive Selection** - Uses operations UI for user-friendly cluster selection and confirmation
- **Force Flag Support** - Allows bypassing confirmation prompts for automated workflows
- **Global Flag Integration** - Inherits and validates global CLI flags for consistent behavior

## Usage Example

```go
// Create the delete command
deleteCmd := getDeleteCmd()

// The command supports multiple usage patterns:
// Interactive selection from available clusters
cmd.Execute() // openframe cluster delete

// Direct cluster specification
cmd.SetArgs([]string{"my-cluster"}) 
cmd.Execute() // openframe cluster delete my-cluster

// Force deletion without confirmation
cmd.SetArgs([]string{"my-cluster", "--force"})
cmd.Execute() // openframe cluster delete my-cluster --force

// Access the underlying deletion logic
err := runDeleteCluster(cmd, []string{"cluster-name"})
if err != nil {
    // Handle deletion errors
}
```

The command provides comprehensive error handling, validates flags during pre-run, and uses the service layer for actual cluster operations while maintaining a clean separation between CLI interface and business logic.
