Implements the `create` command for the OpenFrame CLI to create new Kubernetes clusters. Provides both interactive configuration wizards and non-interactive flag-based creation with validation and dry-run capabilities.

## Key Components

- **`getCreateCmd()`** - Returns configured cobra.Command for cluster creation with flags and validation
- **`runCreateCluster()`** - Main execution function that handles interactive/non-interactive modes and cluster creation
- **Interactive Mode** - Uses UI layer for step-by-step cluster configuration when `--skip-wizard` is not set
- **Non-Interactive Mode** - Builds cluster config directly from command flags and arguments
- **Validation** - Validates cluster names, node counts, and global flags before creation
- **Configuration Summary** - Shows cluster settings for dry-run, skip-wizard, or verbose modes

## Usage Example

```go
// Create cluster with interactive wizard (default)
openframe cluster create

// Create cluster with custom name using wizard  
openframe cluster create my-cluster

// Create cluster with flags, skip wizard
openframe cluster create --nodes 3 --type k3d --skip-wizard

// Dry run to see configuration without creating
openframe cluster create --dry-run --nodes 5
```

The command integrates with the service layer for actual cluster creation and supports k3d cluster types with configurable node counts and Kubernetes versions.
