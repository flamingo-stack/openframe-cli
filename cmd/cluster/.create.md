This file implements the cluster creation command for the OpenFrame CLI, providing both interactive wizard and direct creation modes with comprehensive flag validation and configuration handling.

## Key Components

- **`getCreateCmd()`** - Returns the main cobra command with usage examples, flag setup, and validation hooks
- **`runCreateCluster()`** - Core execution logic that handles interactive vs non-interactive modes and delegates to service layer
- **Interactive Mode** - Uses UI configuration handler when `--skip-wizard` is not set
- **Non-interactive Mode** - Builds cluster config directly from command flags and arguments
- **Validation Pipeline** - Validates cluster names, node counts, and global flags before execution

## Usage Example

```go
// Interactive mode (default)
cmd := getCreateCmd()
cmd.Execute() // Shows selection menu for quick start or wizard

// Non-interactive with defaults
args := []string{"my-cluster", "--skip-wizard"}
cmd.SetArgs(args)
cmd.Execute()

// Custom configuration
args := []string{"production", "--nodes", "5", "--type", "k3d", "--skip-wizard"}
cmd.SetArgs(args)
cmd.Execute()

// Dry run to preview configuration
args := []string{"test", "--dry-run"}
cmd.SetArgs(args)
cmd.Execute()
```

The command supports cluster name validation, automatic defaults for missing values, and integrates with the service layer for actual cluster provisioning.
