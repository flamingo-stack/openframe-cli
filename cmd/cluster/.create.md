Implements the CLI command for creating Kubernetes clusters with support for both interactive and non-interactive modes, including configuration wizards and validation.

## Key Components

- **getCreateCmd()** - Returns the cobra command definition with flags, validation, and help text
- **runCreateCluster()** - Main execution function that handles cluster creation workflow
- **Configuration Modes** - Supports interactive UI wizard or direct creation with command flags
- **Validation Pipeline** - Validates cluster names, node counts, and global flags before execution

## Usage Example

```go
// Create command with interactive mode (default)
createCmd := getCreateCmd()

// Example command invocations handled:
// openframe cluster create                    # Show creation mode selection
// openframe cluster create my-cluster        # Show selection with custom name  
// openframe cluster create --skip-wizard     # Direct creation with defaults
// openframe cluster create --nodes 3 --type k3d --skip-wizard

// The command supports both modes:
// 1. Interactive: Uses UI layer for step-by-step configuration
// 2. Non-interactive: Builds config directly from flags and arguments

// Dry-run example shows configuration without creating:
// openframe cluster create --dry-run --nodes 5 --type kind
```

The command integrates with the UI layer for interactive configuration and the service layer for actual cluster creation, providing flexible cluster management workflows for OpenFrame development environments.
