Handles cluster creation functionality for the OpenFrame CLI, providing both interactive and non-interactive modes for creating Kubernetes clusters.

## Key Components

- **`getCreateCmd()`** - Returns the Cobra command definition for cluster creation with flags and validation
- **`runCreateCluster()`** - Main execution function that handles cluster configuration and creation logic
- **Interactive Mode** - Uses UI handlers for step-by-step cluster configuration when `--skip-wizard` is not set
- **Non-interactive Mode** - Creates clusters directly from command flags and arguments
- **Configuration Summary** - Shows cluster settings before creation in verbose/dry-run modes

## Usage Example

```go
// Basic cluster creation with interactive wizard
cmd := getCreateCmd()
cmd.Execute() // Shows selection menu for configuration options

// Create cluster with specific name
args := []string{"my-dev-cluster"}
runCreateCluster(cmd, args)

// Non-interactive creation with custom settings
cmd.Flags().Set("skip-wizard", "true")
cmd.Flags().Set("nodes", "5")
cmd.Flags().Set("type", "kind")
runCreateCluster(cmd, []string{"production-cluster"})

// Dry run to preview configuration
cmd.Flags().Set("dry-run", "true")
runCreateCluster(cmd, args) // Shows config without creating
```

The command supports cluster types like k3d and kind, validates cluster names and node counts, and integrates with the broader OpenFrame CLI architecture through service and UI layers.
