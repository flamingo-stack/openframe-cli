This file implements the CLI command for creating Kubernetes clusters in the OpenFrame development environment. It provides both interactive and non-interactive modes for cluster creation with flexible configuration options.

## Key Components

- **`getCreateCmd()`** - Creates and configures the `create` subcommand with flags, validation, and help text
- **`runCreateCluster()`** - Main execution logic that handles both interactive wizard and direct creation modes
- **Interactive Mode** - Uses UI layer to guide users through cluster configuration
- **Non-interactive Mode** - Creates clusters directly using command-line flags and arguments
- **Configuration Validation** - Validates cluster names, node counts, and other parameters before creation

## Usage Example

```go
// Interactive mode with cluster selection menu
openframe cluster create

// Quick creation with defaults and custom name
openframe cluster create my-dev-cluster

// Non-interactive with specific configuration
openframe cluster create --nodes 3 --type k3d --skip-wizard

// Dry run to preview configuration
openframe cluster create --dry-run --nodes 5 --k8s-version v1.28.0
```

The command supports various cluster types (k3d by default), automatic validation of inputs, and integrates with the broader OpenFrame CLI infrastructure through the utils package for consistent flag handling and command setup.
