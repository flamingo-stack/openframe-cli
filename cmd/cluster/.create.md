This file implements the CLI command for creating Kubernetes clusters in the OpenFrame development environment, providing both interactive and non-interactive configuration modes.

## Key Components

- **getCreateCmd()** - Returns a configured cobra.Command for cluster creation with flag definitions and validation
- **runCreateCluster()** - Main execution logic that handles both interactive wizard and direct creation modes
- **Interactive Mode** - Uses UI layer to guide users through cluster configuration when `--skip-wizard` is not set
- **Non-Interactive Mode** - Creates clusters directly from command-line flags and arguments with sensible defaults

## Usage Example

```go
// Get the create command for CLI registration
createCmd := getCreateCmd()

// The command supports various usage patterns:
// openframe cluster create                    # Interactive mode with selection menu
// openframe cluster create my-cluster        # Interactive mode with custom name
// openframe cluster create --skip-wizard     # Direct creation with defaults
// openframe cluster create --nodes 3 --type k3d --skip-wizard  # Custom configuration

// Configuration validation example
config := models.ClusterConfig{
    Name:       "openframe-dev",
    Type:       models.ClusterTypeK3d,
    NodeCount:  3,
    K8sVersion: "v1.28",
}

// The command automatically validates cluster names and node counts
if nodeCount <= 0 {
    return fmt.Errorf("node count must be at least 1: %d", nodeCount)
}
```

The command integrates with the UI layer for interactive configuration and the service layer for actual cluster creation, supporting dry-run mode and verbose output for configuration verification.
