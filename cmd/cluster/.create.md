This file implements the `create` command for the OpenFrame CLI tool, enabling users to create new Kubernetes clusters with either quick defaults or interactive configuration.

## Key Components

- **`getCreateCmd()`** - Creates and configures the cobra command with flags, validation, and help text
- **`runCreateCluster()`** - Main execution logic handling both interactive and non-interactive cluster creation flows
- **Command flags integration** - Leverages global flags system for cluster type, node count, K8s version, and operational modes
- **UI/Service separation** - Uses UI layer for interactive configuration and service layer for actual cluster operations

## Usage Example

```go
// Interactive mode (default) - shows selection menu
cmd := getCreateCmd()
cmd.Execute() // User prompted for configuration options

// Non-interactive with defaults
cmd.SetArgs([]string{"--skip-wizard"})
cmd.Execute() // Creates "openframe-dev" cluster with defaults

// Custom cluster with specific settings
cmd.SetArgs([]string{
    "my-cluster", 
    "--nodes", "3", 
    "--type", "k3d", 
    "--skip-wizard",
})
cmd.Execute() // Creates "my-cluster" with 3 nodes using k3d

// Dry run to preview configuration
cmd.SetArgs([]string{"--dry-run", "test-cluster"})
cmd.Execute() // Shows what would be created without executing
```

The command supports flexible cluster naming, validates inputs, and provides both quick setup and detailed configuration workflows for different user preferences.
