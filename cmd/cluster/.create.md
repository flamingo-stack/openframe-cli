This file implements the cluster creation command for the OpenFrame CLI tool, providing both interactive and non-interactive modes for setting up Kubernetes clusters.

## Key Components

- **`getCreateCmd()`** - Creates and configures the cobra command for cluster creation with flags and validation
- **`runCreateCluster()`** - Main execution function that handles both interactive wizard and direct creation modes
- **Interactive Mode** - Uses UI layer for step-by-step cluster configuration when `--skip-wizard` is not set
- **Non-interactive Mode** - Builds cluster config directly from command flags and arguments
- **Validation** - Validates cluster names, node counts, and global flags before execution
- **Configuration Summary** - Shows cluster settings before creation in verbose, dry-run, or skip-wizard modes

## Usage Example

```go
// Interactive mode with cluster selection menu
openframe cluster create

// Quick creation with custom name
openframe cluster create my-cluster

// Non-interactive with specific configuration
openframe cluster create --nodes 3 --type k3d --skip-wizard

// Dry run to preview configuration
openframe cluster create --dry-run --nodes 5

// Custom cluster with all options
openframe cluster create prod-cluster --nodes 3 --type k3d --k8s-version v1.28.0 --skip-wizard
```

The command supports flexible cluster creation workflows, from guided setup for beginners to quick deployment for experienced users, with comprehensive validation and preview capabilities.
