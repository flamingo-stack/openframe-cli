Creates and manages Kubernetes cluster creation with support for both interactive and non-interactive modes. Provides a CLI command interface with configuration validation and dry-run capabilities.

## Key Components

- **getCreateCmd()** - Returns the main cobra command for cluster creation with flags and validation
- **runCreateCluster()** - Main execution logic that handles both interactive and non-interactive cluster creation modes
- **Interactive Mode** - Uses UI configuration handler for step-by-step cluster setup
- **Non-Interactive Mode** - Creates clusters directly from command flags with sensible defaults

## Usage Example

```go
// Interactive cluster creation with name selection
cmd := getCreateCmd()
cmd.Execute() // Shows interactive menu

// Direct creation with defaults
openframe cluster create my-cluster --skip-wizard

// Creation with custom settings
openframe cluster create --nodes 5 --type k3d --k8s-version v1.28.0 --skip-wizard

// Dry run to preview configuration
openframe cluster create test-cluster --dry-run --skip-wizard
```

The command supports flexible cluster naming, automatic validation, and integrates with the service layer for actual cluster provisioning. Default cluster name is "openframe-dev" when none is specified.
