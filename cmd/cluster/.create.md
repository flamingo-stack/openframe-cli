This Go file implements the CLI command for creating Kubernetes clusters, providing both interactive and non-interactive configuration modes with comprehensive validation and user experience features.

## Key Components

**Exported Functions:**
- `getCreateCmd()` - Returns the configured cobra.Command for cluster creation
- `runCreateCluster()` - Main execution logic for the create command

**Key Features:**
- Interactive wizard vs quick start options
- Command-line flag support with validation
- Dry-run capability for testing configurations
- Automatic default value handling

## Usage Example

```go
// Register the create command
createCmd := getCreateCmd()
rootCmd.AddCommand(createCmd)

// Example command usage:
// openframe cluster create                    # Interactive mode with selection
// openframe cluster create my-cluster        # Named cluster with selection
// openframe cluster create --skip-wizard     # Direct creation with defaults
// openframe cluster create --nodes 3 --type k3d --skip-wizard

// The command supports various flags:
// --skip-wizard    - Bypass interactive configuration
// --dry-run        - Show configuration without creating
// --nodes          - Number of cluster nodes
// --type           - Cluster type (k3d, kind, etc.)
```

The implementation follows a layered architecture with UI handlers for interactive configuration, service layer for cluster operations, and comprehensive flag validation. It automatically handles edge cases like invalid node counts and provides helpful error messages for better user experience.
