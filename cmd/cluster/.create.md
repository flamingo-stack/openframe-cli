This file implements the cluster creation command for the OpenFrame CLI, providing both interactive and non-interactive modes for creating Kubernetes clusters.

## Key Components

- **`getCreateCmd()`** - Creates and configures the `cluster create` Cobra command with flags and validation
- **`runCreateCluster()`** - Main execution function that handles cluster configuration and creation logic
- **Interactive Mode** - Uses UI layer for step-by-step cluster configuration when `--skip-wizard` is not set
- **Non-Interactive Mode** - Builds cluster configuration directly from command flags and arguments
- **Validation** - Validates cluster names, node counts, and other configuration parameters
- **Dry Run Support** - Shows configuration summary without actually creating the cluster

## Usage Example

```go
// Create cluster with interactive wizard (default)
openframe cluster create

// Create with specific name and show selection menu
openframe cluster create my-cluster

// Skip wizard and use defaults
openframe cluster create --skip-wizard

// Create with custom configuration
openframe cluster create --nodes 3 --type k3d --skip-wizard my-dev-cluster

// Preview configuration without creation
openframe cluster create --dry-run --nodes 5
```

The command supports both quick defaults for rapid development setup and detailed interactive configuration for customized cluster creation. It integrates with the service layer for actual cluster provisioning and includes comprehensive validation and error handling.
