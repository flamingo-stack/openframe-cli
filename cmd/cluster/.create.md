This file implements the Kubernetes cluster creation command for the OpenFrame CLI, providing both interactive and non-interactive modes for cluster configuration.

## Key Components

- **`getCreateCmd()`** - Returns a configured cobra command for cluster creation with validation and flag handling
- **`runCreateCluster()`** - Main execution function that handles both interactive wizard and direct flag-based cluster creation
- **Interactive Mode** - Uses UI layer for step-by-step cluster configuration when `--skip-wizard` is not set
- **Non-Interactive Mode** - Builds cluster config directly from command flags and arguments
- **Validation** - Performs cluster name validation and node count verification
- **Configuration Summary** - Shows cluster settings before creation in verbose/dry-run modes

## Usage Example

```go
// Interactive cluster creation (default)
// openframe cluster create

// Quick creation with defaults
// openframe cluster create --skip-wizard

// Custom cluster with specific settings
// openframe cluster create my-cluster --nodes 3 --type k3d --skip-wizard

// Dry run to see configuration without creating
// openframe cluster create --dry-run --nodes 5
```

The command supports flexible cluster naming, automatic defaults for missing configuration, and integrates with the service layer for actual cluster provisioning.
