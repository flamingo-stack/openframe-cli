This file implements the "create" subcommand for creating Kubernetes clusters in the OpenFrame CLI tool. It provides both interactive and non-interactive modes for cluster configuration with comprehensive validation and flag handling.

## Key Components

- **`getCreateCmd()`** - Returns configured cobra command with flags, validation, and help text
- **`runCreateCluster()`** - Main execution logic handling interactive/non-interactive modes
- **Interactive mode** - Uses UI layer for step-by-step cluster configuration when `--skip-wizard` not set
- **Non-interactive mode** - Builds configuration directly from CLI flags and arguments
- **Configuration validation** - Validates cluster names, node counts, and other parameters
- **Dry-run support** - Shows configuration summary without creating cluster

## Usage Example

```go
// Create command with interactive wizard
openframe cluster create

// Create with specific name and skip wizard
openframe cluster create my-cluster --skip-wizard --nodes 3 --type k3d

// Dry run to see configuration
openframe cluster create --dry-run --nodes 5

// The command integrates with the broader CLI structure:
cmd := getCreateCmd()
rootCmd.AddCommand(cmd)
```

The command supports flexible cluster creation with defaults for quick setup or detailed configuration through an interactive wizard, making it suitable for both development and production scenarios.
