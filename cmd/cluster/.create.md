This file implements the `create` command for Kubernetes cluster creation in the OpenFrame CLI, providing both interactive and non-interactive modes with flexible configuration options.

## Key Components

- **`getCreateCmd()`** - Creates and configures the cobra command with flags, validation, and help text
- **`runCreateCluster()`** - Main execution logic that handles both interactive wizard and direct flag-based cluster creation
- **Interactive Mode** - Uses UI layer for step-by-step cluster configuration when `--skip-wizard` is not set
- **Non-Interactive Mode** - Creates clusters directly from command flags with sensible defaults
- **Configuration Validation** - Validates cluster names, node counts, and other parameters before creation
- **Dry Run Support** - Shows configuration summary without actually creating the cluster

## Usage Example

```go
// Interactive cluster creation with default name
openframe cluster create

// Create cluster with custom name using interactive wizard
openframe cluster create my-cluster

// Direct creation with defaults, skipping the wizard
openframe cluster create --skip-wizard

// Create 5-node k3d cluster with specific Kubernetes version
openframe cluster create --nodes 5 --type k3d --k8s-version v1.28.0 --skip-wizard

// Preview configuration without creating (dry run)
openframe cluster create my-cluster --dry-run
```

The command supports automatic validation, intelligent defaults (3 nodes, k3d type), and integrates with the broader OpenFrame cluster management system for local development environments.
