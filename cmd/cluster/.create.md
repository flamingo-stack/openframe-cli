This file implements the `create` command for the OpenFrame CLI, enabling users to create new Kubernetes clusters with either quick defaults or interactive configuration.

## Key Components

- **`getCreateCmd()`** - Creates and configures the cobra command with flags, validation, and command setup
- **`runCreateCluster()`** - Executes the cluster creation logic, handling both interactive and non-interactive modes
- **Interactive Mode** - Uses UI components to guide users through cluster configuration 
- **Non-Interactive Mode** - Creates clusters directly from command-line flags with sensible defaults
- **Configuration Validation** - Validates cluster names, node counts, and other parameters before creation

## Usage Example

```go
// Interactive mode with selection menu
cmd := getCreateCmd()
cmd.Execute() // Shows creation mode selection

// Quick creation with defaults
args := []string{"my-cluster"}
flags := &models.CreateFlags{SkipWizard: true}
runCreateCluster(cmd, args)

// Custom configuration
args := []string{"production-cluster"}  
flags := &models.CreateFlags{
    SkipWizard: true,
    NodeCount: 5,
    ClusterType: "k3d",
}
```

The command supports both `openframe cluster create` (interactive) and `openframe cluster create --skip-wizard --nodes 3` (direct) workflows, with built-in validation and dry-run capabilities for safe cluster provisioning.
