This file implements the `install` subcommand for the chart package, providing functionality to install ArgoCD and app-of-apps configurations on Kubernetes clusters with support for multiple deployment modes.

## Key Components

- **getInstallCmd()** - Creates and configures the cobra command for chart installation
- **runInstallCommand()** - Main execution handler that processes flags and delegates to installation service
- **InstallFlags** - Struct containing all installation configuration flags
- **extractInstallFlags()** - Extracts and validates command-line flags with business rule validation
- **getVerboseFlag()** - Retrieves verbose flag with fallback logic from root or current command
- **addInstallFlags()** - Defines all available command-line flags for the install command

## Usage Example

```go
// Create the install command
installCmd := getInstallCmd()

// The command supports various usage patterns:
// Interactive mode: openframe chart install
// Specific cluster: openframe chart install my-cluster  
// CI/CD mode: openframe chart install --deployment-mode=oss-tenant --non-interactive
// Custom repository: openframe chart install --github-repo=https://github.com/custom/repo --github-branch=develop

// Flag validation ensures:
// - deployment-mode must be one of: oss-tenant, saas-tenant, saas-shared
// - non-interactive mode requires deployment-mode to be specified
// - All flags are properly extracted and validated before processing

// The command integrates with the services package for actual installation
// and uses shared error handling for consistent error display
```
