This file provides the Cobra CLI command implementation for installing ArgoCD and app-of-apps charts on Kubernetes clusters with configurable deployment modes.

## Key Components

- **`getInstallCmd()`** - Creates the main install command with comprehensive help text and usage examples
- **`runInstallCommand()`** - Executes the installation by extracting flags and delegating to the services layer
- **`InstallFlags`** - Struct containing all installation configuration options (force, dry-run, GitHub settings, deployment mode, etc.)
- **`extractInstallFlags()`** - Validates and extracts command flags with business logic validation for deployment modes
- **`addInstallFlags()`** - Defines all available command-line flags with defaults and descriptions
- **`getVerboseFlag()`** - Utility to extract verbose flag with fallback logic

## Usage Example

```go
// Get the install command for CLI registration
installCmd := getInstallCmd()

// The command supports multiple usage patterns:
// openframe chart install                                    # Interactive mode
// openframe chart install my-cluster                        # Install on specific cluster  
// openframe chart install --deployment-mode=oss-tenant     # Skip deployment selection
// openframe chart install --dry-run --verbose              # Preview installation
```

The command validates deployment modes (oss-tenant, saas-tenant, saas-shared) and enforces that non-interactive mode requires a deployment mode. It integrates with the services layer for actual chart installation and uses shared error handling for consistent CLI experience.
