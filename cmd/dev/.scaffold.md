This file defines the CLI command for scaffolding development environments with live reloading capabilities using Skaffold.

## Key Components

- **getScaffoldCmd()**: Returns a configured cobra.Command for the scaffold subcommand with comprehensive flag options
- **runScaffold()**: Command execution handler that initializes the scaffold service and runs the workflow
- **models.ScaffoldFlags**: Configuration struct containing development-specific options like port, namespace, and sync directories
- **scaffoldService.NewService()**: Service factory for creating scaffold workflow instances

## Usage Example

```go
// Register the scaffold command in a parent command
devCmd.AddCommand(getScaffoldCmd())

// The command supports various development scenarios:
// Basic scaffolding with interactive cluster selection
// openframe dev skaffold

// Scaffold with specific cluster and custom port
// openframe dev skaffold my-dev-cluster --port 3000

// Advanced configuration with custom image and file syncing
// openframe dev skaffold --image myapp:dev --sync-local ./src --sync-remote /app/src

// Skip cluster bootstrap for existing environments
// openframe dev skaffold --skip-bootstrap --namespace dev-env
```

The scaffold command provides a complete development workflow including prerequisite validation, cluster bootstrapping with development settings, and Skaffold integration for hot reloading.
