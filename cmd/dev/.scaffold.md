This file defines the CLI command for setting up a development environment with live reloading using Skaffold for the OpenFrame platform.

## Key Components

- **`getScaffoldCmd()`** - Creates and configures the `skaffold` cobra command with comprehensive flags for development customization
- **`runScaffold()`** - Command execution handler that initializes the scaffold service and runs the development workflow
- **`models.ScaffoldFlags`** - Configuration structure containing development-specific options like port, namespace, image settings, and sync directories

## Usage Example

```go
// Command line usage examples:
// openframe dev skaffold                    # Interactive setup
// openframe dev skaffold my-dev-cluster    # Named cluster
// openframe dev skaffold --port 3000 --namespace dev-ns

// The command supports various development flags:
cmd := getScaffoldCmd()
// Flags include:
// --port: Local development port (default: 8080)
// --namespace: Kubernetes namespace
// --image: Docker image for service
// --sync-local/sync-remote: Directory sync paths
// --skip-bootstrap: Skip cluster bootstrapping
// --helm-values: Custom Helm values file
```

The scaffold command orchestrates a complete development workflow including prerequisite checks, cluster bootstrapping with development settings, and live reloading capabilities for rapid development cycles.
