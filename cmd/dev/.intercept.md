Provides traffic interception capabilities for routing Kubernetes service traffic to local development environments using Telepresence. Supports both interactive cluster/service selection and direct command-line usage.

## Key Components

- `getInterceptCmd()` - Creates the cobra command with flags for port, namespace, mount, env-file, headers, etc.
- `runIntercept()` - Main entry point that handles both interactive and flag-based intercept modes
- `runInteractiveIntercept()` - Guided workflow for cluster selection and service setup
- `selectClusterForIntercept()` - Cluster selection UI integration
- `setKubectlContext()` - Sets kubectl context to the selected cluster

## Usage Example

```go
// Command-line usage with flags
// openframe dev intercept my-service --port 8080 --namespace production

// Interactive mode - prompts for cluster and service selection
// openframe dev intercept

// Programmatic usage
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Mount:     "/tmp/volumes",
    EnvFile:   ".env",
}

exec := executor.NewRealCommandExecutor(false, true)
service := intercept.NewService(exec, true)
err := service.StartIntercept("my-service", flags)
```

The command integrates with the cluster management system to ensure proper kubectl context and provides a complete Telepresence lifecycle management for local development workflows.
