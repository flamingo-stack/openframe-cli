Provides traffic interception functionality for redirecting Kubernetes service traffic to local development environments using Telepresence. This file implements both interactive and flag-based modes for setting up traffic interception.

## Key Components

- **`getInterceptCmd()`** - Creates the cobra command with flags for port, namespace, mounting, environment variables, and traffic filtering options
- **`runIntercept()`** - Main command handler that routes between interactive and flag-based execution modes
- **`runInteractiveIntercept()`** - Implements the interactive flow with cluster selection and service configuration
- **`selectClusterForIntercept()`** - Handles cluster selection using the cluster UI selector
- **`setKubectlContext()`** - Switches kubectl context to the selected k3d cluster

## Usage Example

```go
// Flag-based intercept
// openframe dev intercept my-service --port 8080 --namespace production

// Interactive mode (no service name specified)
// openframe dev intercept

// Advanced traffic filtering
flags := &models.InterceptFlags{
    Port:           8080,
    Namespace:      "default", 
    Global:         false,
    Header:         []string{"user-type=beta"},
    Mount:          "/tmp/volumes",
    EnvFile:        ".env",
}
```

The command integrates with kubectl providers, cluster utilities, and the intercept service to manage the complete Telepresence lifecycle from connection setup to traffic routing and cleanup.
