Provides Telepresence traffic interception functionality for routing Kubernetes service traffic to local development environments. Supports both interactive cluster/service selection and direct flag-based configuration.

## Key Components

- **getInterceptCmd()** - Creates the cobra command with intercept-specific flags (port, namespace, mount, env-file, etc.)
- **runIntercept()** - Main entry point that handles both interactive and flag-based intercept modes
- **runInteractiveIntercept()** - Orchestrates the full interactive flow: cluster selection, kubectl context setup, service selection, and intercept creation
- **selectClusterForIntercept()** - Handles cluster discovery and selection using the cluster UI service
- **setKubectlContext()** - Switches kubectl context to the selected k3d cluster

## Usage Example

```go
// Interactive mode - prompts for cluster and service selection
cmd := getInterceptCmd()
cmd.Execute() // openframe dev intercept

// Direct service intercept with flags
flags := &models.InterceptFlags{
    Port: 8080,
    Namespace: "my-namespace",
    Global: false,
    Header: []string{"user=developer"},
}
runIntercept(cmd, []string{"my-service"}, flags)

// Set kubectl context for k3d cluster
ctx := context.Background()
err := setKubectlContext(ctx, "my-cluster", true)
```

The command integrates with Telepresence to intercept traffic from Kubernetes services and redirect it to local development environments for debugging and testing.
