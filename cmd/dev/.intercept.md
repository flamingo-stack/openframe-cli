This file implements the traffic interception functionality for local development against Kubernetes clusters using Telepresence.

## Key Components

- `getInterceptCmd()` - Creates the cobra command with flags for port, namespace, mount options, and traffic filtering
- `runIntercept()` - Main handler that routes to either interactive or flag-based intercept modes
- `runInteractiveIntercept()` - Provides guided setup with cluster selection and service discovery
- `selectClusterForIntercept()` - Handles cluster selection using the existing cluster UI
- `setKubectlContext()` - Switches kubectl context to the selected cluster

## Usage Example

```go
// Command-line usage examples:
// openframe dev intercept my-service --port 8080 --namespace production
// openframe dev intercept --global --mount /tmp/volumes

// Programmatic usage:
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Global:    false,
}

exec := executor.NewRealCommandExecutor(false, true)
service := intercept.NewService(exec, true)
err := service.StartIntercept("my-service", flags)
```

The command supports both interactive mode (no service name provided) which walks through cluster and service selection, and direct mode with explicit service names and configuration flags. It integrates with the cluster management system to automatically configure kubectl context and validate connectivity before establishing traffic interception.
