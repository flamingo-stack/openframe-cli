Provides the `intercept` command for the OpenFrame CLI development toolkit. This command enables developers to intercept traffic from Kubernetes services and redirect it to their local development environment using Telepresence for real-time debugging and testing.

## Key Components

- **`getInterceptCmd()`** - Returns the cobra command definition with flags for port forwarding, namespaces, volume mounts, and traffic filtering
- **`runIntercept()`** - Main command handler that routes between interactive and flag-based execution modes
- **`runInteractiveIntercept()`** - Orchestrates the full interactive flow including cluster selection, kubectl context setup, and service configuration
- **`selectClusterForIntercept()`** - Handles cluster discovery and selection using the cluster UI service
- **`setKubectlContext()`** - Switches kubectl context to the selected k3d cluster

## Usage Example

```go
// Flag-based usage with all options
flags := &models.InterceptFlags{
    Port: 8080,
    Namespace: "production",
    Mount: "/tmp/volumes",
    EnvFile: ".env",
    Global: false,
    Header: []string{"x-dev-user=john"},
    Replace: true,
    RemotePortName: "http",
}

// Interactive mode - prompts for cluster and service selection
err := runInteractiveIntercept(ctx, verbose, dryRun)

// Direct service intercept
exec := executor.NewRealCommandExecutor(false, true)
service := intercept.NewService(exec, true)
err := service.StartIntercept("my-service", flags)
```
