This file implements the traffic interception command for the OpenFrame CLI development tools, allowing developers to redirect Kubernetes service traffic to their local development environment using Telepresence.

## Key Components

**Main Command Function:**
- `getInterceptCmd()` - Creates the intercept command with comprehensive flags for port forwarding, namespaces, volume mounting, and traffic filtering

**Core Execution Functions:**
- `runIntercept()` - Orchestrates both interactive and flag-based intercept modes
- `runInteractiveIntercept()` - Provides guided cluster and service selection with UI prompts
- `selectClusterForIntercept()` - Handles cluster discovery and selection using shared cluster utilities
- `setKubectlContext()` - Switches kubectl context to the selected k3d cluster

**Command Flags:**
- Port forwarding (`--port`), namespace selection (`--namespace`)
- Volume mounting (`--mount`), environment variables (`--env-file`)
- Traffic filtering (`--header`, `--global`), intercept management (`--replace`)

## Usage Example

```go
// Interactive mode - prompts for cluster and service selection
cmd := getInterceptCmd()
cmd.Run(cmd, []string{})

// Direct service intercept with specific configuration
flags := &models.InterceptFlags{
    Port: 8080,
    Namespace: "production",
    Mount: "/tmp/volumes",
    EnvFile: ".env",
}
runIntercept(cmd, []string{"my-service"}, flags)

// Set kubectl context for k3d cluster
ctx := context.Background()
err := setKubectlContext(ctx, "my-cluster", true)
```

The intercept functionality integrates with Telepresence to enable real-time debugging by routing cluster traffic to local development servers, supporting both automated service discovery and manual configuration.
