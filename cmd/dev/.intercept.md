Provides Telepresence-based traffic interception functionality for redirecting Kubernetes service traffic to local development environments.

## Key Components

- **getInterceptCmd()** - Creates the main `intercept` command with configurable flags for port forwarding, namespaces, volume mounting, and traffic filtering
- **runIntercept()** - Entry point that delegates to interactive or flag-based intercept modes based on arguments
- **runInteractiveIntercept()** - Orchestrates the full interactive flow: cluster selection, kubectl context switching, service discovery, and intercept setup
- **selectClusterForIntercept()** - Handles cluster selection using the shared cluster UI service
- **setKubectlContext()** - Switches kubectl context to the selected k3d cluster

## Usage Example

```go
// Create the intercept command
interceptCmd := getInterceptCmd()

// Example flag-based usage (called by cobra)
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Global:    false,
}
err := runIntercept(cmd, []string{"my-service"}, flags)

// Example interactive mode (no service name provided)
err := runIntercept(cmd, []string{}, flags)
```

The command supports both interactive mode (with cluster and service selection) and direct flag-based execution, automatically managing the Telepresence lifecycle for local development traffic interception.
