Manages Kubernetes service traffic interception using Telepresence to route cluster traffic to local development environments. Provides both interactive and command-line modes for setting up traffic interception.

## Key Components

**Main Functions:**
- `getInterceptCmd()` - Creates the cobra command with flags for port, namespace, mount options, and traffic filtering
- `runIntercept()` - Handles both interactive and flag-based intercept execution modes
- `runInteractiveIntercept()` - Provides guided cluster and service selection with UI prompts
- `selectClusterForIntercept()` - Lists and allows selection from available k3d clusters
- `setKubectlContext()` - Switches kubectl context to the selected cluster

**Key Services:**
- `intercept.Service` - Manages the actual Telepresence traffic interception
- `ui.Service` - Provides interactive service and port selection interface
- `kubectl.Provider` - Handles Kubernetes cluster connectivity validation

## Usage Example

```go
// Command-line usage with flags
flags := &models.InterceptFlags{
    Port:           8080,
    Namespace:      "default",
    RemotePortName: "http",
    Global:         false,
}

exec := executor.NewRealCommandExecutor(false, true)
service := intercept.NewService(exec, true)
err := service.StartIntercept("my-service", flags)

// Interactive mode - prompts for cluster and service selection
err := runInteractiveIntercept(context.Background(), true, false)
```
