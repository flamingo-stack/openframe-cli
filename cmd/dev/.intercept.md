Provides traffic interception functionality for routing Kubernetes cluster traffic to local development environments using Telepresence. Supports both interactive cluster/service selection and direct command-line execution with comprehensive configuration options.

## Key Components

### Commands
- **`getInterceptCmd()`** - Creates the main intercept cobra command with flags for port, namespace, mounting, environment files, and traffic filtering
- **`runIntercept()`** - Main execution handler that routes to interactive or flag-based modes based on arguments

### Interactive Flow
- **`runInteractiveIntercept()`** - Full interactive workflow including cluster selection, kubectl context switching, and service configuration
- **`selectClusterForIntercept()`** - Cluster selection UI using the shared cluster selector component
- **`setKubectlContext()`** - Switches kubectl context to the selected k3d cluster

### Integration Points
- Uses `intercept.Service` for Telepresence lifecycle management
- Integrates with `kubectl.Provider` for Kubernetes operations
- Leverages `ui.Service` for interactive service selection
- Employs shared cluster utilities and UI components

## Usage Example

```go
// Command-line usage with flags
cmd := getInterceptCmd()
flags := []string{
    "my-service",
    "--port", "8080", 
    "--namespace", "development",
    "--mount", "/tmp/volumes",
    "--env-file", ".env",
    "--header", "x-debug=true"
}

// Interactive mode (no service name)
err := runIntercept(cmd, []string{}, &models.InterceptFlags{})

// Direct service intercept
err := runIntercept(cmd, []string{"my-service"}, &models.InterceptFlags{
    Port: 8080,
    Namespace: "default",
    Global: false,
})
```
