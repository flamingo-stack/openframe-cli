This file implements traffic interception functionality for local Kubernetes development using Telepresence. It allows developers to redirect cluster traffic to their local machine for real-time debugging and development.

## Key Components

- **`getInterceptCmd()`** - Creates and configures the `intercept` cobra command with flags for port, namespace, mount paths, environment files, and traffic filtering
- **`runIntercept()`** - Main entry point that handles both interactive and flag-based intercept modes
- **`runInteractiveIntercept()`** - Provides interactive cluster selection, service discovery, and intercept configuration
- **`selectClusterForIntercept()`** - Handles cluster selection using the shared cluster UI service
- **`setKubectlContext()`** - Switches kubectl context to the selected cluster for proper connection

## Usage Example

```go
// Interactive mode - prompts for cluster and service selection
openframe dev intercept

// Direct service intercept with port forwarding
openframe dev intercept my-service --port 8080 --namespace production

// Advanced intercept with volume mounting and environment variables
openframe dev intercept api-service \
    --port 3000 \
    --mount /tmp/volumes \
    --env-file .env \
    --header "x-user-id=123"

// Global traffic intercept (all requests)
openframe dev intercept web-service --port 8080 --global
```

The intercept workflow includes cluster selection, kubectl context switching, service discovery, and Telepresence-based traffic routing to enable seamless local development against live cluster services.
