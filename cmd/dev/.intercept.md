This file implements the intercept command for routing Kubernetes cluster traffic to a local development environment using Telepresence.

## Key Components

- **`getInterceptCmd()`** - Creates the cobra command with flags for port, namespace, mount, env-file, headers, and other intercept options
- **`runIntercept()`** - Main command handler that routes to either interactive or flag-based mode
- **`runInteractiveIntercept()`** - Interactive flow with cluster selection and service discovery
- **`selectClusterForIntercept()`** - UI for selecting target cluster from available clusters
- **`setKubectlContext()`** - Sets kubectl context to the selected cluster

## Usage Example

```go
// Command-line usage examples:
// Interactive mode with cluster selection
openframe dev intercept

// Direct service intercept with flags
openframe dev intercept my-service --port 8080 --namespace production

// Advanced intercept with volume mounting and environment
openframe dev intercept api-service \
  --port 3000 \
  --mount /tmp/volumes \
  --env-file .env \
  --header "x-dev-user=alice"

// Global traffic intercept (all requests)
openframe dev intercept frontend --port 8080 --global
```

The command integrates with cluster management, kubectl providers, and Telepresence services to provide seamless local development against remote Kubernetes services.
