This file provides the CLI command definition for the OpenFrame bootstrap functionality, which sets up a complete OpenFrame environment in a single command.

## Key Components

- **GetBootstrapCmd()**: Returns a configured Cobra command for bootstrapping OpenFrame environments
- **Command Configuration**: Defines usage patterns, help text, and command-line flags
- **Service Integration**: Delegates execution to the internal bootstrap service

## Usage Example

```go
package main

import (
    "github.com/flamingo-stack/openframe-cli/cmd/bootstrap"
    "github.com/spf13/cobra"
)

func main() {
    rootCmd := &cobra.Command{Use: "openframe"}
    
    // Add the bootstrap command to your CLI
    bootstrapCmd := bootstrap.GetBootstrapCmd()
    rootCmd.AddCommand(bootstrapCmd)
    
    rootCmd.Execute()
}
```

The command supports various execution modes:

```bash
# Interactive mode (default)
openframe bootstrap

# With custom cluster name
openframe bootstrap my-cluster

# CI/CD friendly mode
openframe bootstrap --deployment-mode=oss-tenant --non-interactive

# Verbose logging
openframe bootstrap --verbose --deployment-mode=saas-shared
```

The bootstrap command combines cluster creation and chart installation into a streamlined setup process, supporting three deployment modes (oss-tenant, saas-tenant, saas-shared) with options for interactive or automated execution.
