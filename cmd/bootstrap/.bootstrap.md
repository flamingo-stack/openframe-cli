This file provides the CLI command definition for bootstrapping a complete OpenFrame environment, combining cluster creation and chart installation into a single streamlined command.

## Key Components

- **GetBootstrapCmd()**: Returns a configured Cobra command that orchestrates the complete OpenFrame setup process
- **Command Configuration**: Defines usage patterns, help text, and argument validation for the bootstrap command
- **Flag Definitions**: Sets up command-line flags for deployment mode, interactivity, and verbosity controls

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/cmd/bootstrap"
    "github.com/spf13/cobra"
)

// Add bootstrap command to root command
func addCommands(rootCmd *cobra.Command) {
    bootstrapCmd := bootstrap.GetBootstrapCmd()
    rootCmd.AddCommand(bootstrapCmd)
}

// The command can be executed with various options:
// openframe bootstrap                                    # Interactive mode
// openframe bootstrap my-cluster                        # Custom cluster name  
// openframe bootstrap --deployment-mode=oss-tenant     # Skip deployment selection
// openframe bootstrap --non-interactive --verbose      # Full automation with detailed logs
```

The bootstrap command serves as a convenience wrapper that executes both `openframe cluster create` and `openframe chart install` sequentially, providing a one-command solution for complete OpenFrame environment setup. It supports interactive and non-interactive modes, multiple deployment types, and configurable verbosity levels.
