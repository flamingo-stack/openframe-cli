This file provides standardized testing patterns for validating Cobra CLI commands in the OpenFrame project. It implements a comprehensive test suite that ensures commands follow consistent structure and behavior patterns.

## Key Components

- **`TestClusterCommand`** - Main test orchestrator that runs all standard tests for cluster commands
- **`testCommandStructure`** - Validates basic command properties like name, descriptions, and RunE function
- **`testCommandFlags`** - Tests help flag functionality and invalid flag handling
- **`testCommandCLI`** - Validates command argument parsing and validation
- **`testCommandExecution`** - Tests basic command execution including dry-run and help scenarios

## Usage Example

```go
func TestCreateCommand(t *testing.T) {
    setupFunc := func() {
        // Initialize mocks, set environment variables
        mockClient := &MockClusterClient{}
        os.Setenv("OPENFRAME_CONFIG", "/tmp/test-config")
    }
    
    teardownFunc := func() {
        // Clean up test resources
        os.Unsetenv("OPENFRAME_CONFIG")
    }
    
    cmdFunc := func() *cobra.Command {
        return createCommand() // Your command constructor
    }
    
    TestClusterCommand(t, "create", cmdFunc, setupFunc, teardownFunc)
}
```

The test pattern systematically validates that commands have proper structure, handle flags correctly, validate arguments appropriately, and execute without critical errors. This ensures consistency across all CLI commands in the project.
