A comprehensive test pattern utility that provides standardized testing for cluster commands in the OpenFrame CLI, ensuring consistent command structure, flags, arguments, and execution behavior across all cluster-related commands.

## Key Components

### TestClusterCommand
- `TestClusterCommand` - Main entry point that runs all standard test patterns for cluster commands
- `testCommandStructure` - Validates command naming, descriptions, and basic structure requirements
- `testCommandFlags` - Tests flag handling including help and invalid flag scenarios  
- `testCommandCLI` - Validates command line argument acceptance and validation
- `testCommandExecution` - Tests basic execution paths including dry-run and help functionality

## Usage Example

```go
func TestCreateCommand(t *testing.T) {
    setup := func() {
        // Initialize mocks, set environment variables
        mockClient := &MockClusterClient{}
        // Setup other test dependencies
    }
    
    teardown := func() {
        // Clean up test resources
        os.Unsetenv("TEST_VAR")
    }
    
    commandFunc := func() *cobra.Command {
        return NewCreateCommand()
    }
    
    // Run all standard cluster command tests
    TestClusterCommand(t, "create", commandFunc, setup, teardown)
}

// Test a root command (no RunE function)
func TestRootClusterCommand(t *testing.T) {
    TestClusterCommand(t, "cluster", NewClusterCommand, nil, nil)
}
```

The pattern automatically validates that commands have proper structure (name, descriptions), handle flags correctly, validate arguments appropriately, and support standard CLI patterns like `--help` and `--dry-run` flags.
