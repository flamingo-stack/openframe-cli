A fluent assertion library for testing command output in Go, providing chainable methods to verify stdout, stderr, and error conditions.

## Key Components

- **`AssertCommandOutput`** - Main struct that wraps command execution results (stdout, stderr, error) with fluent assertion methods
- **`NewAssertCommandOutput()`** - Constructor that creates a new assertion instance from command output
- **Error Assertions** - `NoError()`, `HasError()`, `ErrorContains()` for validating error conditions
- **Output Assertions** - `StdoutContains()`, `StderrContains()`, `OutputContains()` for checking command output
- **Helper Functions** - `AssertCommandSuccess()` and `AssertCommandFailure()` for common test patterns

## Usage Example

```go
func TestCommandExecution(t *testing.T) {
    // Execute a command and capture output
    stdout, stderr, err := executeCommand("ls", "-la")
    
    // Fluent assertion chaining
    NewAssertCommandOutput(t, stdout, stderr, err).
        NoError().
        StdoutContains("total").
        StderrEmpty()
    
    // Using helper functions
    AssertCommandSuccess(t, stdout, stderr, err).
        ContainsAll("file1.txt", "file2.txt")
    
    // Testing error conditions
    stdout, stderr, err = executeCommand("invalid-command")
    AssertCommandFailure(t, stdout, stderr, err).
        ErrorContains("command not found").
        StderrContainsAny("not found", "does not exist")
}
```

This utility simplifies command output testing by providing a readable, chainable API that reduces boilerplate test code and improves test maintainability.
