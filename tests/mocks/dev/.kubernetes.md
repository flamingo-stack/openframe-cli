This file provides a mock implementation of a Kubernetes client for testing UI components in the OpenFrame CLI development environment.

## Key Components

- **MockKubernetesClient** - Main mock struct that simulates Kubernetes API operations
- **NewMockKubernetesClient()** - Constructor that creates a mock client with sample data
- **Namespace Operations** - `GetNamespaces()`, `ValidateNamespace()`, `AddNamespace()`, `ClearNamespaces()`
- **Service Operations** - `GetServices()`, `GetService()`, `ValidateService()`, `AddService()`, `GetServicesByType()`
- **Test Controls** - Methods to simulate failures and reset state for testing scenarios

## Usage Example

```go
// Create a mock client with sample data
client := NewMockKubernetesClient()

// Get available namespaces
namespaces, err := client.GetNamespaces(context.Background())
// Returns: ["default", "production", "staging", "development", "kube-system"]

// Get services in a namespace
services, err := client.GetServices(context.Background(), "default")
// Returns services like "my-api" and "my-frontend"

// Test error scenarios
client.SetShouldFailServices(true)
_, err = client.GetServices(context.Background(), "default")
// Returns: mock error for testing

// Add custom test data
client.AddNamespace("test-env")
client.AddService("test-env", intercept.ServiceInfo{
    Name:      "test-service",
    Namespace: "test-env",
    Type:      "ClusterIP",
    Ports:     []intercept.ServicePort{{Port: 8080}},
})
```

The mock comes pre-populated with realistic test data across multiple namespaces and provides methods to simulate both success and failure scenarios for comprehensive testing.
