This file provides a mock implementation of a Kubernetes client specifically designed for testing UI components and development workflows in the OpenFrame CLI tool.

## Key Components

- **MockKubernetesClient** - Main mock struct with predefined namespaces and services
- **NewMockKubernetesClient()** - Factory function that creates mock client with sample data
- **Namespace operations** - GetNamespaces, ValidateNamespace, AddNamespace, ClearNamespaces
- **Service operations** - GetServices, GetService, ValidateService, AddService, GetServicesByType
- **Testing controls** - SetShouldFailNamespaces, SetShouldFailServices, Reset methods
- **Sample data** - Predefined namespaces (default, production, staging) with realistic service configurations

## Usage Example

```go
// Create mock client with sample data
client := NewMockKubernetesClient()

// Get available namespaces
namespaces, err := client.GetNamespaces(context.Background())
// Returns: ["default", "production", "staging", "development", "kube-system"]

// Get services in a namespace
services, err := client.GetServices(context.Background(), "production")
// Returns services like "api-service" and "web-service"

// Test error scenarios
client.SetShouldFailNamespaces(true)
_, err = client.GetNamespaces(context.Background())
// err will be "mock error: failed to list namespaces"

// Add custom test data
client.AddNamespace("test-env")
client.AddService("test-env", intercept.ServiceInfo{
    Name: "my-test-service",
    Namespace: "test-env",
    Type: "ClusterIP",
})
```

The mock includes realistic sample services with different port configurations and supports both success and failure testing scenarios through controllable error flags.
