A mock implementation of Kubernetes client operations for testing UI components and development workflows, providing simulated namespace and service management without requiring an actual Kubernetes cluster connection.

## Key Components

- **MockKubernetesClient** - Main mock struct that simulates Kubernetes API operations with configurable failure modes for testing
- **NewMockKubernetesClient()** - Factory function that creates a pre-populated mock client with sample namespaces and services
- **Namespace Operations** - GetNamespaces(), ValidateNamespace(), AddNamespace(), ClearNamespaces()
- **Service Operations** - GetServices(), GetService(), ValidateService(), AddService(), GetServicesByType()
- **Testing Utilities** - SetShouldFailNamespaces(), SetShouldFailServices(), Reset(), ClearServices()

## Usage Example

```go
// Create a mock client with sample data
mockClient := NewMockKubernetesClient()

// Get available namespaces
namespaces, err := mockClient.GetNamespaces(context.Background())
if err != nil {
    log.Fatal(err)
}
fmt.Println("Namespaces:", namespaces) // ["default", "production", "staging", ...]

// Get services in a specific namespace
services, err := mockClient.GetServices(context.Background(), "default")
if err != nil {
    log.Fatal(err)
}

// Add custom test data
mockClient.AddNamespace("testing")
mockClient.AddService("testing", intercept.ServiceInfo{
    Name:      "test-api",
    Namespace: "testing",
    Type:      "ClusterIP",
    Ports: []intercept.ServicePort{
        {Name: "http", Port: 8080, Protocol: "TCP"},
    },
})

// Test error conditions
mockClient.SetShouldFailServices(true)
_, err = mockClient.GetServices(context.Background(), "default")
// Returns: "mock error: failed to list services"
```
