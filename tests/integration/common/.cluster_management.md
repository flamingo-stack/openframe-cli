This file provides utilities for managing k3d Kubernetes clusters during testing, including creation, cleanup, and resource management operations.

## Key Components

- **GenerateTestClusterName()** - Creates unique cluster names using Unix timestamps
- **CreateTestCluster(name)** - Creates a k3d cluster with one agent and 60-second timeout
- **DeleteTestCluster(name)** - Removes a specific test cluster
- **ClusterExists(name)** - Checks cluster existence using k3d list command
- **StopTestCluster(name)** - Stops a running test cluster
- **CleanupTestCluster(name)** - Comprehensive cleanup of cluster and Docker resources
- **CleanupAllTestClusters()** - Removes all test clusters matching specific patterns
- **cleanupDockerResources()** - Removes orphaned k3d Docker networks and containers
- **cleanupClusterSpecificResources(name)** - Cleans up Docker resources for a specific cluster

## Usage Example

```go
// Create a test cluster
clusterName := GenerateTestClusterName()
err := CreateTestCluster(clusterName)
if err != nil {
    log.Fatal("Failed to create cluster:", err)
}

// Check if cluster exists
exists, err := ClusterExists(clusterName)
if err != nil {
    log.Fatal("Failed to check cluster:", err)
}

// Clean up after testing
defer CleanupTestCluster(clusterName)

// Or clean up all test clusters
CleanupAllTestClusters()
```

The module handles Docker resource cleanup automatically and includes performance optimizations like reduced timeouts and direct k3d command usage to avoid CLI overhead.
