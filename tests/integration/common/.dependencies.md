A testing utility package that provides dependency checking for integration tests, allowing tests to gracefully skip when required external tools are unavailable.

## Key Components

- **`Dependency` struct**: Represents an external tool dependency with name, check command, and installation message
- **`IsAvailable()` method**: Executes the check command to verify if a dependency is installed and accessible
- **`RequireOrSkip()` method**: Logs a warning and skips the test if the dependency is not available
- **Predefined dependencies**: Ready-to-use instances for Docker, k3d, kubectl, and helm
- **Convenience functions**: `RequireClusterDependencies()`, `RequireK8sDependencies()`, and `RequireAllDependencies()` for common dependency groups

## Usage Example

```go
func TestKubernetesDeployment(t *testing.T) {
    // Check all cluster dependencies before running test
    common.RequireAllDependencies(t)
    
    // Test proceeds only if Docker, k3d, kubectl, and helm are available
    // ... actual test implementation
}

func TestDockerOnly(t *testing.T) {
    // Check single dependency
    common.Docker.RequireOrSkip(t, "Docker required for container tests")
    
    // ... test implementation
}

func TestCustomTool(t *testing.T) {
    customDep := &common.Dependency{
        Name:       "custom-tool not found",
        CheckCmd:   []string{"custom-tool", "--version"},
        InstallMsg: "Install custom-tool from https://example.com",
    }
    customDep.RequireOrSkip(t, "custom-tool required")
    
    // ... test implementation
}
```

This approach ensures integration tests fail gracefully with informative messages when dependencies are missing, rather than producing cryptic error messages.
