Provides dependency checking utilities for integration tests, allowing tests to gracefully skip when required external tools are unavailable.

## Key Components

- **`Dependency`** - Struct representing an external dependency with name, check command, and installation message
- **`IsAvailable()`** - Method to verify if a dependency is installed and accessible
- **`RequireOrSkip()`** - Method to check dependency and skip test with warning if missing
- **Predefined Dependencies** - Ready-to-use dependency definitions for Docker, k3d, kubectl, and Helm
- **Convenience Functions** - `RequireClusterDependencies()`, `RequireK8sDependencies()`, and `RequireAllDependencies()` for common dependency groups

## Usage Example

```go
func TestKubernetesIntegration(t *testing.T) {
    // Check all cluster dependencies before running test
    RequireClusterDependencies(t)
    
    // Test will be skipped if Docker or k3d are not available
    // with helpful installation messages logged
    
    // Your integration test code here...
}

func TestCustomDependency(t *testing.T) {
    customDep := &Dependency{
        Name:       "custom-tool not available",
        CheckCmd:   []string{"custom-tool", "--version"},
        InstallMsg: "Install custom-tool with: npm install -g custom-tool",
    }
    
    customDep.RequireOrSkip(t, "custom-tool missing, skipping test")
    
    // Test continues only if custom-tool is available
}
```
