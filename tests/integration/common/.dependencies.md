Provides dependency checking functionality for integration tests, allowing tests to gracefully skip when required external tools are unavailable.

## Key Components

**Types:**
- `Dependency` - Represents an external dependency with availability checking and installation guidance

**Methods:**
- `IsAvailable()` - Checks if dependency is available by running a command
- `RequireOrSkip()` - Skips test with warning message if dependency is missing

**Predefined Dependencies:**
- `Docker` - Docker daemon availability
- `K3d` - k3d cluster tool
- `Kubectl` - Kubernetes CLI
- `Helm` - Helm package manager

**Convenience Functions:**
- `RequireClusterDependencies()` - Checks Docker and k3d
- `RequireK8sDependencies()` - Checks kubectl and Helm
- `RequireAllDependencies()` - Checks all dependencies

## Usage Example

```go
func TestKubernetesIntegration(t *testing.T) {
    // Check specific dependency
    Docker.RequireOrSkip(t, "Docker required for container tests")
    
    // Check multiple related dependencies
    RequireClusterDependencies(t)
    
    // Check all dependencies at once
    RequireAllDependencies(t)
    
    // Create custom dependency
    customDep := &Dependency{
        Name:       "custom tool not found",
        CheckCmd:   []string{"custom-tool", "--version"},
        InstallMsg: "Install custom-tool with: brew install custom-tool",
    }
    customDep.RequireOrSkip(t, "Custom tool required")
    
    // Test continues only if all dependencies are available
    // ... integration test code ...
}
```
