This file provides a dependency checking system for integration tests, allowing tests to gracefully skip when required external tools are unavailable.

## Key Components

- **`Dependency` struct**: Defines a dependency with name, check command, and installation message
- **`IsAvailable()`**: Method to verify if a dependency is installed and functional
- **`RequireOrSkip()`**: Method that skips tests with warnings when dependencies are missing
- **Predefined dependencies**: Pre-configured `Docker`, `K3d`, `Kubectl`, and `Helm` dependency checkers
- **Convenience functions**: `RequireClusterDependencies()`, `RequireK8sDependencies()`, and `RequireAllDependencies()` for common dependency groups

## Usage Example

```go
func TestDockerIntegration(t *testing.T) {
    // Check single dependency
    Docker.RequireOrSkip(t, "Docker required for container tests")
    
    // Test code that requires Docker...
}

func TestK8sCluster(t *testing.T) {
    // Check multiple related dependencies
    RequireClusterDependencies(t)
    
    // Test code that requires Docker and k3d...
}

func TestFullDeployment(t *testing.T) {
    // Check all dependencies at once
    RequireAllDependencies(t)
    
    // Test code that requires all tools...
}

// Create custom dependency
customTool := &Dependency{
    Name:       "custom-tool not found",
    CheckCmd:   []string{"custom-tool", "--version"},
    InstallMsg: "Install from https://example.com/custom-tool",
}

func TestCustomTool(t *testing.T) {
    customTool.RequireOrSkip(t, "Custom tool required")
    // Test code...
}
```

The system prevents integration test failures due to missing development tools while providing clear installation guidance through logged warnings.
