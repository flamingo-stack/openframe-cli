This file contains comprehensive unit tests for the SkaffoldUI functionality, testing the discovery, categorization, and management of Skaffold YAML configuration files across different project directories.

## Key Components

- **Test Functions**: Tests for `NewSkaffoldUI`, `findSkaffoldYamlFiles`, `extractServiceName`, and `categorizeSkaffoldFiles` methods
- **Structure Tests**: Validation of `SkaffoldFile` and `SkaffoldCategory` struct definitions
- **Error Handling**: Tests for `ErrNoSkaffoldFiles` and edge cases like non-existent paths
- **File System Tests**: Creates temporary directory structures to simulate real project layouts

## Usage Example

```go
// Run all tests
go test ./ui -v

// Run specific test
go test -run TestNewSkaffoldUI ./ui

// Test with coverage
go test -cover ./ui

// Example test setup pattern from the file
func TestExample(t *testing.T) {
    // Create temporary test environment
    tmpDir := t.TempDir()
    
    // Set up test files
    testFile := filepath.Join(tmpDir, "skaffold.yaml")
    err := os.WriteFile(testFile, []byte("# Test content"), 0644)
    require.NoError(t, err)
    
    // Test the functionality
    ui := NewSkaffoldUI(false)
    files := ui.findSkaffoldYamlFiles(tmpDir)
    
    // Assertions
    assert.NotEmpty(t, files)
}
```

The tests cover various scenarios including different directory structures (openframe services, integrated tools, client apps), file extensions (.yaml/.yml), service name extraction logic, and proper categorization with sorting.
