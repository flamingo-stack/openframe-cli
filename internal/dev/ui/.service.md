Service manages the user interface for development workflows, providing interactive Kubernetes service interception and command execution capabilities.

## Key Components

- **Service**: Main service struct that coordinates UI interactions and command execution
- **NewService()**: Factory function for creating a service with Kubernetes and service clients
- **NewServiceWithExecutor()**: Factory function for creating a service with command executor for interactive workflows
- **InteractiveInterceptSetup()**: Prompts user for service interception configuration
- **RunFullInteractiveIntercept()**: Executes complete interactive intercept workflow from context checking to service startup
- **InterceptSetup**: Configuration struct containing service name, namespace, ports, and Kubernetes port details

## Usage Example

```go
// Create service with executor for interactive workflows
executor := executor.NewCommandExecutor()
service := ui.NewServiceWithExecutor(executor, true)

// Run full interactive intercept workflow
ctx := context.Background()
err := service.RunFullInteractiveIntercept(ctx)
if err != nil {
    log.Fatal("Failed to run intercept:", err)
}

// Or create service with specific clients
kubectlProvider := kubectl.NewProvider(executor, false)
service = ui.NewService(kubectlProvider, kubectlProvider)

// Get intercept configuration interactively
setup, err := service.InteractiveInterceptSetup(ctx)
if err != nil {
    log.Fatal("Failed to setup intercept:", err)
}
```

The service handles kubectl context validation, cluster connectivity checks, and converts user selections into appropriate intercept flags for service forwarding.
