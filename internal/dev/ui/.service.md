Provides a unified interface for managing interactive development UI operations, particularly for setting up Kubernetes service intercepts through user prompts and workflow orchestration.

## Key Components

- **Service**: Main service struct that coordinates intercept UI operations and command execution
- **NewService/NewServiceWithExecutor**: Factory functions for creating service instances with different configurations
- **InteractiveInterceptSetup**: Guides users through service selection, port configuration, and intercept setup
- **RunFullInteractiveIntercept**: Complete workflow for kubectl validation, cluster connection, and intercept execution
- **InterceptSetup**: Configuration struct containing service details and port mappings

## Usage Example

```go
// Create service with executor for interactive workflows
executor := &executor.RealCommandExecutor{}
service := NewServiceWithExecutor(executor, true)

// Run complete interactive intercept workflow
ctx := context.Background()
if err := service.RunFullInteractiveIntercept(ctx); err != nil {
    log.Fatal("Failed to start intercept:", err)
}

// Or use step-by-step approach
setup, err := service.InteractiveInterceptSetup(ctx)
if err != nil {
    log.Fatal("Setup failed:", err)
}

fmt.Printf("Intercepting %s:%d -> localhost:%d\n", 
    setup.ServiceName, setup.KubernetesPort.Port, setup.LocalPort)
```

The service handles kubectl context validation, cluster connectivity checks, and converts user input into intercept configurations for seamless development workflows.
