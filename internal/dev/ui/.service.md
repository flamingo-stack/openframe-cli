This service provides a unified interface for development UI interactions, primarily focused on interactive Kubernetes service intercept workflows with kubectl integration.

## Key Components

**Service**: Main service struct that orchestrates UI interactions for development workflows
- `interceptUI` - Handler for intercept-specific UI operations
- `executor` - Command executor for running kubectl and other commands
- `verbose` - Controls logging verbosity

**InterceptSetup**: Configuration struct containing intercept parameters (service name, namespace, ports)

**Constructor Functions**:
- `NewService()` - Creates service with intercept UI capabilities
- `NewServiceWithExecutor()` - Creates service with command execution support

**Core Methods**:
- `InteractiveInterceptSetup()` - Prompts user for intercept configuration
- `RunFullInteractiveIntercept()` - Complete interactive intercept workflow
- `checkKubectlContexts()` - Validates kubectl availability and context
- `convertSetupToFlags()` - Converts UI setup to intercept flags

## Usage Example

```go
// Create service with executor for interactive workflows
executor := executor.NewCommandExecutor()
service := ui.NewServiceWithExecutor(executor, true)

// Run complete interactive intercept
ctx := context.Background()
if err := service.RunFullInteractiveIntercept(ctx); err != nil {
    log.Fatal(err)
}

// Or use step-by-step setup
setup, err := service.InteractiveInterceptSetup(ctx)
if err != nil {
    log.Fatal(err)
}
fmt.Printf("Intercepting %s:%d -> localhost:%d\n", 
    setup.ServiceName, setup.KubernetesPort.Port, setup.LocalPort)
```
