Provides a UI component for discovering and selecting Skaffold configuration files across different service categories in OpenFrame projects. The module recursively scans directories to find `skaffold.yaml` files and organizes them by service type for user selection.

## Key Components

- **SkaffoldUI**: Main UI handler with verbose mode support
- **ServiceSelection**: Represents a selected service with name, file path, and directory
- **SkaffoldCategory**: Groups related Skaffold files by project area (OpenFrame Services, Integrated Tools, Client Applications)
- **SkaffoldFile**: Individual Skaffold configuration file representation

## Usage Example

```go
// Create a new Skaffold UI instance
skaffoldUI := NewSkaffoldUI(true) // verbose mode enabled

// Discover and prompt user to select a service
selection, err := skaffoldUI.DiscoverAndSelectService()
if err != nil {
    if err == ErrNoSkaffoldFiles {
        log.Println("No Skaffold files found in project")
        return
    }
    log.Fatalf("Service selection failed: %v", err)
}

// Use the selected service
fmt.Printf("Selected: %s\n", selection.ServiceName)
fmt.Printf("Path: %s\n", selection.FilePath)
fmt.Printf("Directory: %s\n", selection.Directory)
```

The component automatically categorizes services into OpenFrame Services, Integrated Tools (like Authentik, FleetDM, TacticalRMM), Client Applications, and Other Services, providing a structured selection interface with search filtering support.
