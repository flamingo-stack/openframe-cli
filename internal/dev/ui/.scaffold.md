Provides skaffold service discovery and selection functionality for the OpenFrame CLI, allowing users to find and choose from available skaffold.yaml configuration files in their project directory.

## Key Components

- **ServiceSelection**: Struct containing selected service details (name, path, directory)
- **SkaffoldUI**: Main struct for handling skaffold configuration discovery and user interaction
- **SkaffoldCategory**: Represents categorized groups of skaffold files for organization
- **SkaffoldFile**: Represents individual skaffold configuration files

## Key Methods

- `NewSkaffoldUI(verbose bool)`: Creates a new SkaffoldUI instance
- `DiscoverAndSelectService()`: Main method that finds skaffold files and prompts user selection
- `findSkaffoldYamlFiles(rootPath string)`: Recursively searches for skaffold.yaml/yml files
- `categorizeSkaffoldFiles()`: Organizes files by location (OpenFrame services, tools, clients, other)
- `extractServiceName()`: Derives clean service names from file paths

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/commands/deploy/ui"
)

// Create skaffold UI instance
skaffoldUI := ui.NewSkaffoldUI(true) // verbose mode

// Discover and select a service
selection, err := skaffoldUI.DiscoverAndSelectService()
if err != nil {
    if err == ui.ErrNoSkaffoldFiles {
        // Handle case where no skaffold files found
        return
    }
    // Handle other errors
    return
}

// Use the selected service
fmt.Printf("Selected: %s at %s\n", selection.ServiceName, selection.FilePath)
```

The UI automatically categorizes services into OpenFrame Services, Integrated Tools, Client Applications, and Other Services, providing an organized selection experience with search filtering support.
