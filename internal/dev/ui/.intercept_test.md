This file contains unit tests for the InterceptUI component, which manages Kubernetes service interception functionality in a development environment.

## Key Components

- **TestNewInterceptUI**: Tests the constructor for InterceptUI, verifying proper initialization with Kubernetes and service clients
- **TestInterceptUI_findServiceInCluster**: Tests service discovery across namespaces with various scenarios including existing services in different namespaces and non-existent services
- **TestInterceptUI_validatePort**: Tests port validation logic for network port numbers (1-65535 range)

## Usage Example

```go
func TestMyInterceptUIFeature(t *testing.T) {
    // Initialize test environment
    testutil.InitializeTestMode()
    client := devMocks.NewMockKubernetesClient()
    
    // Create InterceptUI instance
    ui := NewInterceptUI(client, client)
    
    // Test service discovery
    ctx := context.Background()
    serviceInfo, err := ui.findServiceInCluster(ctx, "my-service")
    assert.NoError(t, err)
    assert.True(t, serviceInfo.Found)
    
    // Test port validation
    err = ui.validatePort("8080")
    assert.NoError(t, err)
}
```

The tests use mock Kubernetes clients and cover successful service lookups, missing services, and port validation edge cases to ensure robust intercept functionality.
