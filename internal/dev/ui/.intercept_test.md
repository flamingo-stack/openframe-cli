This file contains unit tests for the InterceptUI component, which provides user interface functionality for Kubernetes service interception in the Openframe CLI development tools.

## Key Components

- **`TestNewInterceptUI`** - Tests the constructor for InterceptUI, verifying proper initialization with Kubernetes and service clients
- **`TestInterceptUI_findServiceInCluster`** - Tests service discovery functionality across different namespaces with various port configurations
- **`TestInterceptUI_validatePort`** - Tests port number validation logic for acceptable port ranges (1-65535)

## Usage Example

```go
// Run all intercept UI tests
go test -v ./internal/dev/ui/intercept_test.go

// Run specific test
go test -run TestInterceptUI_findServiceInCluster

// The tests validate InterceptUI behavior:
func TestNewInterceptUI(t *testing.T) {
    client := devMocks.NewMockKubernetesClient()
    ui := NewInterceptUI(client, client)
    
    assert.NotNil(t, ui)
    assert.Equal(t, client, ui.kubernetesClient)
}

// Service discovery testing with mock data
func TestInterceptUI_findServiceInCluster(t *testing.T) {
    result, err := ui.findServiceInCluster(ctx, "my-api")
    assert.NoError(t, err)
    assert.True(t, result.Found)
}
```

The tests use mock Kubernetes clients and cover both successful service discovery scenarios and error cases like non-existent services and invalid port numbers.
