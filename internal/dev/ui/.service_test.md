This file contains unit tests for the UI service component, validating service instantiation, intercept UI retrieval, and intercept setup configuration structures.

## Key Components

- **TestNewService**: Tests the creation of a new UI service instance with proper initialization of dependencies
- **TestService_GetInterceptUI**: Verifies the intercept UI getter method returns the correct instance
- **TestInterceptSetup_Structure**: Validates the InterceptSetup struct properties and embedded ServicePort configuration

## Usage Example

```go
// Run specific test
go test -run TestNewService

// Run all tests in the package
go test ./...

// Example of what the tests are validating:
client := devMocks.NewMockKubernetesClient()
service := NewService(client, client)

// Verify service is properly initialized
assert.NotNil(t, service.interceptUI)
assert.Equal(t, client, service.interceptUI.kubernetesClient)

// Test intercept setup structure
setup := &InterceptSetup{
    ServiceName:    "test-service",
    Namespace:      "default", 
    LocalPort:      8080,
    KubernetesPort: &intercept.ServicePort{
        Name:     "http",
        Port:     8080,
        Protocol: "TCP",
    },
}
```

The tests use mock Kubernetes clients and testutil helpers to ensure proper service initialization and configuration validation in isolated test environments.
