This file contains unit tests for the prerequisite checker functionality, validating tool detection, installation checking, and instruction generation.

## Key Components

- **TestNewPrerequisiteChecker**: Tests the constructor creates a checker with 3 expected tools (Telepresence, jq, Skaffold)
- **TestPrerequisiteChecker_CheckAll**: Validates the `CheckAll()` method returns consistent boolean and slice results
- **TestPrerequisiteChecker_GetInstallInstructions**: Tests instruction generation with various scenarios including case insensitivity and unknown tools
- **TestCheckPrerequisites**: Placeholder test (skipped) for user interaction functionality
- **TestRequirement_Structure**: Validates that all requirement objects have proper structure and non-nil function fields

## Usage Example

```go
func TestMyPrerequisiteFeature(t *testing.T) {
    checker := NewPrerequisiteChecker()
    
    // Test basic functionality
    assert.NotNil(t, checker)
    assert.Len(t, checker.requirements, 3)
    
    // Test instruction generation
    instructions := checker.GetInstallInstructions([]string{"jq"})
    assert.Len(t, instructions, 1)
    assert.NotEmpty(t, instructions[0])
    
    // Test requirement structure
    for _, req := range checker.requirements {
        assert.NotEmpty(t, req.Name)
        assert.NotPanics(t, func() { req.IsInstalled() })
    }
}
```

The tests use table-driven testing patterns and focus on structural validation rather than actual system state to ensure reliable CI/CD execution.
