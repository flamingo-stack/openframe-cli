A configuration file that defines command-line flags for the development commands, providing structured flag management for intercept and scaffold operations.

## Key Components

**InterceptFlags struct**
- Manages flags for the intercept command including port forwarding, namespace targeting, volume mounting, and traffic filtering options
- Supports both global traffic interception and header-based selective interception

**ScaffoldFlags struct**  
- Contains configuration flags for the scaffold command covering Docker image deployment, port forwarding, directory synchronization, and Kubernetes resource mounting
- Includes cluster management options like bootstrap skipping and custom Helm values

**AddGlobalFlags function**
- Adds common persistent flags (verbose, silent, dry-run) to commands for consistent CLI behavior across all operations

## Usage Example

```go
// Using InterceptFlags in a command
var interceptFlags InterceptFlags

interceptCmd := &cobra.Command{
    Use: "intercept",
    RunE: func(cmd *cobra.Command, args []string) error {
        // Access configured flags
        if interceptFlags.Global {
            fmt.Printf("Intercepting all traffic on port %d\n", interceptFlags.Port)
        }
        return nil
    },
}

// Using ScaffoldFlags for deployment
var scaffoldFlags ScaffoldFlags

scaffoldCmd := &cobra.Command{
    Use: "scaffold",
    RunE: func(cmd *cobra.Command, args []string) error {
        // Deploy with specified configuration
        return deployService(scaffoldFlags.Image, scaffoldFlags.Port, scaffoldFlags.Namespace)
    },
}

// Add global flags to root command
rootCmd := &cobra.Command{Use: "dev"}
AddGlobalFlags(rootCmd)
```
