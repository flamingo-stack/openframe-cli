Configuration flags and command structures for the Kubernetes development tool's intercept and scaffold operations.

## Key Components

- **`InterceptFlags`** - Configuration for traffic interception commands including port forwarding, namespace targeting, volume mounting, and header-based routing
- **`ScaffoldFlags`** - Configuration for service scaffolding commands including Docker images, directory synchronization, and cluster management
- **`AddGlobalFlags()`** - Function to add common flags (verbose, silent, dry-run) to cobra commands

## Usage Example

```go
// Setting up intercept flags
interceptFlags := &InterceptFlags{
    Port:      8080,
    Namespace: "development",
    Global:    false,
    Header:    []string{"x-user-id: test-user"},
    Replace:   true,
}

// Setting up scaffold flags  
scaffoldFlags := &ScaffoldFlags{
    Image:       "nginx:latest",
    Port:        80,
    Namespace:   "default",
    SyncLocal:   "./src",
    SyncRemote:  "/app",
    ClusterName: "minikube",
}

// Adding global flags to a command
var rootCmd = &cobra.Command{Use: "dev"}
AddGlobalFlags(rootCmd)
```

The `InterceptFlags` struct supports sophisticated traffic routing scenarios where developers can intercept specific traffic based on headers or globally intercept all traffic. The `ScaffoldFlags` struct provides comprehensive options for deploying and synchronizing local development environments with Kubernetes clusters, including volume mounts for ConfigMaps and Secrets.
