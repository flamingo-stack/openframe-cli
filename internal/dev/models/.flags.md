Defines command-line flag structures and utilities for the dev tool's intercept and scaffold commands.

## Key Components

**InterceptFlags**
- `Port`: Local port for traffic forwarding
- `Namespace`: Kubernetes namespace target
- `Mount`: Remote volume mount path
- `EnvFile`: Environment variables file path
- `Global`: Intercept all traffic flag
- `Header`: Header-based traffic filtering
- `Replace`: Replace existing intercept flag
- `RemotePortName`: Remote port identifier

**ScaffoldFlags**
- `Image`: Docker image specification
- `Port`: Service port configuration
- `Namespace`: Deployment namespace
- `SyncLocal`/`SyncRemote`: Directory synchronization paths
- `ConfigMap`/`Secret`: Kubernetes resource mounting
- `ClusterName`: Target cluster identifier
- `SkipBootstrap`: Bootstrap skip flag
- `HelmValuesFile`: Custom Helm values path

**AddGlobalFlags()**
- Adds common flags (verbose, silent, dry-run) to cobra commands

## Usage Example

```go
// Using InterceptFlags
interceptFlags := &InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Global:    true,
}

// Using ScaffoldFlags
scaffoldFlags := &ScaffoldFlags{
    Image:     "nginx:latest",
    Port:      80,
    Namespace: "production",
}

// Adding global flags to a command
cmd := &cobra.Command{Use: "dev"}
AddGlobalFlags(cmd)
```
