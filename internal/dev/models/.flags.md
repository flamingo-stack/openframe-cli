Defines flag structures and global flag configuration for the CLI commands in a Kubernetes development tool.

## Key Components

- **`InterceptFlags`** - Configuration structure for traffic interception with fields for port forwarding, namespace targeting, volume mounting, environment variables, and header-based routing
- **`ScaffoldFlags`** - Configuration structure for service scaffolding with Docker image settings, port configuration, file synchronization, and cluster management options  
- **`AddGlobalFlags()`** - Function that adds common persistent flags (verbose, silent, dry-run) to cobra commands

## Usage Example

```go
// Create intercept flags for a command
interceptFlags := &InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Header:    []string{"x-dev-user=alice"},
    Global:    false,
}

// Create scaffold flags for deployment
scaffoldFlags := &ScaffoldFlags{
    Image:       "nginx:latest",
    Port:        80,
    Namespace:   "development",
    SyncLocal:   "./src",
    SyncRemote:  "/app/src",
    ClusterName: "dev-cluster",
}

// Add global flags to a cobra command
var rootCmd = &cobra.Command{Use: "dev"}
AddGlobalFlags(rootCmd)
```

These flag structures provide a centralized way to manage command-line arguments for Kubernetes development workflows, supporting both traffic interception for debugging and service scaffolding for rapid deployment.
