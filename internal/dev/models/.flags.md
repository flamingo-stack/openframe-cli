Defines flag structures and utilities for CLI commands in a Kubernetes development tool, providing configuration options for traffic interception and service scaffolding.

## Key Components

**InterceptFlags**: Configuration for intercepting and forwarding Kubernetes service traffic locally
- Port forwarding, namespace selection, volume mounting
- Traffic filtering by headers or global interception
- Environment file loading and intercept replacement options

**ScaffoldFlags**: Configuration for scaffolding new Kubernetes services
- Docker image specification, port forwarding, namespace deployment
- Local/remote directory synchronization
- ConfigMap/Secret mounting and cluster bootstrapping options

**AddGlobalFlags()**: Utility function that adds common flags (verbose, silent, dry-run) to Cobra commands

## Usage Example

```go
// Using InterceptFlags in a command
interceptFlags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Header:    []string{"x-dev-user=john"},
    Global:    false,
}

// Using ScaffoldFlags in a command  
scaffoldFlags := &models.ScaffoldFlags{
    Image:     "nginx:latest",
    Port:      80,
    Namespace: "development",
    SyncLocal: "./app",
}

// Adding global flags to a command
var rootCmd = &cobra.Command{Use: "dev"}
models.AddGlobalFlags(rootCmd)
```
