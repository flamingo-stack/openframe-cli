Test file that validates JSON unmarshaling functionality for Kubernetes service objects parsed from kubectl command output. It ensures the serviceJSON and serviceListJSON types can properly decode real kubectl JSON responses.

## Key Components

- **TestServiceJSON_UnmarshalJSON**: Tests individual service JSON unmarshaling with various port configurations (single/multiple ports, named/unnamed ports, string/numeric target ports)
- **TestServiceListJSON_UnmarshalJSON**: Tests service list JSON unmarshaling including multiple services, empty lists, and error handling
- **TestServiceJSON_RealWorldExamples**: Tests realistic service configurations like web applications, databases, and headless services

## Usage Example

```go
// Run all tests
go test -v

// Run specific test case
go test -run TestServiceJSON_UnmarshalJSON

// Test with real kubectl output
jsonData := `{"metadata":{"name":"my-service"},"spec":{"type":"ClusterIP","ports":[{"port":80}]}}`
var service serviceJSON
err := json.Unmarshal([]byte(jsonData), &service)
if err != nil {
    t.Errorf("Failed to unmarshal: %v", err)
}
assert.Equal(t, "my-service", service.Metadata.Name)
```

The tests cover edge cases like numeric vs string target ports, empty port names, malformed JSON, and various service types (ClusterIP, NodePort, LoadBalancer) to ensure robust kubectl output parsing.
