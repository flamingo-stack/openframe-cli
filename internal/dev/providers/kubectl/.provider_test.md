Test suite for kubectl provider functionality, providing comprehensive coverage for Kubernetes context management and connectivity operations.

## Key Components

- **TestNewProvider**: Tests provider initialization with mock executor
- **TestProvider_GetCurrentContext**: Tests retrieval of active kubectl context with various scenarios (success, whitespace handling, errors)
- **TestProvider_CheckConnection**: Tests cluster connectivity verification using `kubectl cluster-info`
- **TestProvider_SetContext**: Tests switching between kubectl contexts with validation

## Usage Example

```go
func TestKubectlProvider(t *testing.T) {
    testutil.InitializeTestMode()
    mockExecutor := testutil.NewTestMockExecutor()
    provider := NewProvider(mockExecutor, false)

    // Test context retrieval
    mockExecutor.SetResponse("kubectl config current-context", &executor.CommandResult{
        ExitCode: 0,
        Stdout:   "k3d-openframe-dev\n",
    })
    
    context, err := provider.GetCurrentContext(context.Background())
    assert.NoError(t, err)
    assert.Equal(t, "k3d-openframe-dev", context)

    // Test connection check
    mockExecutor.SetResponse("kubectl cluster-info", &executor.CommandResult{
        ExitCode: 0,
        Stdout:   "Kubernetes control plane is running",
    })
    
    err = provider.CheckConnection(context.Background())
    assert.NoError(t, err)
}
```

The tests use table-driven test patterns with mock executors to simulate various kubectl command responses and failure scenarios, ensuring robust error handling and command validation.
