This file defines JSON data structures used to parse kubectl output for Kubernetes service information.

## Key Components

- **`serviceJSON`** - Main structure representing a single Kubernetes service with metadata (name, namespace) and specification details (type, ports configuration)
- **`serviceListJSON`** - Container structure for handling lists of services returned by kubectl commands

## Usage Example

```go
package main

import (
    "encoding/json"
    "fmt"
)

func parseServiceOutput(jsonData []byte) (*serviceListJSON, error) {
    var serviceList serviceListJSON
    err := json.Unmarshal(jsonData, &serviceList)
    if err != nil {
        return nil, err
    }
    
    // Access service information
    for _, service := range serviceList.Items {
        fmt.Printf("Service: %s in namespace %s\n", 
            service.Metadata.Name, 
            service.Metadata.Namespace)
        fmt.Printf("Type: %s\n", service.Spec.Type)
        
        for _, port := range service.Spec.Ports {
            fmt.Printf("Port: %d -> %v (%s)\n", 
                port.Port, 
                port.TargetPort, 
                port.Protocol)
        }
    }
    
    return &serviceList, nil
}
```

These structures enable the kubectl package to parse JSON output from commands like `kubectl get services -o json` and extract service configuration details programmatically.
