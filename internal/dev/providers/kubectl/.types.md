This file defines JSON structures for parsing kubectl output when working with Kubernetes services, enabling Go programs to unmarshal service data from kubectl command responses.

## Key Components

- **`serviceJSON`** - Represents a single Kubernetes service with metadata (name, namespace) and specification (type, ports)
- **`serviceListJSON`** - Container for multiple services, matching kubectl's list output format
- **Port structure** - Nested struct capturing service port configurations including name, port numbers, target ports, and protocols

## Usage Example

```go
import (
    "encoding/json"
    "os/exec"
)

// Parse a single service
func getService(name, namespace string) (*serviceJSON, error) {
    cmd := exec.Command("kubectl", "get", "svc", name, "-n", namespace, "-o", "json")
    output, err := cmd.Output()
    if err != nil {
        return nil, err
    }
    
    var service serviceJSON
    err = json.Unmarshal(output, &service)
    return &service, err
}

// Parse multiple services
func listServices(namespace string) ([]serviceJSON, error) {
    cmd := exec.Command("kubectl", "get", "svc", "-n", namespace, "-o", "json")
    output, err := cmd.Output()
    if err != nil {
        return nil, err
    }
    
    var serviceList serviceListJSON
    err = json.Unmarshal(output, &serviceList)
    return serviceList.Items, err
}
```

The structures handle kubectl's JSON output format, with `TargetPort` as `interface{}` to accommodate both string and numeric port references.
