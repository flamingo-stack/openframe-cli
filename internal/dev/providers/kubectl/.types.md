This file defines JSON structures for unmarshaling kubectl service output, enabling programmatic parsing of Kubernetes service data.

## Key Components

- **`serviceJSON`** - Represents a single Kubernetes service with metadata (name, namespace) and spec (type, ports configuration)
- **`serviceListJSON`** - Container for multiple services, matching the structure returned by `kubectl get services -o json`

## Usage Example

```go
import (
    "encoding/json"
    "fmt"
)

// Parse a single service
var service serviceJSON
err := json.Unmarshal(serviceData, &service)
if err != nil {
    return err
}

fmt.Printf("Service: %s in namespace %s\n", 
    service.Metadata.Name, 
    service.Metadata.Namespace)

// Parse multiple services from kubectl output
var serviceList serviceListJSON
err = json.Unmarshal(servicesListData, &serviceList)
if err != nil {
    return err
}

for _, svc := range serviceList.Items {
    fmt.Printf("Found service: %s with %d ports\n", 
        svc.Metadata.Name, 
        len(svc.Spec.Ports))
}
```

These structs are typically used with `kubectl get service -o json` or `kubectl get services -o json` command output to extract service information programmatically.
