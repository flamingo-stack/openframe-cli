Test file for the kubectl provider's service-related functionality. This file contains comprehensive unit tests for service discovery, retrieval, and validation operations in Kubernetes clusters.

## Key Components

- **TestProvider_GetServices**: Tests listing all services in a namespace with various scenarios including multiple services, empty namespaces, and command failures
- **TestProvider_GetService**: Tests retrieving a specific service by name, covering single/multi-port services and error cases
- **TestProvider_ValidateService**: Tests service existence validation with success and failure scenarios

## Usage Example

```go
// Run all service tests
func TestServiceOperations(t *testing.T) {
    testutil.InitializeTestMode()
    mockExecutor := testutil.NewTestMockExecutor()
    provider := NewProvider(mockExecutor, false)

    // Setup mock response for service listing
    mockExecutor.SetResponse("kubectl get services", &executor.CommandResult{
        ExitCode: 0,
        Stdout:   `{"items": [{"metadata": {"name": "my-service"}}]}`,
    })

    // Test service discovery
    services, err := provider.GetServices(context.Background(), "default")
    assert.NoError(t, err)
    assert.Len(t, services, 1)
}
```

The tests use a mock executor to simulate kubectl command responses and verify proper parsing of Kubernetes service JSON data into `ServiceInfo` structs with ports, types, and metadata.
