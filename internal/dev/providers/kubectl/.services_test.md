Comprehensive test suite for the kubectl service provider functionality. Tests service retrieval, validation, and error handling scenarios using mock executors.

## Key Components

- **TestProvider_GetServices** - Tests retrieving all services in a namespace with various scenarios including multiple services, empty namespaces, and command failures
- **TestProvider_GetService** - Tests fetching a specific service by name, handling single/multiple ports and non-existent services  
- **TestProvider_ValidateService** - Tests service existence validation with proper error handling for missing services and connection issues

## Usage Example

```go
func TestServiceRetrieval(t *testing.T) {
    testutil.InitializeTestMode()
    mockExecutor := testutil.NewTestMockExecutor()
    provider := NewProvider(mockExecutor, false)
    
    // Mock successful kubectl response
    mockExecutor.SetResponse("kubectl get services", &executor.CommandResult{
        ExitCode: 0,
        Stdout:   `{"items": [{"metadata": {"name": "my-service"}}]}`,
    })
    
    services, err := provider.GetServices(context.Background(), "default")
    assert.NoError(t, err)
    assert.Len(t, services, 1)
}
```

Each test uses table-driven testing patterns with mock command executors to simulate kubectl responses, covering success cases, error conditions, and edge cases like empty results and malformed JSON responses.
