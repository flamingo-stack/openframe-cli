This file implements a Telepresence provider for managing Kubernetes service intercepts, enabling developers to redirect cluster traffic to their local development environment.

## Key Components

- **Provider**: Main struct containing command executor and verbose flag
- **NewProvider**: Factory function creating a new Telepresence provider instance
- **SetupIntercept**: Complete workflow for establishing service intercepts with cluster connectivity checks
- **TeardownIntercept**: Removes specific service intercepts from the cluster
- **Disconnect**: Cleanly disconnects from Telepresence daemon
- **checkTelepresenceInstallation**: Validates Telepresence CLI availability and version
- **connectToCluster**: Establishes connection to Kubernetes cluster via Telepresence
- **createIntercept**: Configures intercept rules with port forwarding, headers, and environment settings
- **showInterceptStatus**: Displays active intercept information and usage instructions

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/dev/models"
    "github.com/flamingo-stack/openframe-cli/internal/shared/executor"
)

// Create provider instance
exec := executor.NewRealExecutor()
provider := NewProvider(exec, true) // verbose mode enabled

// Setup intercept with configuration
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Mount:     "/tmp/mounts",
    EnvFile:   ".env.local",
    Global:    false,
    Header:    []string{"x-dev-user=alice"},
    Replace:   false,
}

// Establish intercept
err := provider.SetupIntercept("my-service", flags)
if err != nil {
    log.Fatal(err)
}

// Later, cleanup intercept
provider.TeardownIntercept("my-service", "default")
provider.Disconnect()
```
