A Telepresence integration provider that manages Kubernetes service intercepts for local development, enabling traffic redirection from cluster services to local applications.

## Key Components

- **Provider**: Core struct that wraps Telepresence CLI operations with command execution and verbose logging
- **NewProvider()**: Factory function for creating provider instances with executor and verbosity settings
- **SetupIntercept()**: Complete intercept workflow including installation check, cluster connection, intercept creation, and status display
- **TeardownIntercept()**: Removes specific service intercepts from the cluster
- **Disconnect()**: Terminates all Telepresence connections and cleans up resources

## Usage Example

```go
// Create a new telepresence provider
exec := executor.NewRealExecutor()
provider := telepresence.NewProvider(exec, true)

// Setup an intercept with configuration
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "production",
    Header:    []string{"x-user=dev"},
    EnvFile:   ".env.local",
    Mount:     "/app/data",
    Replace:   false,
    Global:    false,
}

// Start intercepting traffic
err := provider.SetupIntercept("user-service", flags)
if err != nil {
    log.Fatal("Failed to setup intercept:", err)
}

// Later, cleanup the intercept
err = provider.TeardownIntercept("user-service", "production")
if err != nil {
    log.Warn("Failed to teardown intercept:", err)
}

// Disconnect completely
provider.Disconnect()
```

The provider automatically handles Telepresence installation verification, cluster connectivity, and provides detailed status feedback with spinner animations and colored terminal output.
