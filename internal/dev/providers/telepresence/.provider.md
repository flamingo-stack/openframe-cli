Provides Telepresence integration for intercepting Kubernetes service traffic and routing it to local development environments.

## Key Components

- **Provider**: Main struct that handles Telepresence intercept operations with command execution and verbose logging
- **SetupIntercept**: Creates and configures a Telepresence intercept for a specified service with customizable flags
- **TeardownIntercept**: Removes an active intercept and cleans up connections
- **Disconnect**: Cleanly disconnects from the Telepresence session

## Usage Example

```go
// Create a new Telepresence provider
executor := &executor.RealCommandExecutor{}
provider := NewProvider(executor, true)

// Set up intercept configuration
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "production",
    Mount:     "/tmp/mount",
    EnvFile:   ".env.prod",
    Global:    false,
    Header:    []string{"x-user: developer"},
    Replace:   false,
}

// Create intercept for service
err := provider.SetupIntercept("my-service", flags)
if err != nil {
    log.Fatal(err)
}

// Later, tear down the intercept
err = provider.TeardownIntercept("my-service", "production")
if err != nil {
    log.Warn(err)
}

// Disconnect from Telepresence
provider.Disconnect()
```

The provider handles the complete lifecycle of Telepresence intercepts, including installation verification, cluster connection, intercept creation with flexible routing rules, and cleanup operations.
