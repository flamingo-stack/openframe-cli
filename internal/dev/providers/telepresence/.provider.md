This file implements a Telepresence provider for managing Kubernetes service intercepts, enabling local development by routing cluster traffic to local services.

## Key Components

- **Provider struct**: Main service wrapper containing command executor and verbose flag
- **NewProvider()**: Constructor function for creating new provider instances
- **SetupIntercept()**: Orchestrates the complete intercept setup process (installation check, cluster connection, intercept creation, status display)
- **TeardownIntercept()**: Removes specific service intercepts
- **Disconnect()**: Terminates Telepresence connection entirely
- **Private helper methods**: Handle individual steps like installation verification, cluster connection, and status reporting

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/dev/models"
    "github.com/flamingo-stack/openframe-cli/internal/shared/executor"
)

// Create provider
exec := executor.NewCommandExecutor()
provider := NewProvider(exec, true)

// Setup intercept with configuration
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Global:    false,
    Header:    []string{"x-user: developer"},
}

// Start intercepting traffic
err := provider.SetupIntercept("my-service", flags)

// Later, remove the intercept
err = provider.TeardownIntercept("my-service", "default")

// Disconnect completely
err = provider.Disconnect()
```
