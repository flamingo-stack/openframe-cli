Implements Telepresence integration for intercepting Kubernetes service traffic and routing it to local development instances. Provides a complete workflow for setting up, managing, and tearing down service intercepts.

## Key Components

- **Provider**: Main struct that manages Telepresence operations with command execution and verbose logging
- **NewProvider**: Factory function to create a new Provider instance
- **SetupIntercept**: Establishes a complete intercept workflow including installation check, cluster connection, and intercept creation
- **TeardownIntercept**: Removes existing intercepts for specified services
- **Disconnect**: Cleanly disconnects from Telepresence session

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/dev/telepresence"
    "github.com/flamingo-stack/openframe-cli/internal/dev/models"
)

// Create provider
provider := telepresence.NewProvider(executor, true)

// Setup intercept with custom flags
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Mount:     "/tmp/volume",
    EnvFile:   ".env.local",
    Header:    []string{"x-dev-user: alice"},
    Global:    false,
    Replace:   true,
}

// Create intercept
err := provider.SetupIntercept("my-service", flags)

// Teardown when done
err = provider.TeardownIntercept("my-service", "default")
err = provider.Disconnect()
```
