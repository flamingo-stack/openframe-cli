This file implements a Telepresence provider for managing Kubernetes service intercepts in a development workflow.

## Key Components

- **Provider struct**: Main handler with command executor and verbose logging support
- **NewProvider()**: Constructor function for creating provider instances
- **SetupIntercept()**: Orchestrates the complete intercept setup process (installation check, cluster connection, intercept creation, status display)
- **TeardownIntercept()**: Removes active intercepts for specified services
- **Disconnect()**: Terminates Telepresence connection and cleanup
- **Private methods**: Handle individual steps like installation verification, cluster connection, and intercept configuration

## Usage Example

```go
// Create a new provider with command executor
provider := NewProvider(executor, true) // verbose=true

// Setup intercept with configuration
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "default",
    Mount:     "/app",
    EnvFile:   ".env",
    Header:    []string{"x-user=dev"},
    Global:    false,
    Replace:   true,
}

// Start intercepting traffic
err := provider.SetupIntercept("my-service", flags)
if err != nil {
    log.Fatal(err)
}

// Later, teardown the intercept
err = provider.TeardownIntercept("my-service", "default")

// Disconnect when done
provider.Disconnect()
```

The provider supports various intercept modes including header-based routing, global intercepts, volume mounting, and environment file injection for comprehensive local development scenarios.
