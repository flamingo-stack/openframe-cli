This package provides Telepresence integration for intercepting Kubernetes service traffic during local development. It manages the complete lifecycle of service intercepts including setup, configuration, and teardown operations.

## Key Components

- **Provider**: Main struct that handles all Telepresence operations using an injected command executor
- **SetupIntercept()**: Creates a new intercept with configurable flags (port, namespace, headers, etc.)
- **TeardownIntercept()**: Removes an existing intercept for a specified service
- **Disconnect()**: Cleanly disconnects from Telepresence cluster connection

## Usage Example

```go
// Initialize provider with executor and verbose logging
provider := NewProvider(executor, true)

// Setup intercept with configuration
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "production", 
    EnvFile:   ".env.local",
    Header:    []string{"user=developer"},
    Global:    false,
}

err := provider.SetupIntercept("user-service", flags)
if err != nil {
    log.Fatal(err)
}

// Later teardown the intercept
err = provider.TeardownIntercept("user-service", "production")

// Disconnect from cluster
err = provider.Disconnect()
```

The provider automatically handles installation checks, cluster connection, intercept creation with proper argument building, and provides detailed status feedback through pterm styling.
