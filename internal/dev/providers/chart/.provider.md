This file implements a chart provider for managing Helm chart installations in OpenFrame development environments with specialized handling for Skaffold workflows.

## Key Components

**Provider struct**: Main service that orchestrates Helm chart installations with development-specific configurations, featuring context cancellation support and verbose logging capabilities.

**Core Methods**:
- `NewProvider()` - Factory function for creating chart provider instances
- `InstallCharts()` / `InstallChartsWithContext()` - Installs charts with autoSync disabled for development
- `PrepareDevHelmValues()` - Prepares development-specific Helm values files
- `GetDefaultDevValues()` - Returns appropriate development values file path

**Helper Methods**:
- `validateHelmValuesFile()` - Validates Helm values file existence and accessibility
- `createDevHelmValuesFile()` - Creates/merges Helm values with development settings (autoSync disabled)

## Usage Example

```go
// Create a chart provider for development
exec := executor.NewCommandExecutor()
provider := chart.NewProvider(exec, true) // verbose mode

// Install charts with custom values for development
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Minute)
defer cancel()

err := provider.InstallChartsWithContext(ctx, "my-cluster", "custom-values.yaml")
if err != nil {
    log.Fatalf("Chart installation failed: %v", err)
}

// Prepare development Helm values
devValuesPath, err := provider.PrepareDevHelmValues("base-values.yaml")
if err != nil {
    log.Fatalf("Failed to prepare dev values: %v", err)
}
```

The provider automatically disables ArgoCD autoSync in development mode to prevent conflicts with Skaffold's live reload functionality, and includes aggressive cancellation handling to stop background chart installation processes when needed.
