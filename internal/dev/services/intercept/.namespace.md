This file provides namespace management functionality for Telepresence, allowing services to detect the current namespace and switch between Kubernetes namespaces.

## Key Components

- **`getCurrentNamespace(ctx)`** - Retrieves the current Telepresence namespace from status output, defaulting to "default" if none is found
- **`switchNamespace(ctx, current, target)`** - Switches Telepresence connection from current to target namespace by disconnecting and reconnecting

## Usage Example

```go
// Get current namespace
currentNS, err := service.getCurrentNamespace(ctx)
if err != nil {
    log.Fatal(err)
}

// Switch to a different namespace
if currentNS != "production" {
    err = service.switchNamespace(ctx, currentNS, "production")
    if err != nil {
        log.Fatalf("Failed to switch namespace: %v", err)
    }
}
```

The namespace detection uses the same command as the original bash implementation (`telepresence status --output json | jq -r '.user_daemon.namespace'`), ensuring compatibility. The switching operation performs a clean disconnect/reconnect cycle to preserve the traffic manager connection while changing namespaces.
