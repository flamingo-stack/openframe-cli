This file provides namespace management functionality for Telepresence connections within the intercept service, handling namespace detection and switching operations.

## Key Components

- **getCurrentNamespace()** - Retrieves the current Telepresence namespace using `telepresence status` with JSON output parsing
- **switchNamespace()** - Switches Telepresence from one namespace to another by disconnecting and reconnecting
- **Service receiver methods** - Both functions are methods on the Service struct with context support

## Usage Example

```go
// Get current namespace (returns "default" if none set or error occurs)
currentNS, err := service.getCurrentNamespace(ctx)
if err != nil {
    log.Printf("Error getting current namespace: %v", err)
}

// Switch to a different namespace
err = service.switchNamespace(ctx, currentNS, "staging")
if err != nil {
    log.Printf("Failed to switch namespace: %v", err)
}
```

The namespace switching strategy preserves traffic manager connections by using disconnect/reconnect rather than quit/connect, which is more efficient for namespace changes. The current namespace detection gracefully falls back to "default" if Telepresence returns null or encounters errors.
