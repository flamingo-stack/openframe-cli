This file provides namespace management functionality for Telepresence connections, enabling seamless switching between Kubernetes namespaces while preserving traffic manager connections.

## Key Components

- **`getCurrentNamespace()`** - Retrieves the current Telepresence namespace using JSON output parsing, defaulting to "default" if none is set
- **`switchNamespace()`** - Performs namespace switching by disconnecting and reconnecting Telepresence to the target namespace

## Usage Example

```go
// Get current namespace
current, err := service.getCurrentNamespace(ctx)
if err != nil {
    return fmt.Errorf("failed to get current namespace: %w", err)
}

// Switch to production namespace
err = service.switchNamespace(ctx, current, "production")
if err != nil {
    return fmt.Errorf("failed to switch namespace: %w", err)
}
```

The implementation uses a graceful disconnect/reconnect strategy rather than quitting the entire Telepresence session, which helps maintain traffic manager connections and reduces connection overhead. The namespace detection relies on `jq` for JSON parsing of Telepresence status output, matching the behavior of existing bash scripts for consistency.
