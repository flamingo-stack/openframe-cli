This file handles Telepresence namespace operations, including detecting the current namespace and switching between namespaces.

## Key Components

- `getCurrentNamespace(ctx Context) (string, error)` - Retrieves the current Telepresence namespace using the status command and jq parsing, defaulting to "default" if none is found
- `switchNamespace(ctx Context, current, target string) error` - Switches Telepresence from the current namespace to a target namespace by disconnecting and reconnecting

## Usage Example

```go
// Get current namespace
service := &Service{executor: executor, verbose: true}
currentNS, err := service.getCurrentNamespace(ctx)
if err != nil {
    log.Fatal(err)
}
fmt.Printf("Current namespace: %s\n", currentNS)

// Switch to a different namespace
err = service.switchNamespace(ctx, currentNS, "production")
if err != nil {
    log.Fatalf("Failed to switch namespace: %v", err)
}
```

The namespace switching uses a disconnect/reconnect approach rather than quitting Telepresence entirely, which helps preserve the traffic manager connection. The current namespace detection mirrors the bash script implementation for consistency.
