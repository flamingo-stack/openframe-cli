Handles graceful cleanup and signal management for the telepresence intercept service. This component ensures proper teardown of telepresence connections when the service is interrupted or terminated.

## Key Components

- **setupCleanupHandler()** - Configures OS signal handling for SIGINT and SIGTERM to trigger cleanup
- **cleanup()** - Performs orderly shutdown of telepresence intercepts and daemon connections

## Usage Example

The cleanup functionality is automatically initialized when creating a new intercept service:

```go
service := &Service{
    signalChannel: make(chan os.Signal, 1),
    verbose: true,
}

// Setup cleanup handler for graceful shutdown
service.setupCleanupHandler("my-service")

// When user presses Ctrl+C or service receives SIGTERM:
// 1. Leaves current telepresence intercept
// 2. Quits telepresence daemon  
// 3. Restores original Kubernetes namespace
// 4. Exits cleanly with success status
```

The cleanup process includes a 30-second timeout context and provides verbose logging when enabled. It handles edge cases like missing current service or namespace restoration failures gracefully with warning messages.
