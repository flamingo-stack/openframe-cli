This file provides core Telepresence intercept functionality for creating service intercepts in Kubernetes clusters with configurable port mappings and routing options.

## Key Components

- **`createIntercept()`** - Main function that builds and executes telepresence intercept commands with proper port mapping and configuration flags
- **`getRemotePortName()`** - Helper function that determines the remote port name, defaulting to the port number if not explicitly specified

## Usage Example

```go
// Create basic intercept for a service
flags := &models.InterceptFlags{
    Port: 8080,
    Global: false,
    Replace: false,
}

err := service.createIntercept(ctx, "my-service", flags)
if err != nil {
    log.Fatalf("Failed to create intercept: %v", err)
}

// Create intercept with custom headers and environment file
flags = &models.InterceptFlags{
    Port: 3000,
    RemotePortName: "http",
    EnvFile: "/path/to/env.txt",
    Header: []string{"x-custom-header: development"},
    Global: true,
    Replace: true,
}

err = service.createIntercept(ctx, "api-service", flags)
```

The function automatically disables volume mounting and formats port mappings in the `local_port:remote_port_name` format expected by Telepresence, making it easy to redirect traffic from Kubernetes services to local development environments.
