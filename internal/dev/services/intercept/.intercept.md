This file provides functionality for creating Telepresence intercepts to redirect Kubernetes service traffic to local development environments.

## Key Components

- **`createIntercept()`**: Main function that builds and executes a Telepresence intercept command with configurable options like port mapping, environment files, and HTTP headers
- **`getRemotePortName()`**: Helper function that determines the remote port name for the intercept, defaulting to the port number if not explicitly specified

## Usage Example

```go
// Create an intercept with basic configuration
flags := &models.InterceptFlags{
    Port: 8080,
    RemotePortName: "http",
}
err := service.createIntercept(ctx, "my-service", flags)

// Create a global intercept with custom headers and env file
flags := &models.InterceptFlags{
    Port: 3000,
    Global: true,
    EnvFile: "/path/to/.env",
    Header: []string{"x-user-id=123"},
    Replace: true,
}
err := service.createIntercept(ctx, "api-service", flags)
```

The service integrates with a command executor to run Telepresence commands, automatically handling port mapping in the `local_port:remote_port_name` format and supporting various intercept modes including global intercepts and header-based routing.
