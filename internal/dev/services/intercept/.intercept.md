This file implements telepresence intercept creation functionality, building and executing telepresence commands to intercept Kubernetes service traffic for local development.

## Key Components

- **`createIntercept`**: Main method that constructs and executes telepresence intercept commands with configurable options like port mapping, environment files, and HTTP headers
- **`getRemotePortName`**: Helper method that determines the remote port name for the intercept, defaulting to the port number if not explicitly specified

## Usage Example

```go
// Create intercept flags configuration
flags := &models.InterceptFlags{
    Port:           8080,
    RemotePortName: "http",
    EnvFile:        ".env.local",
    Global:         false,
    Header:         []string{"x-user-id: 123"},
    Replace:        true,
}

// Create intercept for a service
err := service.createIntercept(ctx, "my-service", flags)
if err != nil {
    log.Fatal("Failed to create intercept:", err)
}
```

The service builds telepresence commands like:
```bash
telepresence intercept my-service --port 8080:http --mount=false --env-file .env.local --http-header "x-user-id: 123" --replace
```

This enables developers to route traffic from a Kubernetes service to their local development environment while maintaining the cluster context.
