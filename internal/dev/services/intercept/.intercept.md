This file implements the core telepresence intercept creation functionality for the OpenFrame CLI development tools.

## Key Components

- **`createIntercept()`** - Main function that builds and executes telepresence intercept commands with proper port mapping and flag configuration
- **`getRemotePortName()`** - Helper function that determines the remote port name, defaulting to the port number if not explicitly specified

## Usage Example

```go
// Create an intercept with basic configuration
flags := &models.InterceptFlags{
    Port: 8080,
    Global: true,
    EnvFile: ".env",
    Header: []string{"x-debug: true"},
    Replace: true,
}

err := service.createIntercept(ctx, "my-service", flags)
if err != nil {
    log.Fatalf("Failed to create intercept: %v", err)
}

// The function will execute:
// telepresence intercept my-service --port 8080:8080 --mount=false --env-file .env --global --http-header "x-debug: true" --replace
```

The implementation follows the original bash script pattern, automatically disabling mounts and building telepresence commands with the appropriate port mapping format (`local_port:remote_port_name`). It supports all standard telepresence intercept options including global intercepts, environment files, HTTP headers, and service replacement.
