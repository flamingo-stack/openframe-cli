This file implements the core logic for creating Telepresence intercepts, transforming configuration flags into executable commands that enable local development against remote Kubernetes services.

## Key Components

- **`createIntercept`** - Main function that builds and executes telepresence intercept commands with proper port mapping, mount disabling, and optional flags
- **`getRemotePortName`** - Helper function that determines the remote port name for intercept configuration, defaulting to the port number if not explicitly specified

## Usage Example

```go
// Create intercept flags
flags := &models.InterceptFlags{
    Port:           8080,
    RemotePortName: "http",
    EnvFile:        ".env.local",
    Global:         false,
    Header:         []string{"x-user-id: 123"},
    Replace:        true,
}

// Create the intercept
err := service.createIntercept(ctx, "my-service", flags)
if err != nil {
    log.Fatalf("Failed to create intercept: %v", err)
}
```

The intercept creation follows Telepresence patterns with port mapping in `local_port:remote_port_name` format, mount disabling by default for performance, and support for environment files, global intercepts, HTTP headers, and service replacement options.
