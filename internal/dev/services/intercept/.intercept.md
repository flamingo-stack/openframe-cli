This file provides functionality for creating Telepresence intercepts with configurable port mappings and request routing options.

## Key Components

- **`createIntercept()`** - Core method that builds and executes telepresence intercept commands with proper argument formatting
- **`getRemotePortName()`** - Helper function that determines the remote port name, defaulting to the port number if not explicitly specified

## Usage Example

```go
ctx := context.Background()
flags := &models.InterceptFlags{
    Port:           8080,
    RemotePortName: "http",
    EnvFile:        "env.txt",
    Global:         true,
    Header:         []string{"x-user-id: 123"},
    Replace:        false,
}

// Create intercept for a service
err := service.createIntercept(ctx, "my-service", flags)
if err != nil {
    log.Fatal("Intercept failed:", err)
}
```

The implementation follows telepresence command patterns, automatically disabling mount operations and supporting various intercept configurations including port mappings, environment files, global intercepts, HTTP headers, and service replacement options.
