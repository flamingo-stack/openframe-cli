A Go service that provides Telepresence intercept functionality for local development, allowing developers to intercept and redirect Kubernetes service traffic to their local machine.

## Key Components

- **Service**: Main struct that manages Telepresence intercepts with state tracking and cleanup handling
- **TelepresenceStatus**: Represents JSON output from telepresence status commands
- **StartIntercept()**: Primary method to start intercepting a Kubernetes service with validation and setup
- **StopIntercept()**: Manually stops an active intercept
- **validateInputs()**: Validates service names, ports, namespaces, and configuration files
- **checkKubernetesContext()**: Verifies kubectl connectivity and active cluster context
- **ensureCorrectNamespace()**: Manages namespace switching for Telepresence connections

## Usage Example

```go
// Create a new intercept service
exec := executor.NewCommandExecutor()
service := intercept.NewService(exec, true) // verbose mode enabled

// Set up intercept flags
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "development",
    EnvFile:   ".env.local",
    Header:    []string{"x-user-id=123", "x-env=dev"},
}

// Start intercepting a service
err := service.StartIntercept("my-api-service", flags)
if err != nil {
    log.Fatalf("Failed to start intercept: %v", err)
}

// Check intercept status
if service.IsIntercepting() {
    fmt.Printf("Currently intercepting: %s\n", service.GetCurrentService())
}

// Stop the intercept
err = service.StopIntercept("my-api-service")
if err != nil {
    log.Fatalf("Failed to stop intercept: %v", err)
}
```
