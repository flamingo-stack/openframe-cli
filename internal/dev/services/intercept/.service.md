This file provides a service layer for managing Telepresence intercepts in Kubernetes environments, offering a Go implementation of functionality originally written in bash scripts.

## Key Components

- **Service**: Main struct that manages intercept operations with methods for starting, stopping, and monitoring intercepts
- **TelepresenceStatus**: JSON structure for parsing Telepresence daemon status
- **NewService()**: Constructor function that initializes a new intercept service
- **StartIntercept()**: Primary method that validates inputs, sets up namespace context, and creates the intercept
- **StopIntercept()**: Manually terminates an active intercept
- **Validation methods**: Input validation for service names, ports, namespaces, and environment files
- **Kubernetes integration**: Context checking and cluster connectivity verification

## Usage Example

```go
// Create a new intercept service
executor := executor.NewRealExecutor()
service := NewService(executor, true) // verbose mode enabled

// Set up intercept flags
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "development",
    EnvFile:   ".env",
    Header:    []string{"x-user-id=123"},
}

// Start intercepting a service
err := service.StartIntercept("my-service", flags)
if err != nil {
    log.Fatal(err)
}

// Check intercept status
if service.IsIntercepting() {
    fmt.Printf("Currently intercepting: %s\n", service.GetCurrentService())
}
```
