This file provides Telepresence intercept functionality for the OpenFrame CLI, implementing bash script capabilities with enhanced Go features for Kubernetes service interception and debugging.

## Key Components

- **Service**: Main struct managing Telepresence intercepts with state tracking and cleanup handling
- **TelepresenceStatus**: JSON structure for parsing Telepresence daemon status
- **NewService()**: Constructor creating a new intercept service instance
- **StartIntercept()**: Initiates service interception with validation, namespace switching, and signal handling
- **StopIntercept()**: Manually terminates active intercepts
- **State getters**: Methods returning current service, namespace, and intercept status
- **Private helpers**: Input validation, Kubernetes context checking, and namespace management

## Usage Example

```go
// Create intercept service
exec := executor.NewCommandExecutor()
service := intercept.NewService(exec, true) // verbose mode

// Configure intercept flags
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "development",
    EnvFile:   ".env",
    Header:    []string{"user-id=123"},
}

// Start intercepting a service
err := service.StartIntercept("my-api-service", flags)
if err != nil {
    log.Fatal(err)
}

// Check intercept status
if service.IsIntercepting() {
    fmt.Printf("Intercepting: %s in %s\n", 
        service.GetCurrentService(),
        service.GetCurrentNamespace())
}

// Stop intercept programmatically
err = service.StopIntercept("my-api-service")
```
