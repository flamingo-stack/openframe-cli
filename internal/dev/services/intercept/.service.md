This Go service provides Telepresence intercept functionality for Kubernetes development workflows, implementing enhanced versions of bash script capabilities with proper error handling and validation.

## Key Components

- **Service**: Main struct that manages Telepresence intercepts with state tracking and cleanup
- **TelepresenceStatus**: Struct for parsing Telepresence JSON status output
- **StartIntercept()**: Initiates a Telepresence intercept with comprehensive validation and setup
- **StopIntercept()**: Manually stops active intercepts with proper cleanup
- **validateInputs()**: Validates service names, ports, headers, and environment files
- **checkKubernetesContext()**: Verifies kubectl availability and cluster connectivity
- **ensureCorrectNamespace()**: Manages namespace switching for Telepresence connections

## Usage Example

```go
// Create intercept service with command executor
executor := executor.NewRealExecutor()
service := NewService(executor, true) // verbose mode enabled

// Configure intercept flags
flags := &models.InterceptFlags{
    Port:      3000,
    Namespace: "development",
    EnvFile:   ".env.local",
    Header:    []string{"x-user=developer"},
}

// Start intercepting a service
err := service.StartIntercept("my-api-service", flags)
if err != nil {
    log.Fatalf("Failed to start intercept: %v", err)
}

// Check intercept status
if service.IsIntercepting() {
    fmt.Printf("Currently intercepting: %s\n", service.GetCurrentService())
}

// Stop intercept manually
err = service.StopIntercept("my-api-service")
```

The service handles signal cleanup, namespace management, and provides comprehensive validation for production-ready Kubernetes development workflows.
