This file implements a Telepresence intercept service that provides advanced Kubernetes traffic interception capabilities with enhanced Go implementation of bash scripting functionality.

## Key Components

**Core Service**
- `Service` - Main service struct managing Telepresence intercepts with state tracking
- `TelepresenceStatus` - JSON model for parsing Telepresence status output
- `NewService()` - Constructor accepting command executor and verbose flag

**Primary Methods**
- `StartIntercept()` - Initiates traffic intercept for specified service with validation and cleanup
- `StopIntercept()` - Manually terminates active intercept with state verification
- `validateInputs()` - Comprehensive validation of service names, ports, and configuration
- `checkKubernetesContext()` - Verifies kubectl availability and cluster connectivity
- `ensureCorrectNamespace()` - Manages namespace switching with automatic reconnection

**State Management**
- `IsIntercepting()` - Returns current intercept status
- `GetCurrentService()` - Returns name of intercepted service
- `GetCurrentNamespace()` - Returns active namespace
- `GetOriginalNamespace()` - Returns pre-intercept namespace for restoration

## Usage Example

```go
// Initialize the intercept service
executor := executor.NewRealCommandExecutor()
service := NewService(executor, true) // verbose mode enabled

// Configure intercept parameters
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "development",
    EnvFile:   ".env.local",
    Header:    []string{"x-user-id=123", "x-tenant=demo"},
}

// Start intercepting traffic for a microservice
err := service.StartIntercept("user-service", flags)
if err != nil {
    log.Fatalf("Intercept failed: %v", err)
}

// Check intercept status
if service.IsIntercepting() {
    fmt.Printf("Intercepting: %s\n", service.GetCurrentService())
}
```
