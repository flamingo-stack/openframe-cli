This service provides Telepresence intercept functionality for Kubernetes services, implementing enhanced Go capabilities from the original bash functions in develop.sh.

## Key Components

- **Service**: Main struct that manages Telepresence intercepts with validation, cleanup handling, and signal management
- **TelepresenceStatus**: Struct for parsing Telepresence JSON status output
- **NewService()**: Factory function for creating new intercept service instances
- **StartIntercept()**: Validates inputs, ensures correct namespace, and creates intercepts based on service name and flags
- **StopIntercept()**: Manually stops active intercepts
- **State methods**: IsIntercepting(), GetCurrentService(), GetCurrentNamespace(), GetOriginalNamespace()

## Usage Example

```go
// Create new intercept service
executor := &executor.RealExecutor{}
service := NewService(executor, true) // verbose mode enabled

// Set up intercept flags
flags := &models.InterceptFlags{
    Port:      8080,
    Namespace: "development",
    EnvFile:   ".env",
    Header:    []string{"x-user=dev"},
}

// Start intercept (blocks until Ctrl+C)
if err := service.StartIntercept("my-service", flags); err != nil {
    log.Fatalf("Intercept failed: %v", err)
}

// Or stop manually
if err := service.StopIntercept("my-service"); err != nil {
    log.Printf("Stop failed: %v", err)
}
```
