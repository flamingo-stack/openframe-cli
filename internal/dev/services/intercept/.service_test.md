Test suite for the intercept service functionality, validating service creation, input validation, state management, and intercept operations.

## Key Components

- `TestNewService` - Tests service initialization with executor, verbose flag, and internal state
- `TestService_ValidateInputs` - Comprehensive validation testing for service names, ports, headers, and namespaces
- `TestService_ShowInterceptInstructions` - Verifies instruction display functionality
- `TestService_GettersAndSetters` - Tests state accessor methods for intercept status and service details
- `TestService_StopIntercept` - Validates intercept termination logic and error conditions

## Usage Example

```go
// Run all intercept service tests
go test ./internal/intercept -v

// Run specific validation tests
go test ./internal/intercept -run TestService_ValidateInputs

// Test service creation and state management
func TestCustomServiceBehavior(t *testing.T) {
    testutil.InitializeTestMode()
    mockExecutor := testutil.NewTestMockExecutor()
    service := NewService(mockExecutor, true)
    
    assert.False(t, service.IsIntercepting())
    assert.Equal(t, "", service.GetCurrentService())
}
```

The test suite covers edge cases like invalid ports (0, >65535), malformed headers, empty service names, and validates that default namespace assignment works correctly when no namespace is specified.
