Test file that validates the functionality of the intercept service, covering service initialization, input validation, and state management.

## Key Components

- **TestNewService**: Validates proper service initialization with mock executor and configuration
- **TestService_ValidateInputs**: Comprehensive table-driven tests for input validation including service names, ports, namespaces, and headers
- **TestService_ShowInterceptInstructions**: Tests the display of intercept setup instructions
- **TestService_GettersAndSetters**: Validates service state getters and setters for intercept tracking
- **TestService_StopIntercept**: Tests intercept termination logic and error handling

## Usage Example

```go
// Run specific test
go test -run TestNewService

// Run all validation tests
go test -run TestService_ValidateInputs

// Run tests with verbose output
go test -v ./...

// Example test case structure from the file
func TestYourFunction(t *testing.T) {
    testutil.InitializeTestMode()
    mockExecutor := testutil.NewTestMockExecutor()
    service := NewService(mockExecutor, false)

}
```

The tests use table-driven patterns for comprehensive validation coverage and rely on the testutil package for mock setup and test environment initialization.
