Manages Skaffold development workflows for OpenFrame projects, providing automated cluster setup, chart installation, and development session management.

## Key Components

**Service**: Main service struct that orchestrates the complete Skaffold development workflow
- `RunScaffoldWorkflow()`: Executes the full scaffold workflow with cluster setup and chart installation
- `runSkaffoldDev()`: Runs Skaffold dev command with retry logic and signal handling
- `checkPrerequisites()`: Validates and optionally installs Skaffold binary
- `ShowSkaffoldConfigInfoAndSelectService()`: Discovers and allows selection of Skaffold configurations

**Configuration Management**:
- `getClusterName()`: Determines target cluster from arguments or interactive selection
- `determineNamespace()`: Resolves appropriate Kubernetes namespace for deployment
- `buildSkaffoldArgs()`: Constructs command-line arguments for Skaffold execution

**Process Control**:
- `setupSignalHandler()`: Configures graceful shutdown on SIGINT/SIGTERM
- `IsRunning()`: Returns current execution state
- `Stop()`: Manually terminates active development sessions

## Usage Example

```go
// Create scaffold service with command executor
executor := executor.NewCommandExecutor()
service := scaffold.NewService(executor, true)

// Configure scaffold flags
flags := &models.ScaffoldFlags{
    SkipBootstrap:  false,
    HelmValuesFile: "values.yaml",
    Namespace:      "development",
}

// Run complete workflow
ctx := context.Background()
args := []string{"my-cluster"}
err := service.RunScaffoldWorkflow(ctx, args, flags)
if err != nil {
    log.Fatalf("Scaffold workflow failed: %v", err)
}

// Check if still running
if service.IsRunning() {
    fmt.Println("Skaffold session active")
}
```

The service automatically handles prerequisite installation, cluster selection, chart deployment, and manages the Skaffold development session with retry logic and graceful shutdown capabilities.
