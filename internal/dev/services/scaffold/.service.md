This file implements a comprehensive Skaffold development workflow service that orchestrates container development with Kubernetes clusters, including prerequisite checks, cluster management, and automated bootstrapping.

## Key Components

**Core Service**
- `Service` - Main service struct managing Skaffold workflows
- `NewService()` - Constructor with command executor and verbose mode support

**Primary Workflow**
- `RunScaffoldWorkflow()` - Complete workflow including config selection, cluster setup, chart installation, and Skaffold execution
- `checkPrerequisites()` - Validates and optionally installs Skaffold
- `runSkaffoldDev()` - Executes Skaffold with retry logic and signal handling

**Configuration & Selection**
- `ShowSkaffoldConfigInfoAndSelectService()` - Interactive service selection from available Skaffold configs
- `getClusterName()` - Determines target cluster from args or interactive selection
- `determineNamespace()` - Resolves Kubernetes namespace for deployment

**Utility Methods**
- `setupSignalHandler()` - Graceful shutdown handling
- `IsRunning()` / `Stop()` - Service state management

## Usage Example

```go
// Create service with command executor
executor := executor.NewCommandExecutor()
service := scaffold.NewService(executor, true) // verbose mode

// Define flags for workflow
flags := &models.ScaffoldFlags{
    SkipBootstrap:   false,
    HelmValuesFile: "values.yaml",
    Namespace:      "development",
}

// Run complete development workflow
ctx := context.Background()
args := []string{"my-cluster"} // optional cluster name
err := service.RunScaffoldWorkflow(ctx, args, flags)
if err != nil {
    log.Fatal(err)
}
```
