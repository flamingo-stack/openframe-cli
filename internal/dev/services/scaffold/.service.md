This file provides the core service for managing Skaffold development workflows within the OpenFrame CLI. It orchestrates the complete development cycle including cluster selection, chart installation, and Skaffold execution with automatic retry logic.

## Key Components

- **Service**: Main service struct that manages Skaffold workflow execution
- **NewService()**: Creates a new scaffold service instance with command executor and verbose logging
- **RunScaffoldWorkflow()**: Executes the complete development workflow including service selection, cluster setup, and Skaffold execution
- **checkPrerequisites()**: Validates Skaffold installation and offers automatic installation
- **runSkaffoldDev()**: Runs Skaffold dev command with retry logic and signal handling
- **ShowSkaffoldConfigInfoAndSelectService()**: Interactive service selection from available Skaffold configurations

## Usage Example

```go
// Create a new scaffold service
executor := executor.NewCommandExecutor()
scaffoldService := scaffold.NewService(executor, true) // verbose mode

// Define scaffold flags
flags := &models.ScaffoldFlags{
    SkipBootstrap:   false,
    HelmValuesFile: "values.yaml",
    Namespace:      "development",
}

// Run the complete workflow
ctx := context.Background()
args := []string{"my-cluster"} // optional cluster name
err := scaffoldService.RunScaffoldWorkflow(ctx, args, flags)
if err != nil {
    log.Fatal(err)
}

// Check if Skaffold is running
if scaffoldService.IsRunning() {
    fmt.Println("Skaffold is currently running")
}
```
