This file provides the core service for managing Skaffold development workflows in the OpenFrame CLI, orchestrating cluster setup, chart installation, and development environment management.

## Key Components

- **Service**: Main struct that coordinates Skaffold development workflows with dependency injection for executor and kubectl provider
- **NewService()**: Constructor that initializes the service with command executor and verbosity settings
- **RunScaffoldWorkflow()**: Primary workflow method that orchestrates service selection, cluster setup, chart installation, and Skaffold execution
- **checkPrerequisites()**: Validates and optionally installs Skaffold if missing
- **runSkaffoldDev()**: Executes Skaffold development mode with retry logic and signal handling
- **Signal Management**: Graceful shutdown handling for development sessions

## Usage Example

```go
// Initialize the service
executor := executor.NewCommandExecutor()
service := NewService(executor, true) // verbose mode

// Set up flags
flags := &models.ScaffoldFlags{
    SkipBootstrap:   false,
    HelmValuesFile:  "values.yaml",
    Namespace:      "development",
}

// Run the complete workflow
ctx := context.Background()
args := []string{"my-cluster"} // optional cluster name

err := service.RunScaffoldWorkflow(ctx, args, flags)
if err != nil {
    log.Fatal(err)
}

// Check if service is running
if service.IsRunning() {
    service.Stop() // manually stop if needed
}
```

The service handles the complete development workflow: prerequisite checks, interactive service selection, cluster management, Helm chart installation with timeouts, and Skaffold execution with automatic retry logic and graceful shutdown capabilities.
