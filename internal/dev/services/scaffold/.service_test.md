This file contains comprehensive unit tests for the scaffold service, focusing on testing the Skaffold development workflow functionality with mocked dependencies.

## Key Components

- **MockExecutor**: Mock implementation of the executor interface for isolated testing
- **TestNewService**: Validates service initialization with proper dependency injection
- **TestService_GetClusterName**: Tests cluster name resolution from command arguments
- **TestService_BuildSkaffoldArgs**: Tests Skaffold command argument construction with various flag configurations
- **TestService_IsRunning/Stop**: Tests service state management and lifecycle operations

## Usage Example

```go
// Create a mock executor for testing
mockExec := &MockExecutor{}
service := NewService(mockExec, true)

// Test service initialization
assert.NotNil(t, service)
assert.True(t, service.verbose)

// Test Skaffold argument building
flags := &models.ScaffoldFlags{
    Namespace: "my-namespace",
}
mockService := &ui.ServiceSelection{
    ServiceName: "openframe-api",
    FilePath:    "../openframe/services/openframe-api/skaffold.yaml",
}
args := service.buildSkaffoldArgs(mockService, "openframe-api", flags)
```

The tests use testify mocks to isolate the scaffold service logic and verify proper command construction, state management, and error handling without external dependencies.
