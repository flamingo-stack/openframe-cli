This file contains comprehensive unit tests for the SystemService functionality, verifying log directory management and initialization behavior.

## Key Components

- **TestNewSystemService** - Tests default SystemService constructor
- **TestNewSystemServiceWithOptions** - Tests constructor with custom log directory
- **TestSystemService_Initialize** - Table-driven test for directory initialization with various scenarios
- **TestSystemService_GetLogDirectory** - Validates log directory retrieval
- **TestSystemService_InitializeErrorHandling** - Tests error conditions and permission failures
- **TestSystemService_MultipleInitialize** - Ensures safe repeated initialization calls

## Usage Example

```go
// Run all tests
go test ./config -v

// Run specific test
go test ./config -run TestNewSystemService

// Test with race detection
go test ./config -race

// Run tests that verify SystemService behavior
func TestMySystemUsage(t *testing.T) {
    service := NewSystemService()
    err := service.Initialize()
    if err != nil {
        t.Fatal(err)
    }
    
    logDir := service.GetLogDirectory()
    // Use logDir for your application
}
```

The tests cover happy path scenarios, error conditions, edge cases like permission failures, and ensure idempotent behavior for multiple initialization calls. They validate that the SystemService properly manages log directories in both default and custom configurations.
