This file contains unit tests for the SystemService component, which manages system configuration including log directory creation and initialization.

## Key Components

- **TestNewSystemService** - Tests default SystemService creation with temp directory
- **TestNewSystemServiceWithOptions** - Tests SystemService creation with custom log directory
- **TestSystemService_Initialize** - Comprehensive tests for service initialization with various directory scenarios
- **TestSystemService_GetLogDirectory** - Validates log directory retrieval functionality
- **TestSystemService_InitializeErrorHandling** - Tests error handling for invalid directory paths
- **TestSystemService_MultipleInitialize** - Ensures multiple initialization calls are safe

## Usage Example

```go
// Run all tests
go test -v

// Run specific test
go test -run TestNewSystemService

// Test with custom log directory
func TestCustomSetup(t *testing.T) {
    service := NewSystemServiceWithOptions("/tmp/my-logs")
    err := service.Initialize()
    if err != nil {
        t.Fatal("Failed to initialize:", err)
    }
    
    logDir := service.GetLogDirectory()
    // logDir will be "/tmp/my-logs"
}
```

The tests cover successful initialization, error handling, directory creation validation, and cleanup procedures. They ensure the SystemService properly manages log directories in both default (temp directory) and custom configurations.
