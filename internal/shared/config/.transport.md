This file provides utilities for configuring insecure TLS connections to Kubernetes API servers, primarily for local development environments like k3d/WSL2 where certificate verification needs to be bypassed.

## Key Components

- **`ApplyInsecureTLSConfig`** - Main function that configures a `rest.Config` to skip TLS certificate verification while preserving client certificate authentication
- **`ApplyInsecureTransport`** - Deprecated alias for backward compatibility

## Usage Example

```go
package main

import (
    "k8s.io/client-go/rest"
    "k8s.io/client-go/tools/clientcmd"
    "your-project/config"
)

func main() {
    // Load kubeconfig
    restConfig, err := clientcmd.BuildConfigFromFlags("", "/path/to/kubeconfig")
    if err != nil {
        panic(err)
    }
    
    // Apply insecure TLS configuration for local development
    insecureConfig := config.ApplyInsecureTLSConfig(restConfig)
    
    // Use the configured client to create Kubernetes clients
    // The client will skip server certificate verification but still
    // use client certificates for authentication
}
```

**⚠️ Security Warning:** This configuration bypasses TLS certificate verification and should only be used in local development environments. The function preserves client certificate authentication while disabling server certificate validation, making it suitable for k3d clusters with self-signed certificates or hostname mismatches.
