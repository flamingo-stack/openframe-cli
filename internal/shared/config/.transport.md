This file provides utilities for configuring Kubernetes client connections to bypass TLS certificate verification while preserving client authentication, specifically designed for local development environments.

## Key Components

- **`ApplyInsecureTLSConfig`** - Main function that configures a `rest.Config` to skip server certificate verification while maintaining client certificate authentication
- **`ApplyInsecureTransport`** - Deprecated alias for backward compatibility

## Usage Example

```go
package main

import (
    "k8s.io/client-go/rest"
    "k8s.io/client-go/tools/clientcmd"
    "your-project/config"
)

func main() {
    // Load kubeconfig
    kubeconfig, err := clientcmd.BuildConfigFromFlags("", "/path/to/kubeconfig")
    if err != nil {
        panic(err)
    }
    
    // Apply insecure TLS config for local development
    insecureConfig := config.ApplyInsecureTLSConfig(kubeconfig)
    
    // Use the modified config with Kubernetes clients
    clientset, err := kubernetes.NewForConfig(insecureConfig)
    if err != nil {
        panic(err)
    }
    
    // Client is now configured to skip TLS verification
    // but still authenticate with client certificates
}
```

This utility is particularly useful for k3d/WSL2 development setups where API servers use self-signed certificates or hostname mismatches occur. The function preserves client authentication data while disabling server certificate validation - use only in development environments, never in production.
