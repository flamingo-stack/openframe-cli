This file provides a unified interface for prompting and managing various types of authentication credentials in CLI applications.

## Key Components

- **CredentialsPrompter**: Main struct that handles credential prompting with interactive UI
- **Credential Types**: 
  - `GitHubCredentials`: Username and personal access token
  - `DockerCredentials`: Registry, username, and password
  - `GenericCredentials`: Basic username/password pairs
  - `APIKeyCredentials`: API key and secret combinations
- **Prompt Methods**: Specialized methods for each credential type with validation
- **CredentialsOptions**: Configuration struct for customizing prompt behavior
- **Utility Methods**: Validation helpers and requirement checking

## Usage Example

```go
// Create credentials prompter
prompter := config.NewCredentialsPrompter()

// Prompt for GitHub credentials
githubCreds, err := prompter.PromptForGitHubCredentials("https://github.com/private/repo")
if err != nil {
    log.Fatal(err)
}

// Prompt for Docker registry credentials
dockerCreds, err := prompter.PromptForDockerCredentials("registry.example.com")
if err != nil {
    log.Fatal(err)
}

// Prompt with custom options
opts := config.CredentialsOptions{
    AllowEmpty: false,
    DefaultUsername: "admin",
    CustomMessage: "Login to secure service",
}
creds, err := prompter.PromptWithOptions("MyService", opts)
if err != nil {
    log.Fatal(err)
}

// Validate existing credentials
err = prompter.ValidateCredentials(username, password)
if err != nil {
    log.Printf("Invalid credentials: %v", err)
}
```
