This file provides a unified credential management system for prompting users to enter authentication details for various services like GitHub, Docker registries, and APIs.

## Key Components

- **CredentialsPrompter**: Main struct that handles credential prompting with validation
- **GitHubCredentials**: Struct for GitHub username and personal access token
- **DockerCredentials**: Struct for Docker registry authentication (registry, username, password)
- **GenericCredentials**: Basic username/password credentials for general services
- **APIKeyCredentials**: API key and secret pair for API-based authentication
- **CredentialsOptions**: Configuration struct for customizing prompt behavior

## Usage Example

```go
// Create a credentials prompter
prompter := config.NewCredentialsPrompter()

// Prompt for GitHub credentials for a private repository
githubCreds, err := prompter.PromptForGitHubCredentials("https://github.com/owner/private-repo")
if err != nil {
    log.Fatal(err)
}
fmt.Printf("GitHub user: %s\n", githubCreds.Username)

// Prompt for Docker registry credentials
dockerCreds, err := prompter.PromptForDockerCredentials("registry.example.com")
if err != nil {
    log.Fatal(err)
}

// Validate existing credentials
if prompter.IsCredentialsRequired(username, password) {
    err := prompter.ValidateCredentials(username, password)
    if err != nil {
        log.Printf("Invalid credentials: %v", err)
    }
}

// Custom prompting with options
opts := config.CredentialsOptions{
    AllowEmpty:      false,
    DefaultUsername: "admin",
    CustomMessage:   "Enter database credentials:",
}
creds, err := prompter.PromptWithOptions("Database", opts)
```
