Provides a comprehensive credentials management system for prompting and validating various types of authentication credentials including GitHub, Docker, API keys, and generic username/password combinations.

## Key Components

**Types:**
- `CredentialsPrompter` - Main handler for credential prompting operations
- `GitHubCredentials` - GitHub username and token pair
- `DockerCredentials` - Docker registry authentication
- `GenericCredentials` - Basic username/password credentials
- `APIKeyCredentials` - API key and secret pair
- `CredentialsOptions` - Configuration options for custom prompting

**Methods:**
- `PromptForGitHubCredentials()` - Prompts for GitHub repository access
- `PromptForDockerCredentials()` - Prompts for Docker registry login
- `PromptForGenericCredentials()` - Prompts for basic auth credentials
- `PromptForAPIKeyCredentials()` - Prompts for API-based authentication
- `PromptWithOptions()` - Customizable credential prompting
- `ValidateCredentials()` - Validates credential requirements

## Usage Example

```go
prompter := config.NewCredentialsPrompter()

// GitHub credentials
githubCreds, err := prompter.PromptForGitHubCredentials("https://github.com/private/repo")
if err != nil {
    log.Fatal(err)
}

// Docker credentials
dockerCreds, err := prompter.PromptForDockerCredentials("registry.example.com")
if err != nil {
    log.Fatal(err)
}

// Custom options
opts := config.CredentialsOptions{
    AllowEmpty:      true,
    DefaultUsername: "admin",
    CustomMessage:   "Login to service",
}
creds, err := prompter.PromptWithOptions("MyService", opts)
```
