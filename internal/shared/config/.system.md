Provides system-level configuration and initialization for the deployment service, including log directory setup and CLI preparation. This service handles the foundational system requirements needed before other services can operate.

## Key Components

**SystemService**
- Main service struct that manages system-wide configuration
- Handles log directory creation and management
- Provides system initialization workflow

**Constructor Functions**
- `NewSystemService()` - Creates service with default temp directory logging
- `NewSystemServiceWithOptions(logDir)` - Creates service with custom log directory

**Core Methods**
- `Initialize()` - Performs complete system setup and validation
- `GetLogDirectory()` - Returns the configured log directory path
- `setupLogDirectory()` - Internal method for creating directory structure

## Usage Example

```go
// Basic system service with defaults
sysService := NewSystemService()
if err := sysService.Initialize(); err != nil {
    log.Fatal("System initialization failed:", err)
}

// Custom log directory
customService := NewSystemServiceWithOptions("/var/log/openframe")
if err := customService.Initialize(); err != nil {
    log.Fatal("Custom system setup failed:", err)
}

// Get log directory for other services
logPath := sysService.GetLogDirectory()
fmt.Printf("Logs will be stored in: %s\n", logPath)
```

The service automatically creates the log directory with appropriate permissions (0755) and provides error handling for system-level failures during initialization.
