This file provides system-level configuration and initialization functionality for the OpenFrame deployment system, handling CLI setup, logging directory management, and system-wide settings.

## Key Components

- **SystemService** - Main struct that manages system-level configuration with a configurable log directory
- **NewSystemService()** - Constructor that creates a service with default log directory in system temp folder
- **NewSystemServiceWithOptions()** - Constructor allowing custom log directory configuration
- **Initialize()** - Performs system initialization by setting up required directory structures
- **setupLogDirectory()** - Private method that creates the logging directory with proper permissions (0755)
- **GetLogDirectory()** - Getter method that returns the configured log directory path

## Usage Example

```go
// Create system service with default configuration
systemSvc := NewSystemService()

// Initialize the system (creates log directories)
if err := systemSvc.Initialize(); err != nil {
    log.Fatalf("System initialization failed: %v", err)
}

// Get the log directory for other components
logDir := systemSvc.GetLogDirectory()
fmt.Printf("Logs will be stored in: %s\n", logDir)

// Or create with custom log directory
customSystemSvc := NewSystemServiceWithOptions("/var/log/openframe")
if err := customSystemSvc.Initialize(); err != nil {
    log.Fatalf("Custom system initialization failed: %v", err)
}
```

The service ensures proper directory structure exists before other components attempt to write logs, providing a clean separation of system-level concerns from application logic.
