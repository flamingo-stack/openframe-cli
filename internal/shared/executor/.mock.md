A mock implementation of the CommandExecutor interface designed for testing, allowing simulation of command execution with configurable responses and failure scenarios.

## Key Components

- **MockCommandExecutor**: Main struct implementing CommandExecutor with configurable behavior
- **NewMockCommandExecutor()**: Factory function to create a new mock executor with default settings
- **SetShouldFail()**: Configure the mock to simulate command failures
- **SetResponse()**: Set predefined responses for specific command patterns
- **Execute() / ExecuteWithOptions()**: Mock command execution methods that log commands and return configured responses
- **GetExecutedCommands()**: Retrieve the history of executed commands for verification
- **WasCommandExecuted()**: Check if a command containing a specific pattern was executed
- **Reset()**: Clear all command history and reset configuration

## Usage Example

```go
// Create a mock executor
mock := NewMockCommandExecutor()

// Configure specific response for git commands
gitResult := &CommandResult{
    ExitCode: 0,
    Stdout:   "commit abc123",
    Duration: 50 * time.Millisecond,
}
mock.SetResponse("git", gitResult)

// Execute a command
ctx := context.Background()
result, err := mock.Execute(ctx, "git", "log", "--oneline")

// Verify execution in tests
if !mock.WasCommandExecuted("git log") {
    t.Error("Expected git log command to be executed")
}

// Simulate failures
mock.SetShouldFail(true, "network timeout")
result, err = mock.Execute(ctx, "curl", "https://example.com")
// err will contain "mock command failure: network timeout"

// Check command history
commands := mock.GetExecutedCommands()
fmt.Printf("Executed %d commands\n", len(commands))
```
