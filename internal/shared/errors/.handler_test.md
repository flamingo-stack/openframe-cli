This test file provides comprehensive test coverage for error handling functions that manage user confirmation interruptions and error wrapping.

## Key Components

- **TestHandleConfirmationError**: Tests the main error handler that detects "interrupted" errors and triggers application exit
- **TestWrapConfirmationError**: Tests error wrapping functionality that adds contextual information to errors  
- **TestHandleConfirmationErrorIntegration**: Integration tests for error detection logic without actual `os.Exit` calls
- **TestErrorMessageFormatting**: Validates proper error message formatting with context
- **TestNilErrorHandling**: Ensures nil error inputs are handled correctly
- **TestEdgeCases**: Tests edge cases including empty messages, partial string matches, and special characters

## Usage Example

```go
// Run all tests
go test ./errors

// Run specific test
go test -run TestHandleConfirmationError ./errors

// Test with verbose output
go test -v ./errors

// Example of what the tests validate:
err := fmt.Errorf("interrupted")
result := HandleConfirmationError(err) // Would call os.Exit(1) in real usage

wrappedErr := WrapConfirmationError(fmt.Errorf("connection failed"), "database error")
// Returns: "database error: connection failed"
```

The tests use table-driven testing patterns and validate both normal operation and edge cases while working around the limitation of testing functions that call `os.Exit`.
