This file contains comprehensive test cases for error handling functions, specifically focusing on confirmation error handling and user interruption scenarios.

## Key Components

- **TestHandleConfirmationError**: Tests the main error handler function with various error types including nil, regular errors, and interruption errors
- **TestWrapConfirmationError**: Validates error wrapping functionality with contextual information
- **TestHandleConfirmationErrorIntegration**: Integration tests for error detection logic without triggering os.Exit
- **TestErrorMessageFormatting**: Ensures proper error message formatting with context
- **TestNilErrorHandling**: Validates nil error handling behavior
- **TestEdgeCases**: Tests edge cases including empty messages, partial matches, and special characters

## Usage Example

```go
// Run all tests
go test ./errors

// Run specific test
go test -run TestHandleConfirmationError ./errors

// Run with verbose output
go test -v ./errors

// Test the error handling logic (from test patterns)
err := fmt.Errorf("interrupted")
isInterrupted := err.Error() == "interrupted" // true

// Test error wrapping
wrappedErr := WrapConfirmationError(
    fmt.Errorf("connection failed"), 
    "database operation",
)
// Results in: "database operation: connection failed"
```

The tests validate that "interrupted" errors trigger program exit while other errors are properly wrapped with contextual information for better debugging.
