Provides utilities for handling user confirmation prompt errors, particularly user interruptions (Ctrl+C), with graceful exit handling and error wrapping capabilities.

## Key Components

- **`HandleConfirmationError(err error) bool`** - Handles user interruption errors by detecting "interrupted" or "interrupt" messages, displaying a cancellation message, and exiting the program
- **`WrapConfirmationError(err error, context string) error`** - Wraps confirmation errors with additional context while handling interruptions gracefully

## Usage Example

```go
package main

import (
    "github.com/pterm/pterm"
    "your-project/errors"
)

func confirmAction() error {
    confirmed, err := pterm.DefaultConfirm.Show("Continue with operation?")
    
    // Handle interruption gracefully - exits program if user cancels
    if errors.HandleConfirmationError(err) {
        return nil // Won't be reached due to os.Exit
    }
    
    if err != nil {
        return err
    }
    
    if !confirmed {
        return fmt.Errorf("operation declined")
    }
    
    return nil
}

func processWithContext() error {
    confirmed, err := pterm.DefaultConfirm.Show("Delete all files?")
    
    // Wrap error with context while handling interruptions
    return errors.WrapConfirmationError(err, "file deletion confirmation")
}
```

The handler detects both legacy "interrupted" and pterm "interrupt" error messages, ensuring compatibility across different prompt libraries while providing consistent user experience.
