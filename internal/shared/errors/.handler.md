Provides centralized error handling for user confirmation prompts, with special handling for interruption scenarios that gracefully exit the application.

## Key Components

- **HandleConfirmationError(err error) bool** - Handles user interruption errors (Ctrl+C) by displaying an info message and exiting the program. Returns true if the error was handled.
- **WrapConfirmationError(err error, context string) error** - Wraps confirmation errors with additional context while gracefully handling interruptions through HandleConfirmationError.

## Usage Example

```go
// Handle confirmation prompt errors
confirmed, err := pterm.DefaultInteractiveConfirm.Show("Continue with operation?")
if errors.HandleConfirmationError(err) {
    // Program exits if user interrupted (Ctrl+C)
    return
}

// Or wrap with context for non-interruption errors
if err := someConfirmationFunction(); err != nil {
    wrappedErr := errors.WrapConfirmationError(err, "failed to get user confirmation")
    if wrappedErr != nil {
        return wrappedErr
    }
}
```

The handler recognizes both "interrupted" and "interrupt" error messages, making it compatible with different prompt libraries. When an interruption is detected, it displays a user-friendly message via pterm and cleanly exits the application rather than showing an error.
