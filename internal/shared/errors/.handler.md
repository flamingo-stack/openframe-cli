This package provides utilities for handling user confirmation prompt errors, specifically managing interruption signals when users cancel operations.

## Key Components

**`HandleConfirmationError(err error) bool`**
- Handles interruption errors from user confirmation prompts
- Detects "interrupted" and "interrupt" error messages
- Gracefully exits the program with user-friendly messaging
- Returns true if error was handled (program exits), false otherwise

**`WrapConfirmationError(err error, context string) error`**
- Wraps confirmation errors with additional context
- Handles interruptions gracefully via HandleConfirmationError
- Returns wrapped error with context for non-interruption errors

## Usage Example

```go
package main

import (
    "errors"
    "github.com/your-org/your-app/errors"
)

func confirmAction() error {
    // Simulate user confirmation prompt
    userConfirmed, err := getUserConfirmation("Delete this file?")
    
    // Handle interruption gracefully
    if errors.HandleConfirmationError(err) {
        return nil // Won't be reached if interrupted
    }
    
    if err != nil {
        return errors.WrapConfirmationError(err, "failed to get user confirmation")
    }
    
    if !userConfirmed {
        return errors.New("user declined action")
    }
    
    return nil
}
```

The handler automatically displays "Operation cancelled by user." and exits cleanly when users interrupt confirmation prompts with Ctrl+C.
