This file provides utilities for handling user confirmation prompt errors, specifically managing user interruptions and providing graceful exit behavior.

## Key Components

- **`HandleConfirmationError(err error) bool`** - Processes confirmation prompt errors, handling user interruptions by displaying a cancellation message and exiting the program
- **`WrapConfirmationError(err error, context string) error`** - Wraps confirmation errors with additional context while gracefully handling interruptions

## Usage Example

```go
// Handle a confirmation prompt error directly
err := confirmationPrompt()
if HandleConfirmationError(err) {
    // Program will exit if user cancelled
    return
}

// Wrap confirmation error with context
err = getUserConfirmation()
wrappedErr := WrapConfirmationError(err, "failed to get user confirmation")
if wrappedErr != nil {
    log.Fatal(wrappedErr)
}

// In practice with a confirmation flow
confirm, err := pterm.DefaultInteractiveConfirm.Show("Continue?")
if HandleConfirmationError(err) {
    return // User cancelled - program exits gracefully
}
if !confirm {
    fmt.Println("Operation aborted by user choice")
    return
}
```

The handlers recognize both "interrupted" and "interrupt" error messages, making them compatible with different prompt libraries while providing consistent user experience.
