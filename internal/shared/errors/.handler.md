Provides utilities for handling user confirmation prompt errors with graceful interruption handling and contextual error wrapping.

## Key Components

- **HandleConfirmationError(err error) bool**: Handles interruption errors from user confirmation prompts by detecting "interrupted" or "interrupt" messages, displaying a user-friendly cancellation message, and exiting the program
- **WrapConfirmationError(err error, context string) error**: Wraps confirmation errors with additional context while gracefully handling interruptions through the HandleConfirmationError function

## Usage Example

```go
// Handle confirmation prompt errors
err := someConfirmationPrompt()
if HandleConfirmationError(err) {
    // Program will exit if user cancelled
    return
}

// Wrap confirmation errors with context
err = anotherConfirmationPrompt()
wrappedErr := WrapConfirmationError(err, "failed to get user confirmation")
if wrappedErr != nil {
    // Handle other types of errors
    log.Fatal(wrappedErr)
}

// Typical usage in confirmation flows
if err := getUserConfirmation("Continue?"); HandleConfirmationError(err) {
    return // Clean exit on user cancellation
} else if err != nil {
    return WrapConfirmationError(err, "confirmation prompt failed")
}
```
