A utility package that provides standardized error handling for user confirmation prompts, with special handling for user interruptions (Ctrl+C).

## Key Components

- **`HandleConfirmationError(err error) bool`** - Handles interruption errors from confirmation prompts, exits gracefully with user-friendly message
- **`WrapConfirmationError(err error, context string) error`** - Wraps confirmation errors with additional context while handling interruptions

## Usage Example

```go
package main

import (
    "github.com/pterm/pterm"
    "your-project/errors"
)

func confirmAction() error {
    result, err := pterm.DefaultInteractiveConfirm.Show("Do you want to continue?")
    
    // Handle interruption gracefully - exits program if user presses Ctrl+C
    if errors.HandleConfirmationError(err) {
        return nil // Won't be reached
    }
    
    if !result {
        return fmt.Errorf("user declined to continue")
    }
    
    return nil
}

func performOperation() error {
    err := confirmAction()
    
    // Wrap with context, handling interruptions automatically
    return errors.WrapConfirmationError(err, "failed to get user confirmation")
}
```

The handler detects both "interrupted" and "interrupt" error messages, displays a clean cancellation message using pterm, and exits the program gracefully rather than showing a raw error stack trace.
