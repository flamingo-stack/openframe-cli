This file provides centralized error handling utilities specifically for user confirmation prompts and interruption scenarios.

## Key Components

- **`HandleConfirmationError(err error) bool`** - Handles user interruption errors (Ctrl+C) by displaying a cancellation message and exiting gracefully. Returns true if the error was handled.

- **`WrapConfirmationError(err error, context string) error`** - Wraps confirmation errors with additional context while gracefully handling interruptions. Combines error wrapping with interruption handling.

## Usage Example

```go
package main

import (
    "errors"
    "github.com/your-repo/pkg/errors"
    "github.com/pterm/pterm"
)

func confirmDestructiveAction() error {
    result, err := pterm.DefaultInteractiveConfirm.
        WithDefaultText("Delete all files?").
        Show()
    
    // Handle interruption gracefully
    if errors.HandleConfirmationError(err) {
        return nil // Won't reach here due to os.Exit
    }
    
    if err != nil {
        return fmt.Errorf("confirmation failed: %w", err)
    }
    
    if !result {
        return errors.New("user declined")
    }
    
    return nil
}

// Alternative usage with context wrapping
func processWithConfirmation() error {
    _, err := pterm.DefaultInteractiveConfirm.Show()
    
    return errors.WrapConfirmationError(err, "processing confirmation")
}
```

The handlers detect both "interrupted" and "interrupt" error messages to ensure compatibility with different prompt libraries, providing consistent user experience when operations are cancelled via Ctrl+C.
