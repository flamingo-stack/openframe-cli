This file implements a comprehensive retry policy system for handling recoverable errors with various backoff strategies and execution patterns.

## Key Components

**Interfaces:**
- `RetryPolicy` - Defines retry behavior interface with attempt limits and delay calculation
- `RecoverableError` - Interface for errors that support retry with custom retry-after duration

**Policy Implementations:**
- `ExponentialBackoffPolicy` - Implements exponential backoff with jitter and configurable error patterns
- `LinearBackoffPolicy` - Implements linear delay increases for predictable retry timing

**Execution Framework:**
- `RetryExecutor` - Orchestrates retry logic with context cancellation and callback support
- `DefaultRetryCallback`, `QuietRetryCallback`, `VerboseRetryCallback` - Pre-built logging callbacks

**Predefined Policies:**
- `NetworkRetryPolicy()` - Optimized for network timeouts and connection issues
- `ResourceRetryPolicy()` - Designed for resource availability scenarios
- `InstallationRetryPolicy()` - Tailored for deployment and installation operations

## Usage Example

```go
// Create exponential backoff policy
policy := NewExponentialBackoffPolicy(3, 2*time.Second)

// Create executor with logging
executor := NewRetryExecutor(policy).
    WithRetryCallback(DefaultRetryCallback("API call"))

// Execute operation with retries
err := executor.Execute(ctx, func() error {
    return apiClient.CreateResource(resource)
})

// Or execute with result
result, err := executor.ExecuteWithResult(ctx, func() (interface{}, error) {
    return apiClient.GetResource(resourceID)
})

// Use predefined policies for common scenarios
networkExecutor := NewRetryExecutor(NetworkRetryPolicy())
```

The system provides context-aware cancellation, configurable error pattern matching, and both simple operations and result-returning operations support.
