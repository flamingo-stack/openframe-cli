This file implements a comprehensive retry mechanism for handling recoverable errors with configurable backoff policies and execution strategies.

## Key Components

**Interfaces:**
- `RetryPolicy` - Defines retry behavior (should retry, delay calculation, max attempts)
- `RecoverableError` - Interface for errors that can be retried with custom logic

**Retry Policies:**
- `ExponentialBackoffPolicy` - Implements exponential backoff with jitter and configurable error patterns
- `LinearBackoffPolicy` - Implements linear delay increments

**Execution:**
- `RetryExecutor` - Handles retry logic with context cancellation support
- `NewRetryExecutor()` - Creates executor with specified policy
- `Execute()` - Retries operations without return values
- `ExecuteWithResult()` - Retries operations with return values

**Predefined Policies:**
- `NetworkRetryPolicy()` - For network operations (5 attempts, 2s-30s delays)
- `ResourceRetryPolicy()` - For resource availability (10 attempts, 5s-2m delays)
- `InstallationRetryPolicy()` - For installation operations (3 attempts, 10s-5m delays)

**Callback Functions:**
- `DefaultRetryCallback()` - Standard progress indication
- `QuietRetryCallback()` - Minimal logging
- `VerboseRetryCallback()` - Detailed retry information

## Usage Example

```go
// Basic exponential backoff
policy := NewExponentialBackoffPolicy(3, time.Second)
executor := NewRetryExecutor(policy).
    WithRetryCallback(DefaultRetryCallback("API call"))

// Execute with retry
err := executor.Execute(ctx, func() error {
    return callAPI()
})

// Use predefined policy for network operations
networkExecutor := NewRetryExecutor(NetworkRetryPolicy())
result, err := networkExecutor.ExecuteWithResult(ctx, func() (interface{}, error) {
    return httpClient.Get("https://api.example.com")
})
```
