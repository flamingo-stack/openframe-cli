This package provides centralized error handling for CLI applications with user-friendly formatting and standardized error types. It includes custom error types for validation and command failures, plus an error handler that displays errors with consistent styling using pterm.

## Key Components

**Error Types:**
- `ValidationError` - Validation failures with field, value, and message details
- `CommandError` - Command execution errors with command name, args, and underlying error
- `BranchNotFoundError` - Git branch not found errors
- `AlreadyHandledError` - Wrapper for errors already displayed to users

**Error Handler:**
- `ErrorHandler` - Central error processor with verbose mode support
- `HandleError()` - Routes errors to specific handlers based on type
- `HandleGlobalError()` - Global entry point for CLI command error handling

**Helper Functions:**
- `CreateValidationError()`, `CreateCommandError()` - Constructor functions
- `IsValidationError()`, `IsCommandError()` - Type checking utilities

## Usage Example

```go
// Create and handle a validation error
err := CreateValidationError("port", "invalid", "must be a valid port number")
handler := NewErrorHandler(true) // verbose mode
handler.HandleError(err)

// Handle command errors
cmdErr := CreateCommandError("kubectl", []string{"apply", "-f", "config.yaml"}, errors.New("connection failed"))
handler.HandleError(cmdErr)

// Global error handling in CLI commands
func runCommand(cmd *cobra.Command, args []string) error {
    if err := someOperation(); err != nil {
        return HandleGlobalError(err, verbose)
    }
    return nil
}

// Check error types
if IsValidationError(err) {
    // Handle validation-specific logic
}
```
