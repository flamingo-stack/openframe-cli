Provides standardized error handling and user-friendly error display for the OpenFrame CLI application. The package defines custom error types and handlers with colored terminal output and contextual troubleshooting guidance.

## Key Components

**Error Types:**
- `ValidationError` - Field validation failures with field, value, and message details
- `CommandError` - Command execution failures with command name, arguments, and underlying error
- `BranchNotFoundError` - Git branch not found errors with helpful suggestions
- `AlreadyHandledError` - Wrapper for errors already displayed to prevent double-showing

**Error Handler:**
- `ErrorHandler` - Central error processor with verbose mode support
- `HandleError()` - Routes errors to appropriate display handlers based on type
- `HandleGlobalError()` - Global entry point for consistent CLI error handling

**Utility Functions:**
- `CreateValidationError()`, `CreateCommandError()` - Constructor functions
- `IsValidationError()`, `IsCommandError()` - Type checking utilities
- `isUserInterruption()` - Detects Ctrl+C and other user cancellations

## Usage Example

```go
// Create and handle a validation error
handler := NewErrorHandler(verbose)
err := CreateValidationError("port", "invalid", "must be a valid port number")
handler.HandleError(err)

// Global error handling in commands
func (cmd *Command) RunE(cmd *cobra.Command, args []string) error {
    if err := someOperation(); err != nil {
        return HandleGlobalError(err, verbose)
    }
    return nil
}

// Command execution error
cmdErr := CreateCommandError("k3d", []string{"cluster", "create"}, originalError)
handler.HandleError(cmdErr)
```

The package uses [pterm](https://github.com/pterm/pterm) for colored terminal output and provides specific troubleshooting guidance for common failures like Docker/k3d cluster creation issues.
