Provides comprehensive error handling infrastructure with typed errors and standardized user-friendly error display. Features specific error types for validation, command execution, and branch operations, along with an ErrorHandler that formats errors with colored output and appropriate troubleshooting guidance.

## Key Components

- **Error Types**: `ValidationError`, `CommandError`, `AlreadyHandledError`, `BranchNotFoundError`
- **ErrorHandler**: Centralized error processing with verbose/non-verbose modes
- **Helper Functions**: `CreateValidationError`, `CreateCommandError`, `NewBranchNotFoundError`
- **Type Checkers**: `IsValidationError`, `IsCommandError`
- **Global Handler**: `HandleGlobalError` for consistent CLI error handling

## Usage Example

```go
// Create and handle validation errors
validationErr := CreateValidationError("cluster-name", "invalid-name!", "contains invalid characters")
handler := NewErrorHandler(false)
handler.HandleError(validationErr)

// Handle command errors
cmdErr := CreateCommandError("k3d", []string{"cluster", "create"}, fmt.Errorf("docker not running"))
handler.HandleError(cmdErr)

// Global error handling in CLI commands
func runCommand() error {
    // ... command logic
    if err != nil {
        return HandleGlobalError(err, verbose)
    }
    return nil
}

// Check error types
if IsValidationError(err) {
    // Handle validation-specific logic
}
```
