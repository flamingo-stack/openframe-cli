A comprehensive error handling package that provides standardized error types, formatting, and user-friendly display for CLI applications. It includes specialized error types for validation, command execution, and branch operations with consistent visual feedback.

## Key Components

- **ValidationError**: Custom error for field validation failures with optional value display
- **CommandError**: Wraps command execution errors with command context and argument details
- **BranchNotFoundError**: Specialized error for Git branch operations with helpful messaging
- **AlreadyHandledError**: Wrapper to prevent duplicate error display in error chains
- **ErrorHandler**: Main handler with verbose mode support and colored terminal output
- **HandleGlobalError**: Global entry point for consistent CLI error handling with graceful exits

## Usage Example

```go
// Create specialized errors
validationErr := CreateValidationError("port", "invalid", "must be a valid port number")
cmdErr := CreateCommandError("k3d", []string{"cluster", "create"}, fmt.Errorf("exit status 1"))
branchErr := NewBranchNotFoundError("feature-branch")

// Use error handler for consistent display
handler := NewErrorHandler(true) // verbose mode
handler.HandleError(validationErr)

// Global error handling for CLI commands
func runCommand() error {
    // ... command logic
    if err != nil {
        return HandleGlobalError(err, verbose) // Displays error and exits
    }
    return nil
}

    // Handle validation-specific logic
}
```
