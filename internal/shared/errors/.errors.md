This package provides a comprehensive error handling system for the OpenFrame CLI tool, offering specialized error types and user-friendly error display with colored terminal output.

## Key Components

### Error Types
- **`ValidationError`** - Represents field validation failures with field name, value, and message
- **`CommandError`** - Wraps command execution errors with command name, arguments, and underlying error
- **`AlreadyHandledError`** - Wrapper for errors already displayed to avoid double handling
- **`BranchNotFoundError`** - Specific error for Git branch resolution failures

### Error Handler
- **`ErrorHandler`** - Main error processing struct with verbose mode support
- **`HandleError()`** - Routes errors to appropriate specialized handlers
- **`HandleGlobalError()`** - Global entry point for CLI command error handling

### Utility Functions
- **Helper functions** for creating specific error types
- **Type checking functions** (`IsValidationError`, `IsCommandError`)
- **User interruption detection** for graceful Ctrl+C handling

## Usage Example

```go
// Create and handle a validation error
err := CreateValidationError("cluster-name", "my-cluster!", "contains invalid characters")
handler := NewErrorHandler(true) // verbose mode
handler.HandleError(err)

// Global error handling in CLI commands
func myCommand(cmd *cobra.Command, args []string) error {
    err := performOperation()
    return HandleGlobalError(err, verbose)
}

// Command error with troubleshooting
cmdErr := CreateCommandError("k3d", []string{"cluster", "create"}, originalErr)
handler.HandleError(cmdErr)
```

The error handler provides context-aware formatting with colored output, troubleshooting suggestions for common failures (like Docker/k3d issues), and clean handling of user interruptions.
