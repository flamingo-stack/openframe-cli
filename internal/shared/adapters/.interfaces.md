Core interface definitions and base adapter implementations for Cobra command handling, providing a structured approach to command execution, flag management, and validation.

## Key Components

**Interfaces:**
- `CommandHandler` - Defines command execution interface
- `FlagsProvider` - Handles flag definitions and registration
- `CommandAdapter` - Combines command handling and flag management

**Base Implementation:**
- `BaseCommandAdapter` - Common functionality for command adapters with flag extraction and validation

**Error Types:**
- `RequiredFlagError` - Missing required flag validation
- `InvalidFlagError` - Invalid flag value validation

**Helper Types:**
- `FlagDefinition` - Command flag specification
- `CommandMetadata` - Command metadata container
- `ExampleBuilder` - Fluent API for building command examples
- `PreRunEChain` - Chains multiple PreRunE validation functions

## Usage Example

```go
// Create a command adapter
type MyCommandAdapter struct {
    *BaseCommandAdapter
}

func (mca *MyCommandAdapter) Execute(cmd *cobra.Command, args []string) error {
    flags := mca.ExtractFlags(cmd)
    
    name, _ := flags.GetString("name")
    port, _ := flags.GetInt("port")
    
    // Validate required flags
    validation := mca.ValidateRequired(map[string]interface{}{
        "name": name,
        "port": port,
    })
    
    if !validation.IsValid() {
        return validation.Errors[0]
    }
    
    // Execute command logic
    return nil
}

// Build examples with fluent API
examples := NewExampleBuilder().
    Add("Start server on port 8080", "--port 8080").
    Add("Start with custom name", "--name myserver --port 3000").
    Build("mycommand")

// Chain PreRunE functions
preRun := NewPreRunEChain().
    Add(validateConfig).
    Add(checkPermissions).
    Build()
```
