Test suite for UI table rendering and display functionality using the pterm library for terminal output formatting.

## Key Components

- **TestGetStatusColor**: Validates status-to-color mapping for various states (running→green, stopped→yellow, error→red, etc.)
- **TestRenderTableWithFallback**: Tests generic table rendering with optional header display
- **TestRenderKeyValueTable**: Tests property-value table rendering functionality  
- **TestRenderNodeTable**: Tests node-specific table rendering for cluster information
- **TestShowSuccessBox**: Tests success message box display functionality

## Usage Example

```go
// Run specific test
go test -run TestGetStatusColor

// Run all table tests
go test -v ./ui

// Test with coverage
go test -cover ./ui

// Example test data structure used in tests
testData := pterm.TableData{
    {"Name", "Type", "Status", "Nodes", "Created"},
    {"test-cluster", "k3d", "running", "3", "2023-01-01"},
}
```

The tests focus on ensuring UI functions don't panic and handle various input scenarios correctly, particularly testing color mapping for different status strings and table rendering with different configurations.
