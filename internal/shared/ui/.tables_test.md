Test suite for UI table rendering and status color functionality, ensuring proper formatting and error handling for various display components.

## Key Components

- **TestGetStatusColor** - Validates status-to-color mapping for different cluster states (running→green, stopped→yellow, error→red, etc.)
- **TestRenderTableWithFallback** - Tests table rendering with and without headers using sample cluster data
- **TestRenderKeyValueTable** - Verifies key-value pair table formatting for property displays
- **TestRenderNodeTable** - Tests node information table rendering with role and status columns
- **TestShowSuccessBox** - Validates success message box display functionality

## Usage Example

```go
func TestCustomStatus(t *testing.T) {
    // Test status color mapping
    colorFunc := GetStatusColor("running")
    if colorFunc == nil {
        t.Error("Expected color function for running status")
    }
    
    // Test table rendering
    tableData := pterm.TableData{
        {"Name", "Status"},
        {"my-cluster", "running"},
    }
    
    err := RenderTableWithFallback(tableData, true)
    if err != nil {
        t.Errorf("Table rendering failed: %v", err)
    }
}
```

These tests ensure the UI components handle various data formats gracefully and don't panic during rendering operations.
