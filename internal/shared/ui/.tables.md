Provides table rendering utilities with graceful fallbacks for CLI output when the pterm library fails to render styled tables.

## Key Components

- **RenderTableWithFallback()** - Renders a 5-column table with formatted fallback output for cluster commands
- **RenderKeyValueTable()** - Displays data in key-value pairs with simple text fallback
- **RenderNodeTable()** - Specialized table renderer for node information (name, role, status, age)
- **GetStatusColor()** - Returns appropriate color functions based on status values (green for running/ready, yellow for pending, red for errors)
- **ShowSuccessBox()** - Displays success messages in a formatted box with centered title

## Usage Example

```go
// Render a cluster status table
data := pterm.TableData{
    {"Node", "Role", "Status", "Version", "Age"},
    {"master-01", "control", "ready", "v1.28", "5d"},
    {"worker-01", "worker", "running", "v1.28", "5d"},
}
err := RenderTableWithFallback(data, true)

// Display key-value information
kvData := pterm.TableData{
    {"Key", "Value"},
    {"Cluster Name", "production"},
    {"Node Count", "3"},
}
err = RenderKeyValueTable(kvData)

// Color status text
colorFunc := GetStatusColor("running")
fmt.Println(colorFunc("Service is running"))

// Show success message
ShowSuccessBox("Deployment Complete", "All services started successfully")
```
