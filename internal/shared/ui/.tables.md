Provides table rendering utilities with graceful fallbacks for CLI output when advanced terminal features are unavailable.

## Key Components

- **RenderTableWithFallback** - Renders 5-column tables with formatted fallback output for cluster command displays
- **RenderKeyValueTable** - Displays key-value pairs in table format with simple fallback to "key: value" format
- **RenderNodeTable** - Specialized table renderer for node information with fixed column layout
- **GetStatusColor** - Returns appropriate color functions based on status strings (green for running/ready, yellow for pending, red for errors)
- **ShowSuccessBox** - Creates formatted success message boxes with centered titles

## Usage Example

```go
// Render a cluster status table
data := pterm.TableData{
    {"Name", "Status", "Type", "Age", "Ready"},
    {"cluster-1", "Running", "Master", "2d", "True"},
    {"cluster-2", "Pending", "Worker", "1h", "False"},
}
RenderTableWithFallback(data, true)

// Display colored status
colorFunc := GetStatusColor("running")
fmt.Println(colorFunc("Running")) // Prints in green

// Show success message
ShowSuccessBox("Operation Complete", "Cluster successfully created")

// Key-value display
kvData := pterm.TableData{
    {"Key", "Value"},
    {"Name", "my-cluster"},
    {"Version", "1.28.0"},
}
RenderKeyValueTable(kvData)
```
