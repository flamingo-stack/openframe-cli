Package ui provides table rendering utilities with fallback mechanisms for robust terminal output across different environments.

## Key Components

- **RenderTableWithFallback** - Renders a general purpose table with automatic fallback to plain text when PTerm fails
- **RenderKeyValueTable** - Specialized renderer for key-value pair tables with fallback
- **RenderNodeTable** - Dedicated node information table renderer with fixed column formatting
- **GetStatusColor** - Returns color functions based on status values (running=green, error=red, etc.)
- **ShowSuccessBox** - Displays formatted success messages in styled boxes

## Usage Example

```go
// Render a cluster status table
data := pterm.TableData{
    {"Name", "Status", "CPU", "Memory", "Uptime"},
    {"node-1", "running", "45%", "2.1GB", "5d"},
    {"node-2", "stopped", "0%", "0GB", "0s"},
}
err := ui.RenderTableWithFallback(data, true)

// Render key-value configuration
configData := pterm.TableData{
    {"Key", "Value"},
    {"cluster-name", "production"},
    {"region", "us-west-2"},
}
err = ui.RenderKeyValueTable(configData)

// Color status text
colorFunc := ui.GetStatusColor("running")
fmt.Println(colorFunc("Service is running")) // Prints in green

// Show success message
ui.ShowSuccessBox("Deployment Complete", "All nodes are healthy and ready")
```

The module automatically handles terminal compatibility issues by falling back to plain text formatting when PTerm rendering fails, ensuring consistent output across different environments.
