Provides a standardized message templating system for consistent CLI output formatting with predefined templates for different message types and operations.

## Key Components

- **MessageType**: Enum defining message categories (Info, Success, Warning, Error, Progress, Completion)
- **Templates**: Core struct containing predefined message templates with emoji and formatting
- **NewTemplates()**: Factory function creating templates instance with built-in message formats
- **FormatMessage()**: Template formatter that applies arguments to message templates
- **Show* methods**: Public interface for displaying different message types
- **CustomTemplates**: Extended templates allowing custom message additions
- **Formatter**: Convenience wrapper providing specialized formatters
- **InstallationFormatter/ClusterFormatter**: Domain-specific message formatters

## Usage Example

```go
// Basic usage
templates := messages.NewTemplates()
templates.ShowOperationStart("deployment", "my-cluster")
templates.ShowSuccess("operation_complete", "deployment")
templates.ShowError("connection_failed", "database", err)

// Using specialized formatters
formatter := messages.NewFormatter()
formatter.Installation().Starting("nginx", "production")
formatter.Installation().Complete("nginx", []string{
    "Access the service at http://nginx.example.com",
    "Check logs with kubectl logs -n nginx",
})

// Custom templates
custom := messages.NewCustomTemplates()

custom.ShowInfo("custom_op", "my-operation")
```
