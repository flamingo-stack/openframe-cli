A message templating system that provides standardized, emoji-enhanced messages for CLI operations using the pterm library. It includes predefined message templates for different message types and specialized formatters for installation and cluster operations.

## Key Components

- **MessageType**: Enumeration defining message categories (Info, Success, Warning, Error, Progress, Completion)
- **Templates**: Core template system with predefined message formats organized by type
- **CustomTemplates**: Extended templates allowing addition of custom message formats
- **Formatter**: Quick access interface for commonly used formatting functions
- **InstallationFormatter**: Specialized formatter for installation-specific messages
- **ClusterFormatter**: Specialized formatter for cluster-specific operations

## Usage Example

```go
// Basic template usage
templates := messages.NewTemplates()
templates.ShowOperationStart("deployment", "production-cluster")
templates.ShowSuccess("installation_complete", "ArgoCD")
templates.ShowError("connection_failed", "kubernetes-api", errors.New("timeout"))

// Using specialized formatters
formatter := messages.NewFormatter()
installFormatter := formatter.Installation()
installFormatter.Starting("ArgoCD", "production-cluster")
installFormatter.Complete("ArgoCD", []string{
    "Access the UI at https://argocd.example.com",
    "Login with admin credentials",
})

// Custom templates
customTemplates := messages.NewCustomTemplates()
customTemplates.AddTemplate(messages.InfoMessage, "custom_step", "ðŸŽ¯ Executing %s with %s")
customTemplates.ShowInfo("custom_step", "deployment", "helm")

// Step timing
templates.ShowStepComplete("Database Migration", 2*time.Minute)
templates.ShowStepFailed("Service Startup", errors.New("port already in use"), 30*time.Second)
```
