This file provides a comprehensive message templating system for CLI applications, offering standardized, emoji-enhanced messages with consistent formatting for operations like installations, validations, and cluster management.

## Key Components

**Core Types:**
- `MessageType` - Enum for message categories (Info, Success, Warning, Error, Progress, Completion)
- `Templates` - Main template engine with predefined message formats
- `CustomTemplates` - Extended templates allowing custom message additions
- `Formatter` - High-level interface for common formatting operations

**Main Functions:**
- `NewTemplates()` - Creates template instance with predefined messages
- `FormatMessage()` - Formats messages using template keys and arguments
- `Show*()` methods - Display messages with appropriate styling (ShowInfo, ShowSuccess, etc.)
- `ShowOperationStart/Complete/Failed()` - Standardized operation lifecycle messages

**Specialized Formatters:**
- `InstallationFormatter` - Installation-specific message patterns
- `ClusterFormatter` - Cluster operation messages

## Usage Example

```go
// Basic usage
templates := NewTemplates()
templates.ShowOperationStart("installation", "my-cluster")
templates.ShowSuccess("validation_passed", "configuration")
templates.ShowError("connection_failed", "database", err)

// Installation workflow
formatter := NewFormatter()
install := formatter.Installation()
install.Starting("prometheus", "prod-cluster")
install.Complete("prometheus", []string{"Access dashboard at localhost:3000"})
```
