This file provides a comprehensive message templating system for standardized console output with emoji-enhanced formatting using the pterm library.

## Key Components

- **MessageType**: Enum defining message categories (Info, Success, Warning, Error, Progress, Completion)
- **Templates**: Core struct containing predefined message templates organized by type and template name
- **CustomTemplates**: Extended template system allowing runtime addition of custom message formats
- **Formatter**: Convenience wrapper providing specialized formatters for different contexts
- **InstallationFormatter**: Installation-specific message formatting methods
- **ClusterFormatter**: Cluster operation-specific message formatting methods

## Usage Example

```go
// Basic template usage
templates := NewTemplates()
templates.ShowInfo("connecting", "my-cluster")
templates.ShowSuccess("operation_complete", "deployment")
templates.ShowError("validation_failed", "config", fmt.Errorf("invalid syntax"))

// Operation lifecycle messages
templates.ShowOperationStart("installation", "flux-system")
templates.ShowStepComplete("validation", time.Minute)
templates.ShowOperationComplete("installation")

// Specialized formatters
formatter := NewFormatter()

// Installation formatting
installer := formatter.Installation()
installer.Starting("flux", "production-cluster")
installer.Complete("flux", []string{"Configure GitOps", "Deploy applications"})

// Custom templates
custom := NewCustomTemplates()
custom.AddTemplate(InfoMessage, "custom_step", "ðŸŽ¯ %s is now %s")
custom.ShowInfo("custom_step", "service", "configured")
```

The system handles emoji icons, color formatting through pterm, and provides consistent messaging patterns across different operation types. The template indirection bypasses go vet formatting warnings while maintaining type safety.
