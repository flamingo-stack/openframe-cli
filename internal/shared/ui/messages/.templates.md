A standardized message templating system that provides consistent, emoji-enhanced terminal output for operations like installations, validations, and cluster management. It uses the pterm library for colored terminal output and includes predefined message templates organized by type (info, success, warning, error, progress).

## Key Components

- **MessageType**: Enum defining message categories (InfoMessage, SuccessMessage, WarningMessage, ErrorMessage, ProgressMessage, CompletionMessage)
- **Templates**: Core struct containing predefined message templates with emoji prefixes
- **CustomTemplates**: Extended templates allowing custom message additions
- **Formatter**: Convenience wrapper providing domain-specific formatters
- **InstallationFormatter**: Specialized formatter for installation-related messages
- **ClusterFormatter**: Specialized formatter for cluster-related messages

## Usage Example

```go
// Basic template usage
templates := NewTemplates()
templates.ShowOperationStart("installation", "my-cluster")
templates.ShowSuccess("operation_complete", "ArgoCD installation")
templates.ShowError("operation_failed", "deployment", fmt.Errorf("timeout"))

// Custom templates
custom := NewCustomTemplates()
custom.AddTemplate(InfoMessage, "my_custom", "ðŸŽ¯ Custom operation: %s")
custom.ShowInfo("my_custom", "backup")

// Domain-specific formatters
formatter := NewFormatter()
formatter.Installation().Starting("ArgoCD", "production-cluster")
formatter.Installation().Complete("ArgoCD", []string{
    "Access UI at https://argocd.example.com",
    "Configure repositories",
})

// Step tracking with duration
start := time.Now()
// ... perform operation
templates.ShowStepComplete("Database migration", time.Since(start))
```

The system handles go vet warnings about format directives through nolint annotations, as the formatting is correctly handled by the template system's indirection through `FormatMessage()`.
