A comprehensive progress tracking system that provides visual feedback for multi-step operations using pterm for terminal UI elements. It manages step execution states, calculates weighted progress, and handles operation lifecycle with cancellation support.

## Key Components

### Core Types
- **`Tracker`** - Main progress tracking coordinator with spinner and progress bar UI
- **`Step`** - Individual operation step with timing, status, and error tracking
- **`StepStatus`** - Enumeration for step states (Pending, Running, Completed, Failed, Skipped)

### Primary Methods
- **`NewTracker(operation, steps)`** - Creates tracker with cancellation context
- **`Start()`** - Initializes operation timing and spinner UI
- **`StartStep(index)`** - Begins specific step execution
- **`CompleteStep(index)`** - Marks step as successful with timing
- **`FailStep(index, error)`** - Handles step failures with error logging
- **`UpdateProgress(percentage)`** - Updates weighted progress calculation
- **`Complete()/Fail()/Cancel()`** - Operation lifecycle management

## Usage Example

```go
// Define operation steps with weights
steps := []Step{
    {Name: "Initialize", Description: "Setup environment", Weight: 1.0},
    {Name: "Download", Description: "Fetch resources", Weight: 3.0},
    {Name: "Process", Description: "Transform data", Weight: 2.0},
    {Name: "Cleanup", Description: "Remove temp files", Weight: 0.5},
}

// Create and start tracker
tracker := NewTracker("Data Pipeline", steps)
tracker.Start()

// Execute steps with progress updates
for i, step := range steps {
    if err := tracker.StartStep(i); err != nil {
        tracker.Fail(err)
        return
    }
    
    // Simulate work with progress updates
    for progress := 0; progress <= 100; progress += 25 {
        tracker.UpdateProgress(float64(progress))
        time.Sleep(500 * time.Millisecond)
    }
    
    if err := performStep(step); err != nil {
        tracker.FailStep(i, err)
        break
    }
    
    tracker.CompleteStep(i)
}

tracker.Complete()
```

The tracker provides thread-safe progress monitoring with rich terminal output, step timing, cancellation handling, and comprehensive operation summaries.
