A comprehensive progress tracking system that provides real-time visual feedback for multi-step operations with spinners, progress bars, and detailed summaries.

## Key Components

### Core Types
- **`Tracker`** - Main progress tracker with spinner and progress bar support
- **`Step`** - Represents individual operation steps with timing and status
- **`StepStatus`** - Enumeration for step states (Pending, Running, Completed, Failed, Skipped)

### Main Functions
- **`NewTracker()`** - Creates a new tracker with operation name and steps
- **`Start()`** - Begins progress tracking with visual spinner
- **`StartStep()`/`CompleteStep()`/`FailStep()`/`SkipStep()`** - Step lifecycle management
- **`UpdateProgress()`** - Updates progress bar with weighted completion percentage
- **`Complete()`/`Fail()`/`Cancel()`** - Operation completion methods

### Utilities
- **`GetProgress()`** - Returns current completion percentage
- **`GetEstimatedTimeRemaining()`** - Calculates ETA based on step weights
- **`Context()`** - Provides cancellation context for cooperative cancellation

## Usage Example

```go
// Define operation steps with weights
steps := []Step{
    {Name: "Initialize", Weight: 10.0},
    {Name: "Process Data", Weight: 70.0}, 
    {Name: "Cleanup", Weight: 20.0},
}

// Create and start tracker
tracker := NewTracker("Data Migration", steps)
tracker.Start()

// Execute steps
for i := range steps {
    tracker.StartStep(i)
    
    // Simulate work with progress updates
    for progress := 0; progress <= 100; progress += 10 {
        tracker.UpdateProgress(float64(progress))
        time.Sleep(100 * time.Millisecond)
    }
    
    tracker.CompleteStep(i)
}

tracker.Complete() // Shows summary with timing details
```

The tracker provides thread-safe operation with mutex protection, context-based cancellation support, and rich terminal output using pterm for professional progress visualization.
