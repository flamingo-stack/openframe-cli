This file provides a comprehensive progress tracking system for multi-step operations with visual feedback, cancellation support, and detailed reporting capabilities.

## Key Components

- **Tracker**: Main progress tracker that manages operation state, timing, and visual displays
- **Step**: Represents individual steps with name, description, weight, status, and timing information
- **StepStatus**: Enumeration for step states (Pending, Running, Completed, Failed, Skipped)
- **NewTracker()**: Factory function to create a new tracker with operation name and step definitions

## Usage Example

```go
// Define operation steps
steps := []Step{
    {Name: "Initialize", Description: "Setting up environment", Weight: 1.0},
    {Name: "Download", Description: "Downloading files", Weight: 3.0},
    {Name: "Install", Description: "Installing components", Weight: 2.0},
    {Name: "Configure", Description: "Configuring settings", Weight: 1.0},
}

// Create and start tracker
tracker := NewTracker("Application Setup", steps)
tracker.Start()

// Execute steps with progress tracking
for i, step := range steps {
    if err := tracker.StartStep(i); err != nil {
        tracker.Fail(err)
        return
    }
    
    // Simulate work with progress updates
    for progress := 0; progress <= 100; progress += 25 {
        tracker.UpdateProgress(float64(progress))
        time.Sleep(100 * time.Millisecond)
    }
    
    // Complete or fail step based on results
    if err := performStep(step); err != nil {
        tracker.FailStep(i, err)
        tracker.Fail(err)
        return
    }
    
    tracker.CompleteStep(i)
}

tracker.Complete()
```

The tracker provides real-time visual feedback using pterm spinners and progress bars, supports cancellation via context, calculates estimated completion times, and generates detailed operation summaries with step-by-step results and timing information.
