Provides comprehensive progress tracking for long-running operations with multi-step workflows, visual feedback using spinners and progress bars, and detailed operation summaries.

## Key Components

- **Tracker**: Main struct for tracking operation progress with thread-safe operations
- **Step**: Represents individual steps with status, timing, and error information
- **StepStatus**: Enum for step states (Pending, Running, Completed, Failed, Skipped)
- **NewTracker()**: Creates a new tracker with operation name and steps
- **Start/StartStep/CompleteStep/FailStep/SkipStep**: Step lifecycle management
- **UpdateProgress()**: Updates visual progress indicators
- **Complete/Fail/Cancel**: Operation completion methods
- **Context()**: Returns cancellation context for operation coordination

## Usage Example

```go
// Define operation steps
steps := []progress.Step{
    {Name: "Initialize", Description: "Setting up resources", Weight: 1.0},
    {Name: "Process Data", Description: "Processing input data", Weight: 3.0},
    {Name: "Cleanup", Description: "Cleaning up resources", Weight: 1.0},
}

// Create and start tracker
tracker := progress.NewTracker("Data Processing", steps)
tracker.Start()

// Execute steps
for i, step := range steps {
    if err := tracker.StartStep(i); err != nil {
        log.Fatal(err)
    }
    
    // Simulate work with progress updates
    for p := 0; p <= 100; p += 20 {
        tracker.UpdateProgress(float64(p))
        time.Sleep(100 * time.Millisecond)
    }
    
    if err := tracker.CompleteStep(i); err != nil {
        log.Fatal(err)
    }
}

// Complete operation
tracker.Complete()
```
