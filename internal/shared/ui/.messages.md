This file provides a consistent messaging system for CLI operations using styled terminal output via the pterm library.

## Key Components

- **ShowNoResourcesMessage**: Displays a friendly warning box when no resources are found, with helpful next steps
- **ShowOperationStart**: Shows an info message when beginning an operation
- **ShowOperationSuccess**: Displays a success message when operations complete successfully  
- **ShowOperationError**: Shows detailed error messages with optional troubleshooting tips
- **TroubleshootingTip**: Struct for organizing help suggestions with descriptions and commands

## Usage Example

```go
package main

import "github.com/openframe-org/openframe/internal/ui"

func main() {
    // Show message when no resources exist
    ui.ShowNoResourcesMessage(
        "projects", 
        "deploy", 
        "openframe create project", 
        "openframe list projects",
    )
    
    // Show operation progress
    ui.ShowOperationStart("deploy", "my-app", nil)
    
    // Show success
    ui.ShowOperationSuccess("deploy", "my-app", nil)
    
    // Show error with troubleshooting
    tips := []ui.TroubleshootingTip{
        {Description: "Check logs", Command: "openframe logs my-app"},
        {Description: "Verify config", Command: "openframe config show"},
    }
    ui.ShowOperationError("deploy", "my-app", err, tips)
}
```

The functions support custom messages through the `customMessages` parameter and provide consistent, user-friendly CLI feedback with proper styling and formatting.
