Test suite for UI messaging functions that validates error handling and message display functionality without causing panics.

## Key Components

- **TestShowOperationError** - Tests error message display with troubleshooting tips, including edge cases with empty/nil tip arrays
- **TestShowNoResourcesMessage** - Validates "no resources found" message handling with empty string inputs
- **TestShowOperationStart** - Tests operation start messages with custom message maps and fallback scenarios
- **TestShowOperationSuccess** - Tests operation success messages with custom message maps and nil map handling
- **TroubleshootingTip** - Struct used in tests containing Description and Command fields for kubectl operations

## Usage Example

```go
// Run all UI message tests
go test -v ./ui -run TestShow

// Run specific test
go test -v ./ui -run TestShowOperationError

// Test with coverage
go test -cover ./ui -run TestShow
```

The tests focus on panic prevention rather than output validation, ensuring the UI functions remain stable with various input combinations including empty strings, nil values, and missing map keys.
