Test file that validates the robustness of the UI message display functions, ensuring they handle various input conditions without panicking.

## Key Components

- **TestShowOperationError**: Tests error message display with troubleshooting tips, including empty and nil tip slices
- **TestShowNoResourcesMessage**: Tests "no resources found" message display with various string inputs
- **TestShowOperationStart**: Tests operation start message display with custom message maps and fallback scenarios
- **TestShowOperationSuccess**: Tests operation success message display with custom message maps and fallback scenarios

## Usage Example

```go
// Run specific test
go test -run TestShowOperationError

// Run all UI message tests
go test ./ui -v

// Example of what these tests validate:
func ExampleUsage() {
    // These calls should never panic
    tips := []TroubleshootingTip{
        {Description: "Check status", Command: "kubectl get pods"},
    }
    ShowOperationError("delete", "my-cluster", errors.New("failed"), tips)
    
    customMessages := map[string]string{
        "delete": "Deleting cluster...",
    }
    ShowOperationStart("delete", "my-cluster", customMessages)
    ShowOperationSuccess("delete", "my-cluster", customMessages)
    
    ShowNoResourcesMessage("clusters", "delete", "create cmd", "list cmd")
}
```

The tests focus on defensive programming, ensuring the UI functions gracefully handle edge cases like empty inputs, nil parameters, and missing custom message mappings.
