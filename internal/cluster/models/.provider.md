Defines domain interfaces for cluster management operations including provider contracts, business logic services, and provider registry functionality.

## Key Components

- **`ClusterProvider`** - Core interface defining cluster provider contracts with methods for lifecycle management (create, delete, start), status operations, and kubeconfig access
- **`ClusterService`** - Business logic interface representing application use cases, wrapping provider operations with cluster type awareness
- **`ProviderRegistry`** - Registry interface for managing multiple cluster providers, enabling provider registration and retrieval by cluster type

## Usage Example

```go
// Register a provider
var registry ProviderRegistry
var k8sProvider ClusterProvider
registry.RegisterProvider(ClusterTypeKubernetes, k8sProvider)

// Use cluster service for operations
var service ClusterService
config := ClusterConfig{Name: "my-cluster", Type: ClusterTypeKubernetes}

// Create and manage cluster
err := service.CreateCluster(ctx, config)
if err != nil {
    return err
}

// Get cluster status
status, err := service.GetClusterStatus(ctx, "my-cluster")
if err != nil {
    return err
}

// List all clusters
clusters, err := service.ListClusters(ctx)
```

This follows a clean architecture pattern where `ClusterProvider` defines the domain contract, `ClusterService` encapsulates business logic, and `ProviderRegistry` enables multi-provider support through dependency injection.
