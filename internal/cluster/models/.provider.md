Defines the core domain interfaces for cluster management operations, establishing contracts for providers, services, and registry components.

## Key Components

- **`ClusterProvider`** - Primary interface defining cluster lifecycle operations (create, delete, start, list, status)
- **`ClusterService`** - Business logic interface that orchestrates cluster operations across providers
- **`ProviderRegistry`** - Registry interface for managing multiple cluster provider implementations

```go
// Using the service interface
func ExampleUsage(service ClusterService) error {
    ctx := context.Background()
    
    // Create a new cluster
    config := ClusterConfig{Name: "dev-cluster", Type: Docker}
    if err := service.CreateCluster(ctx, config); err != nil {
        return err
    }
    
    // List all clusters
    clusters, err := service.ListClusters(ctx)
    if err != nil {
        return err
    }
    
    return nil
}
```

These interfaces follow clean architecture principles, separating domain contracts from implementation details and enabling multiple cluster provider backends.
