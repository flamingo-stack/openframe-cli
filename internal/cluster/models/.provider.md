This file defines the core interfaces for cluster management operations in OpenFrame, establishing contracts between different layers of the cluster orchestration system.

## Key Components

**ClusterProvider Interface**
- `Create()` - Creates new clusters with specified configuration
- `Delete()` - Removes clusters with optional force deletion
- `Start()` - Starts stopped cluster instances
- `List()` - Retrieves all managed clusters
- `Status()` - Gets detailed cluster status information
- `DetectType()` - Identifies cluster type for existing clusters
- `GetKubeconfig()` - Returns kubeconfig for cluster access

**ClusterService Interface**
- `CreateCluster()` - Business logic for cluster creation, returns `*rest.Config`
- `DeleteCluster()` - Handles cluster deletion with type-specific logic
- `StartCluster()` - Manages cluster startup operations
- `ListClusters()` - Aggregates clusters from all providers
- `GetClusterStatus()` - Retrieves comprehensive cluster information
- `DetectClusterType()` - Determines cluster type for operations
- `GetRestConfig()` - Provides Kubernetes client configuration

**ProviderRegistry Interface**
- `RegisterProvider()` - Registers cluster providers by type
- `GetProvider()` - Retrieves specific provider implementations
- `GetAllProviders()` - Returns complete provider mapping

## Usage Example

```go
// Register a Kind cluster provider
registry.RegisterProvider(KindCluster, kindProvider)

// Create a new cluster
config := ClusterConfig{
    Name: "dev-cluster",
    Version: "1.28.0",
}

restConfig, err := clusterService.CreateCluster(ctx, config)
if err != nil {
    return fmt.Errorf("failed to create cluster: %w", err)
}

// List all clusters
clusters, err := clusterService.ListClusters(ctx)
if err != nil {
    return fmt.Errorf("failed to list clusters: %w", err)
}
```

These interfaces enable OpenFrame to support multiple cluster types (Kind, k3s, etc.) through a unified API while maintaining clean separation between domain logic and provider implementations.
