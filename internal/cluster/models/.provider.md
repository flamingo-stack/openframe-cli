Defines core interfaces for cluster management operations, establishing contracts between domain logic and infrastructure providers.

## Key Components

- **`ClusterProvider`** - Core interface for cluster provider implementations with CRUD operations, status management, and kubeconfig access
- **`ClusterService`** - Business logic interface that orchestrates cluster operations using registered providers
- **`ProviderRegistry`** - Registry pattern interface for managing multiple cluster provider implementations by type

## Usage Example

```go
// Implement a custom cluster provider
type MyClusterProvider struct {
    // provider-specific fields
}

func (p *MyClusterProvider) Create(ctx context.Context, config ClusterConfig) error {
    // Implementation for creating clusters
    return nil
}

func (p *MyClusterProvider) Delete(ctx context.Context, name string, force bool) error {
    // Implementation for deleting clusters
    return nil
}

// Register the provider
registry := NewProviderRegistry()
registry.RegisterProvider(ClusterTypeK8s, &MyClusterProvider{})

// Use through the service layer
service := NewClusterService(registry)
err := service.CreateCluster(ctx, ClusterConfig{
    Name: "my-cluster",
    Type: ClusterTypeK8s,
})
```

This follows clean architecture principles by separating domain interfaces (`ClusterProvider`) from application services (`ClusterService`) and infrastructure concerns (`ProviderRegistry`).
