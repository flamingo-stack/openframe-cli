This file defines core interfaces for the OpenFrame cluster management system, establishing contracts for cluster providers, business logic services, and provider registration.

## Key Components

- **`ClusterProvider`** - Domain interface defining cluster lifecycle operations (create, delete, start, list, status)
- **`ClusterService`** - Business logic interface for application-level cluster operations with Kubernetes integration
- **`ProviderRegistry`** - Registry pattern interface for managing multiple cluster provider implementations

// Registry usage
registry := NewProviderRegistry()

The interfaces follow clean architecture principles, separating domain contracts (`ClusterProvider`) from application use cases (`ClusterService`) and supporting multiple cluster types through the registry pattern.
