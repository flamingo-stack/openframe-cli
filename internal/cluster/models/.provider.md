Defines the core interfaces for cluster management operations, establishing contracts between the domain layer and infrastructure providers.

## Key Components

- **ClusterProvider**: Infrastructure interface defining operations for cluster providers (create, delete, start, list, status, etc.)
- **ClusterService**: Business logic interface representing application use cases for cluster management
- **ProviderRegistry**: Registry pattern interface for managing multiple cluster provider implementations

## Usage Example

```go
// Implementing a cluster service
type MyClusterService struct {
    registry ProviderRegistry
}

func (s *MyClusterService) CreateCluster(ctx context.Context, config ClusterConfig) error {
    provider, err := s.registry.GetProvider(config.Type)
    if err != nil {
        return err
    }
    return provider.Create(ctx, config)
}

// Using the registry
registry.RegisterProvider(KindClusterType, kindProvider)
registry.RegisterProvider(K3sClusterType, k3sProvider)

provider, err := registry.GetProvider(KindClusterType)
if err != nil {
    return err
}

err = provider.Create(ctx, clusterConfig)
```

This follows the dependency inversion principle, allowing different cluster providers (Kind, K3s, etc.) to be plugged in while maintaining consistent business logic.
