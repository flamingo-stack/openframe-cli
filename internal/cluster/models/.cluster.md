This file defines data models for Kubernetes cluster management, providing structures for different cluster types (K3d and GKE) and their configuration options.

## Key Components

- **ClusterType**: String enum defining supported cluster types (k3d, gke)
- **ClusterConfig**: Configuration structure for creating new clusters with basic settings
- **ClusterInfo**: Comprehensive cluster information including status, nodes, and metadata
- **NodeInfo**: Individual node details within a cluster
- **ProviderOptions**: Container for provider-specific configuration options
- **K3dOptions**: K3d-specific settings like port mappings
- **GKEOptions**: GKE-specific settings including zone and project information

## Usage Example

```go
// Create a new K3d cluster configuration
config := &ClusterConfig{
    Name:       "dev-cluster",
    Type:       ClusterTypeK3d,
    NodeCount:  3,
    K8sVersion: "v1.28.0",
}

// Set up K3d-specific options
options := &ProviderOptions{
    K3d: &K3dOptions{
        PortMappings: []string{"8080:80@loadbalancer"},
    },
    Verbose: true,
}

// Example cluster information response
info := &ClusterInfo{
    Name:       "dev-cluster",
    Type:       ClusterTypeK3d,
    Status:     "running",
    NodeCount:  3,
    K8sVersion: "v1.28.0",
    CreatedAt:  time.Now(),
    Nodes: []NodeInfo{
        {Name: "k3d-dev-cluster-server-0", Status: "Ready", Role: "control-plane"},
        {Name: "k3d-dev-cluster-agent-0", Status: "Ready", Role: "worker"},
        {Name: "k3d-dev-cluster-agent-1", Status: "Ready", Role: "worker"},
    },
}
```
