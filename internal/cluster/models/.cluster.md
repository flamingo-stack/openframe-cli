This file defines data models and types for managing Kubernetes cluster configurations and metadata across different provider types like k3d and GKE.

## Key Components

- **ClusterType**: Enum constants for supported cluster types (`k3d`, `gke`)
- **ClusterConfig**: Configuration structure for creating new clusters with name, type, node count, and Kubernetes version
- **ClusterInfo**: Complete cluster information including status, creation time, and node details
- **NodeInfo**: Individual node metadata with name, status, and role
- **ProviderOptions**: Provider-specific configuration container with k3d and GKE options
- **K3dOptions**: k3d-specific settings like port mappings
- **GKEOptions**: GKE-specific settings including zone and project

## Usage Example

```go
// Create a k3d cluster configuration
config := ClusterConfig{
    Name:       "dev-cluster",
    Type:       ClusterTypeK3d,
    NodeCount:  3,
    K8sVersion: "v1.28.0",
}

// Set up provider options with k3d port mappings
options := ProviderOptions{
    K3d: &K3dOptions{
        PortMappings: []string{"8080:80@loadbalancer"},
    },
    Verbose: true,
}

// Create cluster info after deployment
info := ClusterInfo{
    Name:      "dev-cluster",
    Type:      ClusterTypeK3d,
    Status:    "running",
    NodeCount: 3,
    CreatedAt: time.Now(),
    Nodes: []NodeInfo{
        {Name: "k3d-dev-cluster-server-0", Status: "Ready", Role: "control-plane"},
        {Name: "k3d-dev-cluster-agent-0", Status: "Ready", Role: "worker"},
    },
}
```
