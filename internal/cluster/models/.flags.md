This file defines command-line flag structures and validation functions for the OpenFrame CLI tool's cluster management commands.

## Key Components

### Flag Structures
- **`GlobalFlags`** - Alias for common flags shared across all commands
- **`CreateFlags`** - Flags for cluster creation (type, node count, K8s version, skip wizard)
- **`ListFlags`** - Flags for listing clusters (quiet mode)
- **`StatusFlags`** - Flags for cluster status (detailed view, skip apps)
- **`DeleteFlags`** - Flags for cluster deletion (force mode)
- **`CleanupFlags`** - Flags for cleanup operations (force mode)

### Setup Functions
- **`AddGlobalFlags()`** - Adds common flags to any command
- **`Add*Flags()`** - Command-specific flag registration functions

### Validation Functions
- **`ValidateClusterName()`** - Validates cluster names using DNS-1123 subdomain rules
- **`Validate*Flags()`** - Validates flag combinations for each command type

## Usage Example

```go
// Create command with flags
var createFlags models.CreateFlags
cmd := &cobra.Command{
    Use: "create",
    Run: func(cmd *cobra.Command, args []string) {
        if err := models.ValidateCreateFlags(&createFlags); err != nil {
            return err
        }
        // Use validated flags...
    },
}

// Add flags to command
models.AddGlobalFlags(cmd, &createFlags.GlobalFlags)
models.AddCreateFlags(cmd, &createFlags)

// Validate cluster name
if err := models.ValidateClusterName("my-cluster-123"); err != nil {
    log.Fatal(err)
}
```

The validation ensures cluster names are 1-63 characters, alphanumeric with hyphens, and follow Kubernetes naming conventions.
