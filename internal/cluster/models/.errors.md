This file defines domain-specific error types for cluster operations in the models package, providing structured error handling with contextual information for different failure scenarios.

## Key Components

- **ErrClusterNotFound** - Error for when a requested cluster doesn't exist
- **ErrProviderNotFound** - Error for missing providers for specific cluster types
- **ErrInvalidClusterConfig** - Error for configuration validation failures
- **ErrClusterAlreadyExists** - Error for duplicate cluster creation attempts
- **ErrClusterOperation** - General error for cluster operation failures with error wrapping
- **Constructor functions** - Factory functions for creating each error type with proper initialization

## Usage Example

```go
// Check for specific error types
cluster, err := clusterService.GetCluster("my-cluster")
if err != nil {
    var notFound ErrClusterNotFound
    if errors.As(err, &notFound) {
        log.Printf("Cluster %s was not found", notFound.Name)
        return nil, err
    }
}

// Create errors with context
if cluster.Name == "" {
    return NewInvalidConfigError("name", cluster.Name, "name cannot be empty")
}

// Wrap underlying errors
if err := provider.CreateCluster(config); err != nil {
    return NewClusterOperationError("create", config.Name, err)
}
```

The error types implement the standard `error` interface and provide structured access to error details, making them ideal for API responses and logging with proper error categorization.
