Defines domain-specific error types for cluster operations, providing structured error handling with contextual information for different failure scenarios.

## Key Components

**Error Types:**
- `ErrClusterNotFound` - Cluster lookup failures
- `ErrProviderNotFound` - Missing cluster type providers
- `ErrInvalidClusterConfig` - Configuration validation errors
- `ErrClusterAlreadyExists` - Duplicate cluster creation attempts
- `ErrClusterOperation` - General operation failures with error wrapping

**Constructor Functions:**
- `NewClusterNotFoundError()` - Creates cluster not found errors
- `NewProviderNotFoundError()` - Creates provider missing errors
- `NewInvalidConfigError()` - Creates validation errors with field details
- `NewClusterAlreadyExistsError()` - Creates duplicate cluster errors
- `NewClusterOperationError()` - Creates operation errors with cause wrapping

## Usage Example

```go
// Check for specific error types
err := clusterService.GetCluster("my-cluster")
if err != nil {
    var notFoundErr ErrClusterNotFound
    if errors.As(err, &notFoundErr) {
        log.Printf("Cluster %s does not exist", notFoundErr.Name)
        return nil
    }
}

// Create structured errors
if cluster.Name == "" {
    return NewInvalidConfigError("name", "", "cluster name is required")
}

// Wrap operation errors
if err := provider.Deploy(cluster); err != nil {
    return NewClusterOperationError("deploy", cluster.Name, err)
}
```

The error types implement the standard `error` interface and provide structured context for debugging and user feedback. `ErrClusterOperation` supports error unwrapping for examining underlying causes.
