Domain-specific error types for cluster operations, providing structured error handling with contextual information for different failure scenarios.

## Key Components

### Error Types
- **`ErrClusterNotFound`** - Cluster lookup failures with cluster name
- **`ErrProviderNotFound`** - Missing provider for specific cluster types
- **`ErrInvalidClusterConfig`** - Configuration validation errors with field details
- **`ErrClusterAlreadyExists`** - Duplicate cluster creation attempts
- **`ErrClusterOperation`** - General operation failures with error wrapping

### Constructor Functions
- **`NewClusterNotFoundError()`** - Creates cluster not found errors
- **`NewProviderNotFoundError()`** - Creates provider not found errors  
- **`NewInvalidConfigError()`** - Creates configuration validation errors
- **`NewClusterAlreadyExistsError()`** - Creates duplicate cluster errors
- **`NewClusterOperationError()`** - Creates wrapped operation errors

## Usage Example

```go
// Handle cluster lookup
cluster, err := service.GetCluster("my-cluster")
if err != nil {
    var clusterNotFound ErrClusterNotFound
    if errors.As(err, &clusterNotFound) {
        log.Printf("Cluster %s does not exist", clusterNotFound.Name)
        return nil
    }
}

// Create configuration error
if config.Memory < 1 {
    return NewInvalidConfigError("memory", config.Memory, "must be at least 1GB")
}

// Wrap operation errors
if err := provider.CreateCluster(config); err != nil {
    return NewClusterOperationError("create", config.Name, err)
}
```

These structured errors enable type-safe error handling and provide rich context for debugging cluster management operations.
