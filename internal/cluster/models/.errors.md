Defines domain-specific error types for cluster operations, providing structured error handling with contextual information for business-level failures.

## Key Components

**Error Types:**
- `ErrClusterNotFound` - Cluster not found by name
- `ErrProviderNotFound` - No provider available for cluster type
- `ErrInvalidClusterConfig` - Invalid cluster configuration with field details
- `ErrClusterAlreadyExists` - Duplicate cluster name conflict
- `ErrClusterOperation` - General operation failure with cause wrapping

**Constructor Functions:**
- `NewClusterNotFoundError()` - Creates cluster not found error
- `NewProviderNotFoundError()` - Creates provider not found error
- `NewInvalidConfigError()` - Creates configuration validation error
- `NewClusterAlreadyExistsError()` - Creates duplicate cluster error
- `NewClusterOperationError()` - Creates operation failure error

## Usage Example

```go
import "errors"

// Creating domain errors
err1 := NewClusterNotFoundError("my-cluster")
err2 := NewInvalidConfigError("replicas", -1, "must be positive")
err3 := NewClusterOperationError("create", "my-cluster", errors.New("network timeout"))

// Error handling with type assertions
if clusterErr, ok := err.(ErrClusterNotFound); ok {
    fmt.Printf("Missing cluster: %s", clusterErr.Name)
}

// Using error wrapping
if errors.Is(err, ErrClusterOperation{}) {
    cause := errors.Unwrap(err)
    // Handle underlying cause
}
```
