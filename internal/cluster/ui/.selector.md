Provides a centralized cluster selection interface with support for both command-line arguments and interactive prompts, ensuring consistent cluster selection logic across different CLI operations.

## Key Components

- **Selector** - Main struct that manages cluster selection operations with operation-specific context
- **SelectCluster()** - Handles single cluster selection with argument validation and interactive fallback
- **SelectMultipleClusters()** - Manages multi-cluster selection with batch validation
- **ValidateClusterExists()** - Checks cluster existence in the available cluster list
- **GetClusterByName()** - Retrieves specific cluster information by name
- **FilterClusters()** - Applies predicate-based filtering to cluster collections

## Usage Example

```go
// Create a selector for a specific operation
selector := NewSelector("deployment")

// Single cluster selection with argument fallback
clusterName, err := selector.SelectCluster(clusters, args)
if err != nil {
    return err
}

// Multi-cluster selection
selectedClusters, err := selector.SelectMultipleClusters(clusters, args)
if err != nil {
    return err
}

// Validate and retrieve cluster info
if selector.ValidateClusterExists(clusters, "prod") {
    cluster, err := selector.GetClusterByName(clusters, "prod")
    if err != nil {
        return err
    }
}

// Filter clusters by status
activeClusters := selector.FilterClusters(clusters, func(c models.ClusterInfo) bool {
    return c.Status == "active"
})
```
