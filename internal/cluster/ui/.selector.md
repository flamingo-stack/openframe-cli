Provides a unified interface for cluster selection operations, supporting both interactive and argument-based selection modes with validation.

## Key Components

- **Selector**: Main struct that handles cluster selection logic for different operations
- **SelectCluster()**: Selects a single cluster using arguments or interactive prompt
- **SelectMultipleClusters()**: Enables multi-cluster selection with validation
- **ValidateClusterExists()**: Checks if a named cluster exists in the available clusters
- **GetClusterByName()**: Retrieves cluster information by name
- **FilterClusters()**: Filters clusters based on custom predicates

## Usage Example

```go
// Create selector for a specific operation
selector := NewSelector("deployment")

// Single cluster selection with argument
clusters := []models.ClusterInfo{
    {Name: "prod-cluster"},
    {Name: "dev-cluster"},
}
selectedCluster, err := selector.SelectCluster(clusters, []string{"prod-cluster"})
if err != nil {
    // Handle error
}

// Interactive multi-cluster selection
selectedClusters, err := selector.SelectMultipleClusters(clusters, []string{})
if err != nil {
    // Handle error
}

// Validate cluster exists
exists := selector.ValidateClusterExists(clusters, "prod-cluster")

// Filter clusters by status
activeClusters := selector.FilterClusters(clusters, func(c models.ClusterInfo) bool {
    return c.Status == "active"
})
```

The selector provides consistent user experience across different CLI commands by handling both programmatic (via arguments) and interactive cluster selection scenarios.
