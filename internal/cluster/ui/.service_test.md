This file contains comprehensive unit tests for the UI display service functionality in a Kubernetes cluster management tool. The tests verify display output formatting and message generation for various cluster operations.

## Key Components

- **TestNewDisplayService** - Tests DisplayService instantiation
- **ShowCluster* Tests** - Validates cluster operation messages (creation, deletion, start, status)
- **ShowClusterList Tests** - Verifies cluster listing functionality with table formatting
- **ShowConfigurationSummary Tests** - Tests configuration display with dry-run and wizard modes
- **Display Info Structs Tests** - Validates ClusterDisplayInfo and NodeDisplayInfo data structures

## Usage Example

```go
func TestDisplayService_ShowClusterStatus(t *testing.T) {
    service := NewDisplayService()
    var buf bytes.Buffer
    
    status := &ClusterDisplayInfo{
        Name:      "test-cluster",
        Type:      "k3d",
        Status:    "running",
        NodeCount: 3,
        CreatedAt: time.Date(2023, 1, 1, 12, 0, 0, 0, time.UTC),
        Nodes: []NodeDisplayInfo{
            {Name: "node1", Role: "control-plane", Status: "ready"},
        },
    }
    
    service.ShowClusterStatus(status, &buf)
    
    output := buf.String()
    assert.Contains(t, output, "test-cluster")
    assert.Contains(t, output, "running")
}
```

The tests use `bytes.Buffer` for output capture and `testify/assert` for assertions, ensuring proper message formatting and content validation across all display methods.
