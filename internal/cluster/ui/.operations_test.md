Test file for the OperationsUI component that validates cluster operation user interface functionality. Contains comprehensive test coverage for cluster selection, operation messaging, and error handling scenarios.

## Key Components

- **TestOperationsUI_SelectClusterForOperation**: Tests cluster selection logic with various input scenarios (provided args, empty clusters, whitespace handling)
- **TestNewOperationsUI**: Validates OperationsUI constructor
- **TestOperationsUI_ShowOperationStart**: Tests operation start message display
- **TestOperationsUI_ShowOperationSuccess**: Tests success message display 
- **TestOperationsUI_ShowOperationError**: Tests error message display with error objects
- **TestOperationsUI_ShowNoResourcesMessage**: Tests no resources available messaging

## Usage Example

```go
func TestClusterSelection(t *testing.T) {
    ui := NewOperationsUI()
    clusters := []models.ClusterInfo{
        {Name: "my-cluster", Type: models.ClusterTypeK3d},
    }
    args := []string{"my-cluster"}
    
    result, err := ui.SelectClusterForOperation(clusters, args, "cleanup")
    if err != nil {
        t.Errorf("unexpected error: %v", err)
    }
    if result != "my-cluster" {
        t.Errorf("expected 'my-cluster', got %s", result)
    }
}
```

The tests primarily focus on ensuring UI methods don't panic and handle edge cases like empty inputs, whitespace, and missing resources appropriately.
