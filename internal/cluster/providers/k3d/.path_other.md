A platform-specific utility function that handles path expansion on non-Windows systems by implementing a no-op version of `expandShortPath`.

## Key Components

- **`expandShortPath`** - Returns the input path unchanged on non-Windows platforms, as short filename expansion is only needed on Windows systems

## Usage Example

```go
// On Linux, macOS, and other non-Windows platforms
path := "/home/user/my-project"
expandedPath, err := expandShortPath(path)
if err != nil {
    log.Fatal(err)
}
// expandedPath == "/home/user/my-project" (unchanged)
```

This file uses Go's build constraint `//go:build !windows` to ensure it's only compiled on non-Windows platforms, providing a consistent API across different operating systems while avoiding unnecessary processing where Windows-specific short filename handling isn't required.
