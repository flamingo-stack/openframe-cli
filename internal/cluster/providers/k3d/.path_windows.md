Windows-specific path utilities for the k3d package that handles Windows 8.3 short filename expansion using the Windows API, particularly useful for WSL compatibility.

## Key Components

- **expandShortPath()** - Core function that converts Windows short filenames (8.3 format) to their full long path names
- **kernel32** - Lazy-loaded Windows kernel32.dll library handle
- **procGetLongPathNameW** - Windows API function pointer for GetLongPathNameW

## Usage Example

```go
// Expand a Windows short path to its full form
shortPath := "C:\\Users\\RUNNER~1\\AppData\\Local"
longPath, err := expandShortPath(shortPath)
if err != nil {
    log.Printf("Failed to expand path: %v", err)
    return
}
// Result: "C:\\Users\\runneradmin\\AppData\\Local"
fmt.Printf("Expanded: %s -> %s\n", shortPath, longPath)

// Handle empty or invalid paths gracefully
emptyPath, _ := expandShortPath("")  // Returns empty string
invalidPath, _ := expandShortPath("Z:\\nonexistent")  // Returns original path
```

The function uses Windows' `GetLongPathNameW` API to resolve 8.3 short filenames that are commonly generated by Windows in user directories and temporary paths. This is essential for WSL interoperability since WSL doesn't understand Windows short filename format. The implementation includes proper error handling and fallback behavior, returning the original path if expansion fails.
