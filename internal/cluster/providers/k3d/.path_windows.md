This Windows-specific file handles expansion of Windows 8.3 short filenames to their full long path names, which is essential for WSL compatibility in the k3d package.

## Key Components

- **expandShortPath()** - Main function that converts Windows short paths (e.g., `RUNNER~1`) to full long paths
- **kernel32 DLL** - Lazy-loaded Windows system library for path operations
- **procGetLongPathNameW** - Windows API procedure for path expansion
- **Build constraint** - `//go:build windows` ensures this code only compiles on Windows

## Usage Example

```go
// Expand a Windows short path to its full form
shortPath := "C:\\Users\\RUNNER~1\\AppData\\Local\\Temp"
longPath, err := expandShortPath(shortPath)
if err != nil {
    log.Printf("Failed to expand path: %v", err)
    return
}
// Result: "C:\\Users\\runneradmin\\AppData\\Local\\Temp"

// Function safely handles empty paths
emptyPath, _ := expandShortPath("")
// Returns: ""

// If path expansion fails, original path is returned as fallback
invalidPath, _ := expandShortPath("C:\\NonExistent\\Path")
// Returns: "C:\\NonExistent\\Path" (original)
```

The function uses the Windows `GetLongPathNameW` API with a two-step process: first determining the required buffer size, then retrieving the expanded path. This ensures WSL can properly interpret Windows file paths that contain short filename components.
