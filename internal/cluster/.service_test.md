Test file for the cluster service package that provides comprehensive unit testing for ClusterService functionality.

## Key Components

- **`createTestExecutor()`** - Creates a mock executor with predefined responses for k3d cluster commands
- **Constructor Tests** - Tests for `NewClusterService()` and `NewClusterServiceWithOptions()`
- **Service Method Tests** - Tests for all ClusterService methods including:
  - `CreateCluster()` and `DeleteCluster()`
  - `ListClusters()` and `GetClusterStatus()`
  - `DetectClusterType()` and `CleanupCluster()`
  - `ShowClusterStatus()` and `DisplayClusterList()`
- **Real Executor Test** - Tests service initialization with actual executor in dry-run mode

## Usage Example

```go
// Run all tests
go test ./internal/cluster

// Run specific test
go test -run TestNewClusterService ./internal/cluster

// Test with verbose output
go test -v ./internal/cluster

// Example of how mock executor is configured
mock := executor.NewMockCommandExecutor()
mockJSON := `[{"name":"test-cluster","serversCount":1,...}]`
mock.SetResponse("k3d cluster list", &executor.CommandResult{
    ExitCode: 0,
    Stdout:   mockJSON,
    Duration: 100,
})
```

The tests focus on ensuring methods don't panic and handle mock data appropriately, with error tolerance for incomplete mock responses.
