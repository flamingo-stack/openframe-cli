This file provides automated installation capabilities for OpenFrame's prerequisite tools (Docker, kubectl, k3d, and Helm) with interactive and non-interactive modes.

## Key Components

- **Installer**: Main struct that orchestrates prerequisite installation with progress tracking and user interaction
- **NewInstaller()**: Creates a new installer instance with an embedded prerequisite checker
- **InstallMissingPrerequisites()**: Automatically detects and installs all missing tools with spinner progress indicators
- **CheckAndInstall()**: Interactive mode that prompts user for confirmation before installing missing tools
- **CheckAndInstallNonInteractive()**: Non-interactive mode for automated environments, handles Docker startup separately from installation
- **installTool()**: Routes installation to specific tool installers (docker, kubectl, k3d, helm packages)
- **showManualInstructions()**: Displays platform-specific manual installation instructions in a formatted table

## Usage Example

```go
package main

import (
    "github.com/flamingo-stack/openframe-cli/internal/cluster/prerequisites"
)

func main() {
    installer := prerequisites.NewInstaller()
    
    // Interactive installation with user prompts
    if err := installer.CheckAndInstall(); err != nil {
        log.Fatal(err)
    }
    
    // Non-interactive installation for CI/CD
    if err := installer.CheckAndInstallNonInteractive(true); err != nil {
        log.Fatal(err)
    }
    
    // Install only missing prerequisites
    if err := installer.InstallMissingPrerequisites(); err != nil {
        log.Fatal(err)
    }
}
```

The installer provides comprehensive error handling, platform-specific instructions, and graceful fallbacks when automatic installation fails, making it suitable for both developer workstations and automated deployment pipelines.
