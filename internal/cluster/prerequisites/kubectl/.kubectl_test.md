This file contains comprehensive unit tests for the kubectl installer functionality, validating installer creation, platform-specific installation help, and installation behavior across different operating systems.

## Key Components

- **TestNewKubectlInstaller**: Tests the creation of a new kubectl installer instance
- **TestKubectlInstaller_GetInstallHelp**: Validates that installation help text is provided and contains appropriate platform-specific guidance (brew for macOS, package manager for Linux, HTTPS for Windows)
- **TestKubectlInstaller_Install**: Tests installation behavior and error handling for different platforms and scenarios
- **TestCommandExists**: Validates the command existence detection utility
- **containsSubstring**: Helper function for string matching in tests

## Usage Example

```go
func TestYourKubectlFeature(t *testing.T) {
    // Test installer creation
    installer := NewKubectlInstaller()
    if installer == nil {
        t.Error("Expected kubectl installer to be created")
    }
    
    // Test platform-specific help
    help := installer.GetInstallHelp()
    if help == "" {
        t.Error("Install help should not be empty")
    }
    
    // Test command existence
    if !commandExists("echo") {
        t.Error("Expected 'echo' command to exist")
    }
}
```

The tests cover platform-specific behaviors for macOS (Homebrew), Linux (package managers), and Windows (manual installation), ensuring proper error handling when dependencies are missing or platforms are unsupported.
