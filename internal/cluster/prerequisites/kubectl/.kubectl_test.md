Test suite for validating kubectl installer functionality across different operating systems. Tests the installer creation, help text generation, installation behavior, and command existence checking.

## Key Components

- **TestNewKubectlInstaller**: Validates that a kubectl installer instance can be created successfully
- **TestKubectlInstaller_GetInstallHelp**: Tests platform-specific installation help text generation for macOS, Linux, and Windows
- **TestKubectlInstaller_Install**: Tests installation behavior and error handling for different platforms and scenarios
- **TestCommandExists**: Validates the command existence checking utility function
- **containsSubstring**: Helper function for substring matching in test assertions

## Usage Example

```go
func TestCustomInstaller(t *testing.T) {
    // Create installer and validate creation
    installer := NewKubectlInstaller()
    if installer == nil {
        t.Error("Failed to create installer")
    }
    
    // Test help text contains expected content
    help := installer.GetInstallHelp()
    if !containsSubstring(help, "kubectl") {
        t.Error("Help should mention kubectl")
    }
    
    // Test command existence checking
    if !commandExists("go") {
        t.Error("Go command should exist in test environment")
    }
}
```

The tests cover platform-specific behaviors, error conditions, and edge cases without performing actual installations, making them suitable for CI environments.
