Test file for the prerequisites checker that validates prerequisite management functionality for cluster deployment tools.

## Key Components

- **TestNewPrerequisiteChecker**: Validates that the prerequisite checker initializes with correct requirements (Docker, kubectl, k3d)
- **TestCommandExists**: Tests command existence validation through installer interfaces
- **TestInstallHelp**: Verifies platform-specific installation help text contains appropriate references (brew for macOS, package managers for Linux, chocolatey for Windows)
- **TestCheckAllWithMissingTools/TestCheckAllWithAllTools**: Tests the complete prerequisite checking workflow
- **TestGetInstallInstructions**: Validates installation instruction retrieval for missing tools
- **containsAny**: Helper function for substring matching in help text validation

## Usage Example

```go
// Run tests to verify prerequisite checker functionality
go test ./internal/cluster/prerequisites/

// Test specific functionality
go test -run TestNewPrerequisiteChecker ./internal/cluster/prerequisites/

// Test with verbose output to see detailed validation
go test -v ./internal/cluster/prerequisites/
```

The tests mock prerequisite states to verify both successful and failure scenarios, ensuring the checker properly identifies missing tools and provides appropriate installation guidance across different operating systems.
