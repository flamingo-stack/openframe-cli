This file contains comprehensive unit tests for the prerequisites checking system that validates whether required development tools (Docker, kubectl, k3d) are installed on the system.

## Key Components

- **TestNewPrerequisiteChecker** - Verifies the prerequisite checker is properly initialized with three requirements
- **TestCommandExists** - Tests the command existence detection functionality through Docker installer
- **TestInstallHelp** - Validates platform-specific installation help messages for all three tools
- **TestCheckAllWithMissingTools/TestCheckAllWithAllTools** - Tests the bulk checking functionality with different scenarios
- **TestGetInstallInstructions** - Verifies installation instruction retrieval for missing tools
- **containsAny** - Helper function to check if a string contains any of the provided substrings

## Usage Example

```go
func TestCustomPrerequisites(t *testing.T) {
    checker := NewPrerequisiteChecker()
    
    // Test with all tools present
    for i := range checker.requirements {
        checker.requirements[i].IsInstalled = func() bool { return true }
    }
    
    allPresent, missing := checker.CheckAll()
    if !allPresent {
        t.Error("Expected all tools to be present")
    }
    
    // Get installation instructions for missing tools
    instructions := checker.GetInstallInstructions([]string{"Docker"})
    if len(instructions) == 0 {
        t.Error("Expected installation instructions")
    }
}
```

The tests ensure cross-platform compatibility by validating that installation help contains appropriate package managers or download URLs for macOS, Linux, and Windows.
