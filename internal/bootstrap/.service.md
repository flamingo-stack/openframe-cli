Orchestrates the complete OpenFrame bootstrap process by combining cluster creation with chart installation into a single command workflow.

## Key Components

**Service**: Main bootstrap service that coordinates cluster setup and chart deployment
- `NewService()`: Creates a new bootstrap service instance
- `Execute()`: Handles command execution with flag parsing and validation
- `bootstrap()`: Core orchestration method that sequences cluster creation and chart installation
- `createClusterSuppressed()`: Creates cluster with minimal UI output
- `installChartWithMode()`: Installs charts with specified deployment mode
- `initializeWSL()`: Windows-specific WSL2 initialization (runs on Windows only)

**Configuration**:
- Supports deployment modes: `oss-tenant`, `saas-tenant`, `saas-shared`
- Non-interactive mode for automated deployments
- Verbose output control
- Windows WSL2 auto-configuration

## Usage Example

```go
// Create and execute bootstrap service
service := NewService()

// Execute with cobra command (typically called from CLI)
err := service.Execute(cmd, []string{"my-cluster"})
if err != nil {
    log.Fatal(err)
}

// Direct bootstrap execution
err = service.bootstrap(
    "openframe-dev",    // cluster name
    "oss-tenant",       // deployment mode
    false,              // non-interactive
    true,               // verbose
)
```

The service automatically handles Windows WSL2 setup, creates a K3d cluster with the specified name (defaults to "openframe-dev"), and installs the appropriate OpenFrame charts based on the deployment mode.
