This service orchestrates the bootstrap process for OpenFrame clusters by combining cluster creation with chart installation into a single streamlined operation.

## Key Components

- **Service**: Main bootstrap service struct that coordinates cluster and chart operations
- **NewService()**: Factory function to create a new bootstrap service instance
- **Execute()**: Primary command handler that validates flags and delegates to bootstrap logic
- **bootstrap()**: Core orchestration method that sequentially creates cluster and installs charts
- **createClusterSuppressed()**: Creates a k3d cluster with minimal UI output for automation
- **buildClusterConfig()**: Generates default cluster configuration with sensible defaults
- **installChartWithMode()**: Installs OpenFrame charts with specified deployment mode

## Usage Example

```go
// Create and execute bootstrap service
service := NewService()

// Execute with cobra command (handles flag parsing internally)
err := service.Execute(cmd, []string{"my-cluster"})
if err != nil {
    log.Fatal(err)
}

// The service will:
// 1. Create a k3d cluster named "my-cluster" (or "openframe-dev" if not specified)
// 2. Install OpenFrame charts with specified deployment mode
// 3. Handle both interactive and non-interactive modes
// 4. Provide verbose output when requested
```

The service validates deployment modes (`oss-tenant`, `saas-tenant`, `saas-shared`) and ensures non-interactive mode has required parameters. It uses the default OpenFrame OSS tenant repository and creates a 3-node k3d cluster with Kubernetes version auto-detection.
