This service orchestrates the complete OpenFrame bootstrap process by combining cluster creation and chart installation into a single command.

## Key Components

- **Service**: Main bootstrap service struct providing orchestration functionality
- **NewService()**: Factory function to create a new bootstrap service instance
- **Execute()**: Primary command handler that processes CLI flags and delegates to bootstrap logic
- **bootstrap()**: Core orchestration method that sequentially creates cluster and installs charts
- **createClusterSuppressed()**: Creates a Kubernetes cluster with minimal UI output
- **buildClusterConfig()**: Builds cluster configuration with sensible defaults
- **installChartWithMode()**: Installs charts with specified deployment mode configuration

## Usage Example

```go
// Create and execute bootstrap service
service := bootstrap.NewService()

// Execute with cobra command (typically called from CLI)
err := service.Execute(cmd, []string{"my-cluster"})
if err != nil {
    log.Fatal(err)
}

// Or use the internal bootstrap method directly
err = service.bootstrap(
    "openframe-dev",     // cluster name
    "oss-tenant",        // deployment mode
    false,               // non-interactive
    true,                // verbose
)
```

The service validates deployment modes (oss-tenant, saas-tenant, saas-shared), handles both interactive and non-interactive modes, and provides consistent error handling throughout the bootstrap process.
