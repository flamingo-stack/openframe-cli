This file defines data models for Helm chart management, providing structures to represent chart information and categorize different chart types within a Kubernetes deployment system.

## Key Components

- **`ChartInfo`** - Struct containing essential metadata about an installed Helm chart including name, namespace, deployment status, and version information
- **`ChartType`** - String-based type alias for categorizing charts
- **Chart type constants:**
  - `ChartTypeArgoCD` - Represents ArgoCD application charts
  - `ChartTypeAppOfApps` - Represents App-of-Apps pattern charts for managing multiple applications

## Usage Example

```go
package main

import (
    "fmt"
    "your-project/models"
)

func main() {
    // Create chart information
    chart := models.ChartInfo{
        Name:       "my-application",
        Namespace:  "production",
        Status:     "deployed",
        Version:    "1.2.3",
        AppVersion: "2.1.0",
    }
    
    // Use chart type constants
    chartType := models.ChartTypeArgoCD
    
    fmt.Printf("Chart: %s in %s namespace (Status: %s)\n", 
        chart.Name, chart.Namespace, chart.Status)
    fmt.Printf("Chart Type: %s\n", string(chartType))
}
```

This model is commonly used in GitOps workflows where ArgoCD manages application deployments and the App-of-Apps pattern orchestrates multiple related applications as a single unit.
