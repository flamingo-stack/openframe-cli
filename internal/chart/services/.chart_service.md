This service orchestrates the complete chart installation process for the OpenFrame CLI, handling interactive and non-interactive deployment modes with support for graceful cancellation and cleanup.

## Key Components

- **ChartService**: Main service struct managing chart operations with Helm, Git, and Kubernetes integrations
- **NewChartService()**: Creates service with immediate HelmManager initialization using provided kubeConfig
- **NewChartServiceDeferred()**: Creates service with deferred HelmManager initialization for standalone installations
- **Install/InstallWithContext()**: Entry points for chart installation with optional context cancellation
- **InstallationWorkflow**: Internal workflow orchestrator handling the multi-step installation process
- **Signal Handling**: Built-in CTRL-C interruption support with graceful cleanup

## Usage Example

```go
// Create service with existing kubeConfig
kubeConfig := &rest.Config{Host: "https://kubernetes.local"}
service, err := NewChartService(kubeConfig, false, true)
if err != nil {
    return fmt.Errorf("failed to create chart service: %w", err)
}

// Install chart with context cancellation
ctx, cancel := context.WithTimeout(context.Background(), 30*time.Minute)
defer cancel()

req := utilTypes.InstallationRequest{
    DeploymentMode:  "production",
    NonInteractive:  true,
    DryRun:         false,
    Verbose:        true,
    Args:           []string{"my-cluster"},
}

err = service.InstallWithContext(ctx, req)
if err != nil {
    return fmt.Errorf("installation failed: %w", err)
}
```

The service supports three installation modes: fully interactive (configuration wizard), partially interactive (pre-selected deployment mode), and fully non-interactive (CI/CD environments). It includes automatic file cleanup, certificate regeneration, and ArgoCD synchronization.
