This service manages Helm chart installation operations for the OpenFrame platform, providing both interactive and non-interactive deployment workflows with cluster selection and configuration management.

## Key Components

- **ChartService**: Main service struct that orchestrates chart operations
- **InstallationWorkflow**: Workflow orchestrator for the complete installation process
- **NewChartService()**: Constructor with immediate HelmManager initialization
- **NewChartServiceDeferred()**: Constructor with deferred HelmManager initialization for cluster selection scenarios
- **Install/InstallWithContext()**: Main installation methods with context support
- **ExecuteWithContext()**: Core workflow execution with signal handling and cleanup

## Usage Example

```go
// Create chart service with existing Kubernetes config
chartService, err := services.NewChartService(kubeConfig, false, true)
if err != nil {
    return fmt.Errorf("failed to create chart service: %w", err)
}

// Create installation request
req := utilTypes.InstallationRequest{
    Args:           []string{"cluster-name"},
    DeploymentMode: "production",
    NonInteractive: false,
    DryRun:        false,
    Verbose:       true,
}

// Execute installation with context
ctx, cancel := context.WithTimeout(context.Background(), 30*time.Minute)
defer cancel()

err = chartService.InstallWithContext(ctx, req)
if err != nil {
    return fmt.Errorf("installation failed: %w", err)
}
```

The service supports three operational modes: fully interactive (default), partial non-interactive (pre-selected deployment mode), and fully non-interactive (CI/CD). It includes comprehensive error handling, signal interruption support, and automatic cleanup of temporary files.
