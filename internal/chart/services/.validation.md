Provides validation logic for Helm chart configurations in non-interactive deployment modes. The validator ensures required configuration values are present based on the selected deployment mode (OSS, SaaS, or SaaS Shared).

## Key Components

- **ConfigurationValidator**: Main struct that performs validation operations
- **NewConfigurationValidator()**: Factory function to create validator instances
- **ValidateConfiguration()**: Primary validation method that routes to mode-specific validators
- **validateOSSConfiguration()**: Validates OSS deployment requirements
- **validateSaaSConfiguration()**: Validates SaaS deployment requirements including credentials
- **validateSaaSSharedConfiguration()**: Validates SaaS Shared deployment requirements
- Helper methods: `isDeploymentEnabled()`, `hasPassword()`, `hasGHCRCredentials()`, `hasBranch()`

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/chart/utils/types"
)

// Create validator
validator := NewConfigurationValidator()

// Prepare configuration
config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeSaaS,
    ExistingValues: map[string]interface{}{
        "deployment": map[string]interface{}{
            "saas": map[string]interface{}{
                "enabled": true,
                "repository": map[string]interface{}{
                    "password": "secret123",
                },
            },
        },
        "registry": map[string]interface{}{
            "ghcr": map[string]interface{}{
                "username": "user",
                "password": "token",
            },
        },
    },
}

// Validate configuration
if err := validator.ValidateConfiguration(config); err != nil {
    log.Fatal(err)
}
```
