Validates Helm values configuration for different deployment modes in the OpenFrame CLI tool. Ensures required settings are present for OSS, SaaS, and SaaS Shared deployments.

## Key Components

- **ConfigurationValidator** - Main validator struct that validates helm-values.yaml configurations
- **ValidateConfiguration()** - Primary validation method that routes to specific deployment mode validators
- **validateOSSConfiguration()** - Validates Open Source deployment requirements
- **validateSaaSConfiguration()** - Validates Software-as-a-Service deployment requirements  
- **validateSaaSSharedConfiguration()** - Validates shared SaaS deployment requirements
- Helper methods for checking deployment enablement, passwords, and registry credentials

## Usage Example

```go
// Create validator
validator := NewConfigurationValidator()

// Validate configuration for non-interactive mode
config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeSaaS,
    ExistingValues: helmValues, // map[string]interface{}
}

err := validator.ValidateConfiguration(config)
if err != nil {
    log.Fatalf("Configuration validation failed: %v", err)
}

// Configuration is valid for the specified deployment mode
```

The validator ensures critical settings like deployment enablement, repository passwords, and registry credentials are properly configured before proceeding with deployments.
