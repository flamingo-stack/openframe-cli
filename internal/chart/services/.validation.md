This file provides validation services for Helm chart configurations in non-interactive deployment modes, ensuring all required settings are present before deployment.

## Key Components

- **ConfigurationValidator** - Main validator struct that validates helm-values.yaml configurations
- **ValidateConfiguration()** - Primary validation method that routes to specific validators based on deployment mode
- **validateOSSConfiguration()** - Validates Open Source Software deployment settings
- **validateSaaSConfiguration()** - Validates Software as a Service deployment settings  
- **validateSaaSSharedConfiguration()** - Validates SaaS Shared deployment settings
- **Helper methods** - Utility functions for checking deployment enablement, passwords, and GHCR credentials

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/chart/services"
    "github.com/flamingo-stack/openframe-cli/internal/chart/utils/types"
)

// Create validator
validator := services.NewConfigurationValidator()

// Prepare configuration
config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeSaaS,
    ExistingValues: map[string]interface{}{
        "deployment": map[string]interface{}{
            "saas": map[string]interface{}{
                "enabled": true,
                "repository": map[string]interface{}{
                    "password": "secret123",
                },
            },
        },
        "registry": map[string]interface{}{
            "ghcr": map[string]interface{}{
                "username": "user",
                "password": "token",
            },
        },
    },
}

// Validate configuration
if err := validator.ValidateConfiguration(config); err != nil {
    log.Fatal("Configuration validation failed:", err)
}
```
