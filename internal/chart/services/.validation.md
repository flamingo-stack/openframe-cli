Provides validation logic for Helm chart configurations in non-interactive deployment modes. The validator ensures required settings are present based on deployment type (OSS, SaaS, or SaaS Shared).

## Key Components

- **ConfigurationValidator**: Main validator struct with validation methods
- **NewConfigurationValidator()**: Constructor function
- **ValidateConfiguration()**: Primary validation method that delegates to mode-specific validators
- **validateOSSConfiguration()**: Validates OSS deployment requirements
- **validateSaaSConfiguration()**: Validates SaaS deployment requirements  
- **validateSaaSSharedConfiguration()**: Validates SaaS Shared deployment requirements
- Helper methods for checking deployment settings, passwords, and GHCR credentials

## Usage Example

```go
// Create validator and validate configuration
validator := NewConfigurationValidator()

config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeSaaS,
    ExistingValues: map[string]interface{}{
        "deployment": map[string]interface{}{
            "saas": map[string]interface{}{
                "enabled": true,
                "repository": map[string]interface{}{
                    "password": "secret123",
                },
            },
        },
        "registry": map[string]interface{}{
            "ghcr": map[string]interface{}{
                "username": "user",
                "password": "token",
            },
        },
    },
}

if err := validator.ValidateConfiguration(config); err != nil {
    log.Fatal(err)
}
```
