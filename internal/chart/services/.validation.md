Validates Helm chart configuration for OpenFrame CLI deployments, ensuring required values are present for non-interactive mode operations.

## Key Components

- **ConfigurationValidator**: Main validator struct for chart configurations
- **ValidateConfiguration**: Entry point that routes validation based on deployment mode
- **validateOSSConfiguration**: Validates Open Source Software deployment settings
- **validateSaaSConfiguration**: Validates Software as a Service deployment requirements
- **validateSaaSSharedConfiguration**: Validates shared SaaS deployment configuration
- **Helper methods**: `isDeploymentEnabled`, `hasPassword`, `hasGHCRCredentials`, etc.

## Usage Example

```go
// Create a new validator
validator := NewConfigurationValidator()

// Validate configuration for a specific deployment mode
config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeSaaS,
    ExistingValues: map[string]interface{}{
        "deployment": map[string]interface{}{
            "saas": map[string]interface{}{
                "enabled": true,
                "repository": map[string]interface{}{
                    "password": "secret123",
                },
            },
        },
        "registry": map[string]interface{}{
            "ghcr": map[string]interface{}{
                "username": "user",
                "password": "token",
            },
        },
    },
}

// Validate the configuration
if err := validator.ValidateConfiguration(config); err != nil {
    log.Fatalf("Configuration validation failed: %v", err)
}
```

The validator ensures deployment-specific requirements are met, such as enabled flags, repository credentials, and registry authentication for GHCR access.
