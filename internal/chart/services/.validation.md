Validates Helm chart configuration for different OpenFrame deployment modes, ensuring required settings are present for non-interactive deployments.

## Key Components

- **ConfigurationValidator** - Main validator struct for helm-values.yaml configuration
- **ValidateConfiguration()** - Primary validation method that routes to deployment-specific validators
- **validateOSSConfiguration()** - Validates Open Source deployment requirements
- **validateSaaSConfiguration()** - Validates SaaS deployment requirements including repository passwords and GHCR credentials
- **validateSaaSSharedConfiguration()** - Validates SaaS Shared deployment requirements
- Helper methods for checking deployment enablement, passwords, branches, and registry credentials

## Usage Example

```go
// Create validator
validator := services.NewConfigurationValidator()

// Validate configuration for a specific deployment mode
config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeSaaS,
    ExistingValues: map[string]interface{}{
        "deployment": map[string]interface{}{
            "saas": map[string]interface{}{
                "enabled": true,
                "repository": map[string]interface{}{
                    "password": "secret123",
                },
            },
        },
        "registry": map[string]interface{}{
            "ghcr": map[string]interface{}{
                "username": "myuser",
                "password": "mytoken",
            },
        },
    },
}

if err := validator.ValidateConfiguration(config); err != nil {
    log.Fatal("Configuration validation failed:", err)
}
```
