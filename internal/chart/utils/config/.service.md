This service provides centralized configuration management for chart operations, combining shared system configuration with chart-specific path resolution and installation configuration building.

## Key Components

- **Service**: Main configuration service struct that wraps shared system config and path resolution
- **NewService()**: Constructor that initializes a new configuration service instance
- **Path Methods**: Certificate directory, Helm values file, and log directory path getters
- **Initialize()**: Sets up necessary directories and initializes the shared system service
- **BuildInstallConfig()**: Creates complete chart installation configurations with defaults
- **GetDefaultManifestsPath()**: Locates manifests directory with fallback logic

## Usage Example

```go
// Create and initialize configuration service
configSvc := config.NewService()
if err := configSvc.Initialize(); err != nil {
    log.Fatal(err)
}

// Get certificate paths
certFile, keyFile := configSvc.GetCertificateFiles()
valuesFile := configSvc.GetHelmValuesFile()

// Build installation configuration
installConfig := configSvc.BuildInstallConfig(
    false, // force
    true,  // dryRun
    true,  // verbose
    "my-cluster",
    &models.AppOfAppsConfig{
        Name: "my-apps",
    },
)

// Get manifests path with fallback
manifestsPath := configSvc.GetDefaultManifestsPath()
```

The service automatically creates required directories during initialization and provides sensible defaults for certificate directories and manifest locations.
