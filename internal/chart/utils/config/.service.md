Configuration service that centralizes chart-specific settings and paths management for Helm operations. It provides a unified interface for accessing certificate directories, Helm values files, and other chart-related configuration paths.

## Key Components

- **`Service`** - Main configuration service struct with system service and path resolver dependencies
- **`NewService()`** - Factory function that creates a new configuration service instance
- **`GetCertificateDirectory()`** - Returns the certificate directory path for TLS operations
- **`GetCertificateFiles()`** - Retrieves paths to certificate and key files
- **`GetHelmValuesFile()`** - Returns the path to Helm values configuration file
- **`Initialize()`** - Performs configuration initialization including directory creation
- **`BuildInstallConfig()`** - Creates complete installation configuration with defaults
- **`GetDefaultManifestsPath()`** - Resolves default manifests path with fallback logic

## Usage Example

```go
// Create and initialize configuration service
configService := config.NewService()
if err := configService.Initialize(); err != nil {
    log.Fatal("Failed to initialize config:", err)
}

// Get certificate paths
certFile, keyFile := configService.GetCertificateFiles()
fmt.Printf("Cert: %s, Key: %s\n", certFile, keyFile)

// Build installation configuration
installConfig := configService.BuildInstallConfig(
    false,     // force
    true,      // dryRun
    false,     // verbose
    "my-cluster",
    &models.AppOfAppsConfig{},
)

// Get manifests path with fallback
manifestsPath := configService.GetDefaultManifestsPath()
```

The service automatically handles directory creation, path resolution, and provides sensible defaults for chart operations while integrating with the shared system configuration.
