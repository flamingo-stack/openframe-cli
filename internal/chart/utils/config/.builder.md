Handles construction and building of OpenFrame installation configurations, integrating with Helm values files to determine appropriate Git branches and deployment settings.

## Key Components

- **Builder**: Main configuration builder that coordinates between config service and UI operations
- **HelmValues**: YAML structure mapping for parsing Helm values files with deployment mode configurations
- **NewBuilder()**: Constructor that initializes a new builder with operations UI integration
- **BuildInstallConfig()**: Constructs basic installation configuration with GitHub repository settings
- **BuildInstallConfigWithCustomHelmPath()**: Advanced builder supporting custom Helm values paths and deployment modes
- **getBranchForDeploymentMode()**: Determines Git branch based on deployment mode (OSS, SaaS Tenant, SaaS Shared)
- **getBranchFromHelmValues()**: Extracts branch information from Helm values files

## Usage Example

```go
// Create builder with UI operations
operationsUI := chartUI.NewOperationsUI()
builder := config.NewBuilder(operationsUI)

// Build basic installation config
config, err := builder.BuildInstallConfig(
    false,              // force
    false,              // dryRun  
    true,               // verbose
    "my-cluster",       // clusterName
    "my-org/my-repo",   // githubRepo
    "main",             // githubBranch
    "/certs",           // certDir
)

// Build with custom deployment mode
config, err := builder.BuildInstallConfigWithCustomHelmPath(
    false,              // force
    false,              // dryRun
    true,               // verbose
    false,              // nonInteractive
    "my-cluster",       // clusterName
    "my-org/my-repo",   // githubRepo
    "main",             // githubBranch
    "/certs",           // certDir
    "custom-values.yaml", // helmValuesPath
    "saas-shared",      // deploymentMode
)
```

The builder automatically reads Helm values files to override Git branches based on deployment configuration, supporting different branch strategies for OSS, SaaS Tenant, and SaaS Shared modes.
