Handles construction of installation configurations for OpenFrame CLI, including Helm values parsing and GitHub repository integration.

## Key Components

### Types
- **Builder** - Main configuration builder with dependencies on config service and UI operations
- **HelmValues** - Structured representation of Helm values YAML with deployment configurations for OSS and SaaS modes

### Methods
- **NewBuilder()** - Factory function creating a new Builder instance
- **getBranchForDeploymentMode()** - Extracts appropriate Git branch based on deployment mode (OSS/SaaS)
- **getBranchFromHelmValues()** - Reads Helm values file and determines which branch to use
- **BuildInstallConfig()** - Constructs standard installation configuration
- **BuildInstallConfigWithCustomHelmPath()** - Builds configuration with custom Helm values file path

## Usage Example

```go
// Create builder with UI operations
operationsUI := chartUI.NewOperationsUI()
builder := config.NewBuilder(operationsUI)

// Build standard install configuration
config, err := builder.BuildInstallConfig(
    false,                    // force
    false,                    // dryRun
    true,                     // verbose
    "my-cluster",             // clusterName
    "owner/repo",             // githubRepo
    "main",                   // githubBranch
    "/path/to/certs",         // certDir
)

// Build with custom Helm values path
customConfig, err := builder.BuildInstallConfigWithCustomHelmPath(
    false,                    // force
    false,                    // dryRun
    true,                     // verbose
    false,                    // nonInteractive
    "my-cluster",             // clusterName
    "owner/repo",             // githubRepo
    "main",                   // githubBranch
    "/path/to/certs",         // certDir
    "/custom/values.yaml",    // helmValuesPath
    "saas-shared",           // deploymentMode
)
```

The Builder automatically determines the correct Git branch based on deployment mode settings in Helm values files, supporting OSS tenant, SaaS tenant, and SaaS shared configurations.
