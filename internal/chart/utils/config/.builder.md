Configuration builder that constructs installation configurations for OpenFrame deployments, handling Helm values parsing and branch selection logic based on deployment modes.

## Key Components

- **Builder**: Main struct that orchestrates configuration building with configService and operationsUI dependencies
- **HelmValues**: Struct representing the YAML structure of Helm values files with deployment mode configurations
- **BuildInstallConfig()**: Constructs standard installation configuration with GitHub repository settings
- **BuildInstallConfigWithCustomHelmPath()**: Extended builder method supporting custom Helm values files and deployment modes
- **getBranchForDeploymentMode()**: Determines appropriate Git branch based on deployment mode (OSS, SaaS Tenant, SaaS Shared)
- **getBranchFromHelmValues()**: Extracts branch configuration from Helm values files

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/config"
    chartUI "github.com/flamingo-stack/openframe-cli/internal/chart/ui"
)

// Create builder with operations UI
operationsUI := chartUI.NewOperationsUI()
builder := config.NewBuilder(operationsUI)

// Build standard configuration
config, err := builder.BuildInstallConfig(
    false,                    // force
    false,                    // dryRun
    true,                     // verbose
    "my-cluster",             // clusterName
    "github.com/org/repo",    // githubRepo
    "main",                   // githubBranch
    "/path/to/certs",         // certDir
)

// Build with custom Helm values
config, err = builder.BuildInstallConfigWithCustomHelmPath(
    false,                    // force
    false,                    // dryRun
    true,                     // verbose
    false,                    // nonInteractive
    "my-cluster",             // clusterName
    "github.com/org/repo",    // githubRepo
    "main",                   // githubBranch
    "/path/to/certs",         // certDir
    "/custom/values.yaml",    // helmValuesPath
    "saas-tenant",            // deploymentMode
)
```
