Provides configuration building functionality for OpenFrame CLI chart installations, handling Helm values parsing and deployment mode-specific branch selection.

## Key Components

- **Builder**: Main configuration builder that constructs installation configurations
- **HelmValues**: Struct representing the structure of Helm values files with deployment settings
- **NewBuilder()**: Constructor function that creates a new Builder instance
- **BuildInstallConfig()**: Constructs installation configuration with standard parameters
- **BuildInstallConfigWithCustomHelmPath()**: Constructs configuration using custom Helm values file path
- **getBranchForDeploymentMode()**: Extracts appropriate branch based on deployment mode (OSS/SaaS)
- **getBranchFromHelmValues()**: Reads Helm values file to determine branch configuration

## Usage Example

```go
import (
    "github.com/flamingo-stack/openframe-cli/internal/chart/ui"
    "github.com/flamingo-stack/openframe-cli/internal/config"
)

// Create a new builder
operationsUI := &ui.OperationsUI{}
builder := config.NewBuilder(operationsUI)

// Build standard installation configuration
config, err := builder.BuildInstallConfig(
    false,                    // force
    false,                    // dryRun  
    true,                     // verbose
    "my-cluster",             // clusterName
    "owner/repo",             // githubRepo
    "main",                   // githubBranch
    "/path/to/certs",         // certDir
)

// Build configuration with custom Helm values
config, err := builder.BuildInstallConfigWithCustomHelmPath(
    false,                    // force
    false,                    // dryRun
    true,                     // verbose
    false,                    // nonInteractive
    "my-cluster",             // clusterName
    "owner/repo",             // githubRepo
    "main",                   // githubBranch
    "/path/to/certs",         // certDir
    "/custom/values.yaml",    // helmValuesPath
    "saas-shared",            // deploymentMode
)
```

The builder automatically handles branch selection from Helm values files and supports different deployment modes (OSS, SaaS Tenant, SaaS Shared) with appropriate repository branch mapping.
