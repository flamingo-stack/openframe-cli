Configuration model for Helm chart installation operations with optional app-of-apps pattern support.

## Key Components

### Types
- **`ChartInstallConfig`** - Main configuration struct for chart installation with cluster settings, execution flags, and optional app-of-apps configuration

### Methods
- **`HasAppOfApps()`** - Validates whether app-of-apps configuration is properly set with required GitHub repository

## Usage Example

```go
// Basic chart installation configuration
config := &ChartInstallConfig{
    ClusterName: "production",
    Force:       false,
    DryRun:      true,
    Verbose:     true,
}

// Configuration with app-of-apps pattern
appConfig := &ChartInstallConfig{
    ClusterName:    "staging",
    NonInteractive: true,
    AppOfApps: &models.AppOfAppsConfig{
        GitHubRepo: "myorg/gitops-repo",
        // additional app-of-apps settings...
    },
}

// Check if app-of-apps is configured
if appConfig.HasAppOfApps() {
    // Handle app-of-apps installation
}
```

The configuration supports both standalone chart installations and GitOps app-of-apps patterns, with flags for controlling execution behavior like dry runs, verbosity, and interactive modes.
