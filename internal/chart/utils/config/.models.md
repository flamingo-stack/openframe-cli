Core configuration model for OpenFrame chart installation operations, defining installation parameters and app-of-apps integration settings.

## Key Components

- **`ChartInstallConfig`** - Primary configuration struct containing installation parameters including cluster targeting, execution modes (dry-run, force, verbose), UI behavior controls, and CRD handling options
- **`HasAppOfApps()`** - Validation method that checks if app-of-apps configuration is properly configured with a GitHub repository

## Usage Example

```go
import "github.com/flamingo-stack/openframe-cli/internal/config"

// Create installation configuration
installConfig := &config.ChartInstallConfig{
    ClusterName:    "production",
    Force:          false,
    DryRun:         true,
    Verbose:        true,
    NonInteractive: false,
    SkipCRDs:       false,
    AppOfApps: &models.AppOfAppsConfig{
        GitHubRepo: "myorg/argocd-apps",
    },
}

// Check if app-of-apps is configured
if installConfig.HasAppOfApps() {
    fmt.Println("App-of-apps configuration detected")
}
```

The configuration supports both standalone chart installations and GitOps app-of-apps patterns, with flexible execution modes for testing and production deployments.
