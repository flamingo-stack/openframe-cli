Configuration models for OpenFrame CLI chart installation operations, providing structured configuration options and app-of-apps pattern support.

## Key Components

- **ChartInstallConfig**: Main configuration struct for chart installation operations
- **HasAppOfApps()**: Method to check if app-of-apps configuration is available

## Configuration Fields

The `ChartInstallConfig` struct provides comprehensive options for chart installations:

- **ClusterName**: Target cluster identifier for deployment
- **Force**: Bypass safety checks and force installation
- **DryRun**: Preview changes without applying them
- **Verbose**: Enable detailed logging output
- **Silent**: Suppress normal output messages
- **NonInteractive**: Disable interactive UI elements and spinners for automated environments
- **AppOfApps**: Configuration for the app-of-apps pattern using external chart models

## Usage Example

```go
config := &ChartInstallConfig{
    ClusterName:    "production-cluster",
    Force:          false,
    DryRun:         true,
    Verbose:        true,
    NonInteractive: true, // For CI/CD environments
    AppOfApps:      &models.AppOfAppsConfig{
        GitHubRepo: "myorg/helm-charts",
    },
}

// Check if app-of-apps is configured
if config.HasAppOfApps() {
    // Proceed with app-of-apps installation
    fmt.Println("Installing with app-of-apps pattern")
} else {
    // Standard chart installation
    fmt.Println("Installing standard chart")
}
```
