This file defines configuration types and constants for managing chart deployments across different deployment modes (OSS, SaaS, SaaS Shared) with support for various ingress types and registry configurations.

## Key Components

**Types:**
- `DeploymentMode` - Enum for deployment modes (OSS, SaaS, SaaS Shared)
- `IngressType` - Enum for ingress types (localhost, ngrok, GCP)
- `DockerRegistryConfig` - Docker registry credentials
- `NgrokConfig` - Ngrok tunnel configuration with IP allowlisting
- `SaaSConfig` - SaaS-specific repository settings
- `IngressConfig` - Ingress configuration wrapper
- `ChartConfiguration` - Main configuration struct for chart installations

**Constants:**
- `NgrokRegistrationURLs` - Struct containing Ngrok dashboard and documentation URLs

**Functions:**
- `GetRepositoryURL()` - Returns appropriate Git repository URL based on deployment mode

## Usage Example

```go
// Create a chart configuration
config := &ChartConfiguration{
    BaseHelmValuesPath: "/path/to/helm-values.yaml",
    DeploymentMode: &DeploymentModeSaaS,
    DockerRegistry: &DockerRegistryConfig{
        Username: "myuser",
        Password: "mypassword",
        Email: "user@example.com",
    },
    IngressConfig: &IngressConfig{
        Type: IngressTypeNgrok,
        NgrokConfig: &NgrokConfig{
            AuthToken: "your-token",
            Domain: "mydomain.ngrok.io",
            UseAllowedIPs: true,
            AllowedIPs: []string{"192.168.1.1"},
        },
    },
}

// Get repository URL for deployment
repoURL := GetRepositoryURL(DeploymentModeSaaS)
```
