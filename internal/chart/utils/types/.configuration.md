This file defines configuration types and constants for managing deployment settings across different modes (OSS, SaaS, SaaS Shared) and ingress types.

## Key Components

**Types:**
- `DockerRegistryConfig` - Docker registry credentials (username, password, email)
- `DeploymentMode` - Enum for deployment modes (oss, saas, saas-shared)
- `IngressType` - Enum for ingress types (localhost, ngrok, gcp)
- `NgrokConfig` - Ngrok tunnel configuration with auth, domain, and IP allowlist settings
- `SaaSConfig` - SaaS-specific settings including repository password and branch info
- `IngressConfig` - Wrapper for ingress type and associated configuration
- `ChartConfiguration` - Main configuration struct for Helm chart installation

**Variables:**
- `NgrokRegistrationURLs` - Predefined URLs for Ngrok registration and documentation

**Functions:**
- `GetRepositoryURL(mode DeploymentMode)` - Returns appropriate Git repository URL based on deployment mode

## Usage Example

```go
// Create a chart configuration
config := &ChartConfiguration{
    BaseHelmValuesPath: "./helm-values.yaml",
    TempHelmValuesPath: "/tmp/values.yaml",
    DeploymentMode:     &DeploymentModeSaaS,
    DockerRegistry: &DockerRegistryConfig{
        Username: "user",
        Password: "pass",
        Email:    "user@example.com",
    },
    IngressConfig: &IngressConfig{
        Type: IngressTypeNgrok,
        NgrokConfig: &NgrokConfig{
            AuthToken: "token123",
            Domain:    "myapp.ngrok.io",
        },
    },
}

// Get repository URL for deployment
repoURL := GetRepositoryURL(DeploymentModeSaaS)
```
