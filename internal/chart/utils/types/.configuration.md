Defines configuration types and constants for managing OpenFrame deployment settings across different modes (OSS, SaaS, SaaS Shared) and ingress types (localhost, Ngrok, GCP). Provides structured configuration management for Helm chart installations with support for Docker registries, networking, and deployment-specific parameters.

## Key Components

- **DeploymentMode**: Constants for OSS, SaaS, and SaaS Shared deployment modes
- **IngressType**: Constants for localhost, Ngrok, and GCP ingress configurations  
- **ChartConfiguration**: Main configuration struct for Helm chart installations with modification tracking
- **NgrokConfig**: Complete Ngrok tunnel configuration including auth, domains, and IP allowlists
- **SaaSConfig**: SaaS-specific settings for repository passwords and branch management
- **DockerRegistryConfig**: Docker registry credentials for private image access
- **GetRepositoryURL()**: Function to resolve correct repository URLs based on deployment mode

## Usage Example

```go
// Create a chart configuration for SaaS deployment
config := &ChartConfiguration{
    BaseHelmValuesPath: "/path/to/helm-values.yaml",
    TempHelmValuesPath: "/tmp/modified-values.yaml",
    DeploymentMode: &[]DeploymentMode{DeploymentModeSaaS}[0],
    DockerRegistry: &DockerRegistryConfig{
        Username: "registry-user",
        Password: "registry-pass",
        Email:    "user@example.com",
    },
    IngressConfig: &IngressConfig{
        Type: IngressTypeNgrok,
        NgrokConfig: &NgrokConfig{
            AuthToken: "your-ngrok-token",
            Domain:    "myapp.ngrok.io",
            UseAllowedIPs: true,
            AllowedIPs: []string{"192.168.1.0/24"},
        },
    },
}

// Get the appropriate repository URL
repoURL := GetRepositoryURL(DeploymentModeSaaS)
// Returns: "https://github.com/flamingo-stack/openframe-saas-tenant"
```
