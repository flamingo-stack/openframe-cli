This file defines a comprehensive set of interfaces for chart installation orchestration, providing contracts for Helm operations, Git management, ArgoCD services, and user interactions.

## Key Components

**Core Services:**
- `ChartInstaller` - Main orchestrator for chart installation processes
- `ServiceOrchestrator` - Manages coordination between services
- `WorkflowExecutor` - Executes complex workflows with step tracking

**Provider Interfaces:**
- `HelmProvider` - Manages Helm chart operations and ArgoCD installation
- `GitProvider` - Handles Git repository cloning and cleanup
- `ClusterLister` - Provides cluster listing capabilities

**Service Components:**
- `ArgoCDService` - Manages ArgoCD lifecycle (install, status, wait)
- `AppOfAppsService` - Manages app-of-apps pattern implementation

**Configuration & UI:**
- `ConfigBuilder` - Constructs installation configurations
- `PathResolver` - Resolves certificate and configuration file paths
- `OperationsUI` - Provides user interaction capabilities
- `ServiceFactory` - Creates service instances with dependency injection

**Result Types:**
- `WorkflowResult` - Contains workflow execution results with timing
- `InstallationRequest` - Encapsulates all installation parameters

## Usage Example

```go
// Implementing a basic chart installer
type MyChartInstaller struct {
    helmProvider HelmProvider
    gitProvider  GitProvider
    ui          OperationsUI
}

func (c *MyChartInstaller) InstallCharts(config config.ChartInstallConfig) error {
    // Use the orchestrator pattern
    orchestrator := &MyServiceOrchestrator{
        installer: c,
        ui:       c.ui,
    }
    
    request := InstallationRequest{
        Force:   true,
        DryRun:  false,
        Verbose: true,
    }
    
    return orchestrator.ExecuteInstallation(request)
}
```
