Defines interfaces and types for orchestrating Helm chart installations, particularly for ArgoCD and app-of-apps patterns. Provides a clean separation of concerns between chart management, cluster operations, Git repositories, and user interactions.

## Key Components

**Core Services:**
- `ChartInstaller` - Main orchestrator for chart installation workflows
- `ServiceOrchestrator` - Coordinates service interactions
- `WorkflowExecutor` - Executes multi-step workflows with tracking

**Provider Interfaces:**
- `HelmProvider` - Manages Helm chart operations and status checking
- `GitProvider` - Handles Git repository cloning and cleanup
- `ClusterLister` - Lists available clusters

**Service Components:**
- `ArgoCDService` - ArgoCD lifecycle management
- `AppOfAppsService` - App-of-apps pattern management

**Configuration & UI:**
- `ConfigBuilder` - Builds installation configurations
- `OperationsUI` - User interaction and prompts
- `ClusterSelector` - Cluster selection logic

**Result Types:**
- `WorkflowResult` - Execution results with timing and status
- `InstallationRequest` - Installation parameters container

## Usage Example

```go
// Basic installation workflow
installer := factory.CreateInstaller()
config := configBuilder.BuildInstallConfig(false, false, true, 
    "my-cluster", "github.com/org/charts", "main", "/certs")

err := installer.InstallCharts(config)
if err != nil {
    log.Fatal(err)
}

// Service orchestration
orchestrator := factory.CreateOrchestrator()
req := InstallationRequest{
    Args:         []string{"cluster-name"},
    Force:        false,
    DryRun:       false,
    GitHubRepo:   "github.com/org/charts",
    GitHubBranch: "main",
}
err := orchestrator.ExecuteInstallation(req)
```
