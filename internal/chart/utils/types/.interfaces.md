This file defines the core interfaces for the OpenFrame CLI chart installation system, establishing contracts for services, providers, and orchestration components.

## Key Components

**Core Services:**
- `ChartInstaller` - Main orchestrator for chart installation process
- `ArgoCDService` - Manages ArgoCD lifecycle and operations
- `AppOfAppsService` - Handles app-of-apps pattern installations

**Provider Interfaces:**
- `HelmProvider` - Abstracts Helm chart operations and status checking
- `GitProvider` - Manages Git repository cloning and cleanup
- `ClusterLister` - Provides cluster discovery capabilities

**Configuration & UI:**
- `ConfigBuilder` - Constructs installation configurations from parameters
- `PathResolver` - Resolves file system paths for certificates and values
- `OperationsUI` - Handles user interactions and confirmations
- `ClusterSelector` - Manages cluster selection logic

**Orchestration:**
- `ServiceOrchestrator` - Coordinates service execution
- `WorkflowExecutor` - Manages step-by-step workflow execution with tracking
- `ServiceFactory` - Creates service instances with dependency injection

## Usage Example

```go
// Create a workflow executor for installation
var executor WorkflowExecutor
executor.AddStep("validate-config", "Validate installation config", func() error {
    return validateConfig()
}, true)
executor.AddStep("install-argocd", "Install ArgoCD", func() error {
    return argoCDService.Install(ctx, config)
}, true)

// Execute the workflow
result := executor.Execute()
if !result.Success {
    return result.Error
}

// Use the orchestrator for complete installation
req := InstallationRequest{
    Args:      []string{"my-cluster"},
    Force:     false,
    DryRun:    false,
    Verbose:   true,
    GitHubRepo: "my-org/my-charts",
}
return orchestrator.ExecuteInstallation(req)
```
