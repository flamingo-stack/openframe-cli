Comprehensive error handling system for Helm chart operations with enhanced context, recovery mechanisms, and specialized error types for different failure scenarios.

## Key Components

- **ChartError**: Core error type with operation context, cluster info, and retry capabilities
- **InstallationError**: Specialized error for installation failures with phase tracking and troubleshooting steps
- **ValidationError**: Handles configuration validation failures with field-specific details
- **ConfigurationError**: Manages configuration file parsing and validation errors
- **SkippedInstallationError**: Non-error type for tracking intentionally skipped installations
- **Error Constants**: Predefined common error types (ErrChartNotFound, ErrClusterNotReady, etc.)
- **Helper Functions**: Utilities for error type checking, retry logic, and error wrapping

## Usage Example

```go
// Create a basic chart error
err := NewChartError("install", "prometheus", fmt.Errorf("connection failed"))
err = err.WithCluster("production").WithRecovery(30 * time.Second)

// Handle installation errors with troubleshooting
installErr := NewInstallationError("grafana", "pre-install", ErrClusterNotReady)
installErr = installErr.WithSuggestions([]string{
    "Check node resources",
    "Verify storage classes",
})

// Validation errors with field context
validationErr := NewValidationError("replicas", "-1", "must be positive integer")

// Check error types and recovery
if IsRecoverable(err) {
    delay := GetRetryDelay(err)
    // Wait and retry after delay
}

// Combine multiple errors
errors := []error{err, installErr, validationErr}
combined := CombineErrors(errors)
```
