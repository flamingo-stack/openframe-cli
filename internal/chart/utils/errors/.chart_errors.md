Defines chart-specific error types for Kubernetes Helm chart operations with enhanced context, recovery mechanisms, and troubleshooting support.

## Key Components

- **ChartError**: Base error type with operation context, cluster info, and retry capabilities
- **InstallationError**: Specialized error for chart installation failures with phase tracking
- **ValidationError**: Handles configuration validation failures with field-specific details
- **ConfigurationError**: Manages config file and missing key errors
- **SkippedInstallationError**: Represents intentionally skipped installations (not failures)
- **Predefined Errors**: Common chart operation error constants
- **Helper Functions**: Utilities for error type checking, retry logic, and error combination

## Usage Example

```go
package main

import (
    "fmt"
    "time"
)

func installChart() error {
    // Create recoverable error with retry delay
    err := NewRecoverableChartError(
        "install", 
        "prometheus", 
        fmt.Errorf("connection timeout"),
        30*time.Second,
    ).WithCluster("production")

    return err
}

func handleError(err error) {
    // Check if error is recoverable
    if IsRecoverable(err) {
        delay := GetRetryDelay(err)
        fmt.Printf("Retrying after %v\n", delay)
    }

    // Handle specific error types
    if installErr, ok := err.(*InstallationError); ok {
        steps := installErr.GetTroubleshootingSteps()
        fmt.Printf("Troubleshooting: %v\n", steps)
    }
}

// Create validation error
func validateConfig(field, value string) error {
    return NewValidationError(
        field, 
        value, 
        "must be between 1-100",
    )
}
```
