Provides specialized error types and utilities for Helm chart operations, offering structured error handling with recovery mechanisms and detailed context for troubleshooting chart installations and configurations.

## Key Components

**Core Error Types:**
- `ChartError` - Base error type with operation context, cluster info, and retry capabilities
- `InstallationError` - Installation-specific errors with phase tracking and troubleshooting steps
- `ValidationError` - Configuration validation errors with field-level details
- `ConfigurationError` - Config file parsing errors with missing key tracking
- `SkippedInstallationError` - Non-error type for intentionally skipped installations

**Predefined Error Variables:**
- Common chart errors: `ErrChartNotFound`, `ErrChartAlreadyInstalled`, `ErrInvalidConfiguration`
- Infrastructure errors: `ErrClusterNotReady`, `ErrHelmNotAvailable`, `ErrInsufficientResources`
- Network/Auth errors: `ErrNetworkTimeout`, `ErrAuthenticationFailed`, `ErrPermissionDenied`

**Utility Functions:**
- `IsTimeout()`, `IsRecoverable()`, `GetRetryDelay()` - Error classification helpers
- `WrapAsChartError()`, `CombineErrors()` - Error transformation utilities

## Usage Example

```go
// Create a recoverable chart error with retry delay
err := NewRecoverableChartError("install", "prometheus", 
    ErrNetworkTimeout, 30*time.Second).WithCluster("prod-cluster")

if IsRecoverable(err) {
    delay := GetRetryDelay(err)
    // Retry after delay
}

// Create installation error with troubleshooting steps
installErr := NewInstallationError("grafana", "pre-install", ErrClusterNotReady).
    WithSuggestions([]string{"Check node resources", "Verify storage class"})

steps := installErr.GetTroubleshootingSteps()

// Create validation error for config fields
validationErr := NewValidationError("replicas", "0", "must be greater than 0")
```
