Provides comprehensive error handling for chart operations with detailed context, retry logic, and specialized error types for different failure scenarios.

## Key Components

**Core Error Types:**
- `ChartError` - Base error type with operation context, cluster info, and retry capabilities
- `InstallationError` - Installation-specific errors with phase tracking and troubleshooting steps
- `ValidationError` - Configuration validation failures with field-level details
- `ConfigurationError` - Configuration file parsing and validation errors
- `SkippedInstallationError` - Non-error type for intentionally skipped installations

**Predefined Errors:**
- Common chart errors like `ErrChartNotFound`, `ErrClusterNotReady`, `ErrNetworkTimeout`

**Helper Functions:**
- `IsTimeout()`, `IsRecoverable()`, `GetRetryDelay()` - Error type checking utilities
- `WrapAsChartError()` - Converts generic errors to chart errors
- `CombineErrors()` - Merges multiple errors into a single error

## Usage Example

```go
// Create a recoverable chart error with retry logic
err := NewRecoverableChartError("install", "nginx", 
    ErrNetworkTimeout, 30*time.Second).WithCluster("prod-cluster")

// Check if error can be retried
if IsRecoverable(err) {
    retryDelay := GetRetryDelay(err)
    time.Sleep(retryDelay)
}

// Create installation error with troubleshooting steps
installErr := NewInstallationError("grafana", "pre-install", ErrClusterNotReady).
    WithSuggestions([]string{"Check node resources", "Verify network connectivity"})

// Create validation error for configuration
validationErr := NewValidationError("replicas", "0", "must be greater than 0")

// Combine multiple errors
errors := []error{installErr, validationErr}
combined := CombineErrors(errors)
```
