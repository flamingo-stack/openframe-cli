This file provides a comprehensive error handling system specifically designed for Helm chart operations, with enhanced context, retry capabilities, and specialized error types for different failure scenarios.

## Key Components

**Core Error Types:**
- `ChartError` - Main error type with operation context, cluster info, and retry capabilities
- `InstallationError` - Installation-specific errors with phase tracking and troubleshooting steps
- `ValidationError` - Configuration validation failures with field-level details
- `ConfigurationError` - Configuration file parsing and validation errors
- `SkippedInstallationError` - Non-error case for intentionally skipped installations

**Predefined Error Variables:**
- Common chart errors like `ErrChartNotFound`, `ErrClusterNotReady`, `ErrNetworkTimeout`

**Helper Functions:**
- `IsTimeout()`, `IsRecoverable()`, `GetRetryDelay()` - Error classification utilities
- `WrapAsChartError()`, `CombineErrors()` - Error wrapping and aggregation

## Usage Example

```go
// Create a basic chart error
err := NewChartError("install", "nginx", fmt.Errorf("connection failed"))

// Create recoverable error with retry delay
retryErr := NewRecoverableChartError("deploy", "app", ErrNetworkTimeout, 30*time.Second).
    WithCluster("production")

// Handle installation error with troubleshooting
installErr := NewInstallationError("prometheus", "pre-install", ErrInsufficientResources).
    WithSuggestions([]string{"Scale up cluster nodes", "Reduce resource requests"})

// Check error properties
if IsRecoverable(err) {
    time.Sleep(GetRetryDelay(err))
    // retry operation
}
```
