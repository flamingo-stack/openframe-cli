Provides comprehensive error handling for Helm chart operations with rich context, recovery mechanisms, and specialized error types for different failure scenarios.

## Key Components

### Core Types
- `ChartError` - Base error type with operation context, cluster info, and retry capabilities
- `InstallationError` - Installation-specific errors with phase tracking and troubleshooting steps
- `ValidationError` - Field validation errors with constraint details
- `ConfigurationError` - Configuration file errors with missing key tracking
- `SkippedInstallationError` - Represents intentionally skipped installations

### Error Constants
- Pre-defined errors: `ErrChartNotFound`, `ErrClusterNotReady`, `ErrNetworkTimeout`, etc.
- Common failure scenarios with consistent error messages

### Utility Functions
- `IsTimeout()`, `IsRecoverable()`, `GetRetryDelay()` - Error classification helpers
- `WrapAsChartError()` - Converts generic errors to chart errors
- `CombineErrors()` - Aggregates multiple errors into one

## Usage Example

```go
// Create a recoverable installation error
err := NewRecoverableChartError("install", "nginx", 
    ErrNetworkTimeout, 30*time.Second).
    WithCluster("production")

// Handle with retry logic
if IsRecoverable(err) {
    time.Sleep(GetRetryDelay(err))
    // retry operation
}

// Create validation error with field details
validationErr := NewValidationError("replicas", "0", "must be > 0")

// Installation error with troubleshooting
installErr := NewInstallationError("prometheus", "helm-install", cause).
    WithSuggestions([]string{
        "Check Helm repository access",
        "Verify namespace exists",
    })
```
