This file contains comprehensive unit tests for the operations UI component, which handles user interactions during cluster operations like installations.

## Key Components

- **`testError`** - Simple error implementation for testing error handling scenarios
- **`TestNewOperationsUI`** - Tests UI initialization
- **`TestSelectClusterForInstall_*`** - Tests cluster selection logic with various input scenarios
- **`TestShow*`** - Tests UI display methods for different operation states
- **`TestOperationsUI_Integration`** - End-to-end integration test covering complete workflow

## Usage Example

```go
func TestOperationFlow(t *testing.T) {
    ui := NewOperationsUI()
    
    clusters := []models.ClusterInfo{
        {Name: "test-cluster", Status: "running"},
    }
    
    // Test cluster selection with arguments
    cluster, err := ui.SelectClusterForInstall(clusters, []string{"test-cluster"})
    assert.NoError(t, err)
    assert.Equal(t, "test-cluster", cluster)
    
    // Test UI methods don't panic
    assert.NotPanics(t, func() {
        ui.ShowInstallationStart(cluster)
        ui.ShowInstallationComplete()
    })
}
```

The tests cover both argument-based and interactive cluster selection modes, error handling for invalid inputs, and ensure all UI display methods execute without panicking. Some interactive tests are skipped as they require user input.
