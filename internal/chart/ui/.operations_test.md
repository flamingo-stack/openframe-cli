This file contains comprehensive unit tests for the operations UI functionality, testing cluster selection, installation workflows, and user interaction methods.

## Key Components

- **`testError`** - Simple error implementation for testing scenarios
- **Test Functions**:
  - `TestNewOperationsUI()` - Tests OperationsUI constructor
  - `TestSelectClusterForInstall_*` - Tests cluster selection with arguments and interactive mode
  - `TestShowOperationCancelled()` - Tests operation cancellation display
  - `TestShowNoClusterMessage()` - Tests no cluster available message
  - `TestShowInstallation*()` - Tests installation status messages
  - `TestOperationsUI_Integration()` - Tests complete workflow scenarios

## Usage Example

```go
// Run specific test
go test -run TestSelectClusterForInstall_WithClusterArgument

// Run all operations UI tests
go test ./ui/operations_test.go

// Test cluster selection functionality
clusters := []models.ClusterInfo{
    {Name: "test-cluster", Status: "running"},
}
ui := NewOperationsUI()
selected, err := ui.SelectClusterForInstall(clusters, []string{"test-cluster"})

// Test error handling
ui.ShowInstallationError(errors.New("test error"))
```

The tests cover both argument-based and interactive cluster selection modes, ensuring proper error handling for edge cases like empty cluster lists, invalid names, and user cancellation scenarios.
