Test suite for the ConfigurationWizard factory function and component initialization. Verifies that the wizard creates properly initialized instances with all required configuration components.

## Key Components

- **TestNewConfigurationWizard**: Tests basic wizard creation and nil-checks for all components
- **TestConfigurationWizard_Structure**: Validates proper initialization of the wizard structure and its sub-components
- **TestConfigurationWizard_Components**: Ensures each wizard instance has unique component instances (no shared references)

## Usage Example

```go
func TestCustomWizardUsage(t *testing.T) {
    // Create a new configuration wizard
    wizard := NewConfigurationWizard()
    
    // Verify wizard is properly initialized
    assert.NotNil(t, wizard)
    assert.NotNil(t, wizard.modifier)
    assert.NotNil(t, wizard.branchConfig)
    assert.NotNil(t, wizard.dockerConfig)
    assert.NotNil(t, wizard.ingressConfig)
    
    // Create multiple wizards to ensure independence
    wizard1 := NewConfigurationWizard()
    wizard2 := NewConfigurationWizard()
    assert.NotSame(t, wizard1, wizard2)
}
```

The tests use testify assertions to validate that the `NewConfigurationWizard()` constructor properly initializes all configuration components and ensures each wizard instance maintains its own separate component instances.
