Package that orchestrates the Helm chart configuration workflow through an interactive wizard interface.

## Key Components

**ConfigurationWizard** - Main orchestrator that coordinates different configuration steps:
- `modifier` - HelmValuesModifier for updating chart values
- `branchConfig` - BranchConfigurator for Git branch settings
- `dockerConfig` - DockerConfigurator for container settings  
- `ingressConfig` - IngressConfigurator for ingress/routing settings

**Primary Methods:**
- `NewConfigurationWizard()` - Factory function that initializes all configurators
- `ConfigureHelmValues()` - Interactive configuration with deployment mode selection
- `ConfigureHelmValuesWithMode()` - Configuration with pre-selected deployment mode

**Configuration Flow:**
1. Deployment mode selection (local/remote/cloud)
2. Configuration approach selection (default vs interactive)
3. Delegates to specific configurators based on user choices

## Usage Example

```go
// Create and run configuration wizard
wizard := NewConfigurationWizard()

// Interactive configuration with mode selection
config, err := wizard.ConfigureHelmValues()
if err != nil {
    return fmt.Errorf("configuration failed: %w", err)
}

// Configuration with predefined mode
config, err = wizard.ConfigureHelmValuesWithMode(types.CloudDeployment)
if err != nil {
    return fmt.Errorf("cloud configuration failed: %w", err)
}

fmt.Printf("Chart configured for %s deployment\n", config.DeploymentMode)
```

The wizard provides a user-friendly interface for configuring OpenFrame deployments across different environments while maintaining separation of concerns through specialized configurator components.
