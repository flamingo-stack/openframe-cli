A configuration wizard that orchestrates the Helm chart configuration workflow by coordinating branch, Docker, and ingress configuration steps through an interactive user interface.

## Key Components

- **ConfigurationWizard**: Main orchestrator struct that manages the configuration workflow
- **NewConfigurationWizard()**: Factory function that creates a wizard with all required configurators
- **ConfigureHelmValues()**: Full configuration workflow with deployment mode and configuration mode selection
- **ConfigureHelmValuesWithMode()**: Streamlined workflow that skips deployment mode selection when mode is pre-determined

The wizard integrates four key components:
- `templates.HelmValuesModifier`: Handles Helm values file modifications
- `BranchConfigurator`: Manages Git branch configuration
- `DockerConfigurator`: Handles Docker image and registry settings
- `IngressConfigurator`: Configures ingress routing and domain settings

## Usage Example

```go
// Create a new configuration wizard
wizard := NewConfigurationWizard()

// Interactive configuration with mode selection
config, err := wizard.ConfigureHelmValues()
if err != nil {
    log.Fatal("Configuration failed:", err)
}

// Or configure with predetermined deployment mode
config, err := wizard.ConfigureHelmValuesWithMode(types.ProductionMode)
if err != nil {
    log.Fatal("Configuration failed:", err)
}

// The returned ChartConfiguration contains all user selections
fmt.Printf("Deployment mode: %s\n", config.DeploymentMode)
```

The wizard provides both default and interactive configuration modes, allowing users to either accept sensible defaults or customize their deployment settings through guided prompts.
