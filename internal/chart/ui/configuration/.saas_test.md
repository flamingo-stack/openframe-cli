Test suite for the SaaS branch extraction functionality in the configuration wizard, validating how the system retrieves and handles SaaS deployment branch configurations.

## Key Components

- **TestConfigurationWizard_GetSaaSBranchFromValues**: Comprehensive test for `getSaaSBranchFromValues()` method with various input scenarios
- **TestConfigurationWizard_SaaSBranchExtraction**: Integration test for complex nested configuration structures
- **TestConfigurationWizard_SaaSConfigStructure**: Structural validation tests for SaaS configuration handling

## Usage Example

```go
// Run tests to validate SaaS branch extraction
go test -v ./configuration -run TestConfigurationWizard_GetSaaSBranchFromValues

// Test with specific configuration structure
wizard := NewConfigurationWizard()
values := map[string]interface{}{
    "deployment": map[string]interface{}{
        "saas": map[string]interface{}{
            "repository": map[string]interface{}{
                "branch": "develop",
            },
        },
    },
}

branch := wizard.getSaaSBranchFromValues(values)
// Returns: "develop"

// Test edge cases
emptyValues := map[string]interface{}{}
defaultBranch := wizard.getSaaSBranchFromValues(emptyValues)
// Returns: "main" (default value)
```

The test suite covers edge cases including nil values, empty configurations, incomplete structures, and validates that the default branch "main" is returned when no SaaS branch is specified.
