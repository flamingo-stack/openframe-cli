This file implements ingress configuration management for the OpenFrame CLI, handling different deployment modes including localhost, Ngrok tunneling, and GCP cloud deployments.

## Key Components

### IngressConfigurator
- **IngressConfigurator**: Main struct that manages ingress configuration using a Helm values modifier
- **NewIngressConfigurator()**: Factory function to create a new configurator instance

### Primary Methods
- **Configure()**: Main entry point that prompts users for ingress type selection based on deployment mode
- **configureNgrok()**: Handles complete Ngrok tunnel setup including registration prompts and credential collection
- **getCurrentNgrokSettings()**: Extracts existing Ngrok configuration from Helm values
- **collectNgrokCredentials()**: Interactive collection of Ngrok domain, API key, and auth token
- **configureNgrokIPAllowlist()**: Sets up IP restriction policies for Ngrok tunnels

### Configuration Appliers
- **applyLocalhostConfig()**: Configures local-only access for development
- **applyNgrokConfig()**: Applies Ngrok tunnel settings with credentials and IP restrictions
- **applyGCPConfig()**: Sets up GCP ingress with tenant ID configuration

## Usage Example

```go
modifier := templates.NewHelmValuesModifier()
configurator := NewIngressConfigurator(modifier)

config := &types.ChartConfiguration{
    DeploymentMode: &types.DeploymentModeOSS,
    ExistingValues: make(map[string]interface{}),
}

// Interactive configuration
err := configurator.Configure(config)
if err != nil {
    log.Fatal(err)
}

// Access the configured ingress settings
if config.IngressConfig.Type == types.IngressTypeNgrok {
    fmt.Printf("Ngrok domain: %s\n", config.IngressConfig.NgrokConfig.Domain)
}
```

The configurator adapts its options based on deployment mode - SaaS deployments offer localhost/GCP choices while OSS deployments include Ngrok tunneling for external access.
