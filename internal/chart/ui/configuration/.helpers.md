Helper functions for the OpenFrame configuration wizard that manage Helm values and provide user feedback during the configuration process.

## Key Components

- **loadBaseValues()** - Loads existing helm-values.yaml from current directory or creates default configuration
- **createTemporaryValuesFile()** - Applies configuration changes and creates temporary Helm values file for installation
- **ShowConfigurationSummary()** - Displays a formatted summary of all configuration changes made during the wizard

## Usage Example

```go
// Load base configuration
config, err := wizard.loadBaseValues()
if err != nil {
    return fmt.Errorf("failed to load base values: %w", err)
}

// Apply configuration changes
err = wizard.createTemporaryValuesFile(config)
if err != nil {
    return fmt.Errorf("failed to create temporary values: %w", err)
}

// Display summary to user
wizard.ShowConfigurationSummary(config)
```

The `loadBaseValues` method creates a `ChartConfiguration` struct with the base helm-values.yaml file path and existing values. The `createTemporaryValuesFile` method applies all configuration modifications and generates a temporary file for Helm installation. The `ShowConfigurationSummary` method provides user-friendly output showing which configuration sections were modified, including deployment mode, SaaS settings, branch selection, Docker registry, and ingress configuration details.
