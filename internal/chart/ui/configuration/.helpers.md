Utility functions for managing Helm chart configuration during the OpenFrame CLI setup wizard process.

## Key Components

- **loadBaseValues()** - Loads existing Helm values from `helm-values.yaml` or creates default configuration
- **createTemporaryValuesFile()** - Applies configuration changes and creates a temporary values file for installation
- **ShowConfigurationSummary()** - Displays a formatted summary of all configuration changes made during the wizard

## Usage Example

```go
wizard := &ConfigurationWizard{
    modifier: valuesModifier,
}

// Load base configuration
config, err := wizard.loadBaseValues()
if err != nil {
    log.Fatal(err)
}

// Make configuration changes...
config.ModifiedSections = append(config.ModifiedSections, "deployment")
config.DeploymentMode = &types.DeploymentModeDevelopment

// Create temporary file for installation
if err := wizard.createTemporaryValuesFile(config); err != nil {
    log.Fatal(err)
}

// Show summary of changes
wizard.ShowConfigurationSummary(config)
```

These helper functions work together to manage the lifecycle of Helm chart configuration, from loading existing values through applying changes and preparing files for deployment. The summary function provides user-friendly feedback about deployment mode, SaaS settings, branch selection, Docker registry, and ingress configuration changes.
