This file provides SaaS (Software as a Service) configuration functionality for the OpenFrame CLI, handling repository credentials, branch selection, and Docker registry setup for SaaS deployments.

## Key Components

- **`configureSaaSDefaults`** - Configures required SaaS settings in default mode, using existing values for branches
- **`configureSaaSInteractive`** - Configures SaaS settings with full user interaction for branch selection
- **`configureSaaSBranch`** - Interactive configuration for SaaS repository branch selection
- **`configureOSSBranchForSaaS`** - Interactive configuration for OSS repository branch in SaaS context
- **`getSaaSBranchFromValues`** - Extracts current SaaS branch from existing Helm values

## Usage Example

```go
// Configure SaaS in default mode (minimal prompts)
config := &types.ChartConfiguration{
    ExistingValues: existingHelmValues,
}
err := wizard.configureSaaSDefaults(config)
if err != nil {
    log.Fatal(err)
}

// Configure SaaS interactively (full prompts)
err = wizard.configureSaaSInteractive(config)
if err != nil {
    log.Fatal(err)
}

// Access configured SaaS settings
fmt.Printf("SaaS Branch: %s\n", config.SaaSConfig.SaaSBranch)
fmt.Printf("OSS Branch: %s\n", config.SaaSConfig.OSSBranch)
```

Both configuration modes collect repository passwords and GitHub Container Registry credentials, but differ in branch configuration - default mode preserves existing branch values while interactive mode allows full customization.
