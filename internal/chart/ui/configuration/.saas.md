Provides SaaS (Software as a Service) deployment configuration functionality for the OpenFrame CLI, handling credentials, repository settings, and branch configurations for both default and interactive modes.

## Key Components

- **configureSaaSDefaults**: Configures SaaS settings in default mode, prompting only for required credentials while using existing branch values
- **configureSaaSInteractive**: Full interactive SaaS configuration allowing users to modify all settings including repository branches
- **configureSaaSBranch**: Interactive branch selection for SaaS repository with current branch detection
- **configureOSSBranchForSaaS**: OSS repository branch configuration specifically for SaaS deployments
- **getSaaSBranchFromValues**: Extracts current SaaS branch from existing Helm values with fallback to "main"

## Usage Example

```go
// Configure SaaS in default mode (minimal prompts)
wizard := &ConfigurationWizard{}
config := &types.ChartConfiguration{
    ExistingValues: existingHelmValues,
}

err := wizard.configureSaaSDefaults(config)
if err != nil {
    log.Fatal("SaaS default configuration failed:", err)
}

// Configure SaaS interactively (full prompts)
err = wizard.configureSaaSInteractive(config)
if err != nil {
    log.Fatal("SaaS interactive configuration failed:", err)
}

// Access configured values
fmt.Printf("SaaS Branch: %s\n", config.SaaSConfig.SaaSBranch)
fmt.Printf("OSS Branch: %s\n", config.SaaSConfig.OSSBranch)
fmt.Printf("Registry Username: %s\n", config.DockerRegistry.Username)
```

The module handles sensitive credential collection with masked input, integrates with GitHub Container Registry configuration, and maintains consistency between SaaS and OSS repository branch settings.
