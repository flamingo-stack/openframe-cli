This file provides SaaS-specific configuration functionality for the OpenFrame CLI configuration wizard, handling authentication credentials and repository branch settings for SaaS deployments.

## Key Components

- **`configureSaaSDefaults`** - Configures required SaaS settings in default mode, prompting only for essential credentials while using existing branch values
- **`configureSaaSInteractive`** - Full interactive SaaS configuration mode with branch selection prompts
- **`configureSaaSBranch`** - Interactive configuration for SaaS repository branch selection
- **`configureOSSBranchForSaaS`** - Interactive configuration for OSS repository branch in SaaS context
- **`getSaaSBranchFromValues`** - Extracts current SaaS branch from existing helm values

## Usage Example

```go
// Configure SaaS in default mode (minimal prompts)
err := wizard.configureSaaSDefaults(config)
if err != nil {
    return fmt.Errorf("SaaS default configuration failed: %w", err)
}

// Configure SaaS in interactive mode (full prompts)
err = wizard.configureSaaSInteractive(config)
if err != nil {
    return fmt.Errorf("SaaS interactive configuration failed: %w", err)
}

// The resulting config will have populated SaaSConfig and DockerRegistry:
// config.SaaSConfig.RepositoryPassword
// config.SaaSConfig.SaaSBranch
// config.SaaSConfig.OSSBranch
// config.DockerRegistry.Username/Password/Email
```

Both configuration modes collect repository access tokens and GitHub Container Registry credentials, with the interactive mode additionally prompting for branch selections while the default mode preserves existing branch configurations.
