This test file validates the extraction and configuration of GitHub Container Registry (GHCR) credentials from Helm chart values, ensuring proper handling of various credential scenarios.

## Key Components

- **TestConfigurationWizard_ExtractGHCRCredentials**: Comprehensive test for extracting GHCR credentials from existing chart values
- **TestConfigurationWizard_GHCRCredentialsInConfig**: Test for verifying GHCR credentials storage in configuration objects
- **NewConfigurationWizard()**: Constructor test for the configuration wizard

## Test Scenarios

The main test covers multiple credential extraction scenarios:
- No existing credentials (defaults applied)
- Valid existing GHCR credentials
- Default/empty credentials (treated as non-existent)
- Partial registry configurations
- Missing GHCR sections

## Usage Example

```go
// Test runs automatically with go test
go test ./internal/configuration -v

// Example of the credential extraction logic being tested:
config := &types.ChartConfiguration{
    ExistingValues: map[string]interface{}{
        "registry": map[string]interface{}{
            "ghcr": map[string]interface{}{
                "username": "myuser",
                "email": "user@example.com",
                "password": "token123",
            },
        },
    },
}

// The test validates that credentials are properly extracted
// and that default values are applied when credentials are missing
```
