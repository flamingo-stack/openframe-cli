Test file for configuration wizard functionality, covering Helm values management, deployment modes, and integration workflows for chart configuration.

## Key Components

- **TestConfigurationWizard_ConfigureWithDefaults_OSS**: Tests OSS mode configuration with default values and temporary file creation
- **TestConfigurationWizard_ConfigureWithExistingFile**: Validates loading and parsing existing Helm values files
- **TestConfigurationWizard_Integration_LoadAndApply**: End-to-end integration test covering the complete configuration workflow
- **TestConfigurationWizard_DeploymentModes**: Unit tests for deployment mode constants (OSS/SaaS)
- **TestConfigurationWizard_LoadBaseValues**: Tests base configuration loading functionality

## Usage Example

```go
// Run specific test
go test -run TestConfigurationWizard_ConfigureWithDefaults_OSS

// Run all configuration wizard tests
go test -run TestConfigurationWizard

// Run with verbose output to see detailed test execution
go test -v ./configuration/modes_test.go
```

The tests verify Helm values modification, file I/O operations, configuration merging, and proper handling of both OSS and SaaS deployment modes. They use temporary files and directories to ensure clean test execution without affecting the actual configuration files.
