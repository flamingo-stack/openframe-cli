Test suite for the UI display service that handles terminal output formatting and user feedback. Contains unit tests validating display methods for installation progress, success/error states, and dry-run results.

## Key Components

- **NewDisplayService()** - Constructor test for display service initialization
- **ShowInstallProgress()** - Tests progress indicator display during chart installation
- **ShowInstallSuccess()** - Validates success message formatting with chart information
- **ShowInstallError()** - Tests error message display functionality
- **ShowPreInstallCheck()** - Tests pre-installation validation message display
- **ShowDryRunResults()** - Tests dry-run output formatting to buffers
- **getChartDisplayName()** - Tests chart type to display name conversion

## Usage Example

```go
// Running the display service tests
func TestDisplayServiceUsage(t *testing.T) {
    service := NewDisplayService()
    
    // Test progress display
    assert.NotPanics(t, func() {
        service.ShowInstallProgress(models.ChartTypeArgoCD, "Installing...")
    })
    
    // Test dry-run output capture
    var buf bytes.Buffer
    results := []string{"Would install ArgoCD", "Would create namespace"}
    service.ShowDryRunResults(&buf, results)
    
    // Verify output contains expected results
    output := buf.String()
    assert.Contains(t, output, "Would install ArgoCD")
}
```

The tests primarily use panic validation since the display service uses pterm for terminal output, making direct output capture challenging in most cases.
