Test file for the DisplayService component that handles terminal output formatting and user interface display operations.

## Key Components

- **TestNewDisplayService** - Validates DisplayService constructor
- **TestDisplayService_ShowInstallProgress** - Tests installation progress display
- **TestDisplayService_ShowInstallSuccess** - Tests successful installation output
- **TestDisplayService_ShowInstallError** - Tests error display functionality
- **TestDisplayService_ShowPreInstallCheck** - Tests pre-installation check messages
- **TestDisplayService_ShowDryRunResults** - Tests dry-run output with various scenarios
- **TestDisplayService_getChartDisplayName** - Tests chart type name formatting

## Usage Example

```go
func TestDisplayServiceExample(t *testing.T) {
    service := NewDisplayService()
    
    // Test basic functionality
    assert.NotNil(t, service)
    
    // Test dry run output capture
    var buf bytes.Buffer
    results := []string{"Would install ArgoCD v8.2.7"}
    service.ShowDryRunResults(&buf, results)
    
    output := buf.String()
    assert.Contains(t, output, "Would install ArgoCD v8.2.7")
    
    // Test chart name formatting
    displayName := service.getChartDisplayName(models.ChartTypeArgoCD)
    assert.Equal(t, "ArgoCD", displayName)
}
```

The tests focus on ensuring methods don't panic and that output formatting works correctly, particularly for dry-run results which can be captured and validated.
