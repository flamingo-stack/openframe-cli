The DisplayService provides formatted console output for chart installation operations, using pterm for styled terminal output and specific icons for different chart types.

## Key Components

- **DisplayService** - Main service struct for handling chart-related UI display operations
- **NewDisplayService()** - Factory function that creates a new display service instance
- **ShowInstallProgress()** - Displays installation progress with chart-type-specific icons (üöÄ for ArgoCD, üì¶ for App-of-Apps)
- **ShowInstallSuccess()** - Handles successful installation display (currently delegated to service layer)
- **ShowInstallError()** - Shows installation error messages with formatted output
- **ShowSkippedInstallation()** - Displays when component installation is skipped with reason
- **ShowPreInstallCheck()** - Shows pre-installation check messages with üîç icon
- **ShowDryRunResults()** - Outputs dry-run results in a formatted list
- **getChartDisplayName()** - Returns user-friendly names for chart types

## Usage Example

```go
// Create a display service
display := NewDisplayService()

// Show installation progress
display.ShowInstallProgress(models.ChartTypeArgoCD, "Installing ArgoCD charts...")

// Show pre-installation check
display.ShowPreInstallCheck("Verifying cluster connectivity")

// Show skipped installation
display.ShowSkippedInstallation("ArgoCD", "already installed")

// Show dry-run results
results := []string{"helm install argocd", "kubectl apply -f manifests"}
display.ShowDryRunResults(os.Stdout, results)

// Handle installation error
err := fmt.Errorf("connection timeout")
display.ShowInstallError(models.ChartTypeArgoCD, err)
```
