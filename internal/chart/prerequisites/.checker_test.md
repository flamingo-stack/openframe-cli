Test suite for the prerequisite checker functionality that validates system requirements for the Flamingo Stack OpenFrame CLI tool.

## Key Components

- **TestNewPrerequisiteChecker**: Validates proper initialization of the checker with all 4 requirements (Git, Helm, Memory, Certificates)
- **TestInstallHelp**: Ensures each prerequisite checker provides non-empty installation instructions
- **TestCheckAllWithMissingTools**: Tests detection of missing tools and proper reporting
- **TestCheckAllWithAllTools**: Verifies behavior when all prerequisites are satisfied
- **TestGetInstallInstructions**: Validates generation of installation instructions for missing tools

## Usage Example

```go
// Run specific test
go test -run TestNewPrerequisiteChecker

// Run all tests in the package
go test

// Run with verbose output
go test -v

// Test specific functionality
go test -run TestCheckAll
```

The tests use mocking to simulate various installation states and verify that the checker correctly identifies missing tools and provides appropriate installation guidance. Each test focuses on a specific aspect of the prerequisite checking workflow, ensuring robust validation of system requirements.
