Handles the automatic installation of missing system prerequisites for OpenFrame deployment.

The installer orchestrates the detection and installation of required tools like Git, Helm, certificates, and memory validation. It provides both interactive and non-interactive modes for CI/CD environments and includes comprehensive error handling and user feedback.

## Key Components

- **Installer**: Main installation coordinator with prerequisite checker
- **InstallMissingPrerequisites()**: Detects and installs all missing tools
- **CheckAndInstallNonInteractive()**: Supports automated installation for CI/CD pipelines
- **installToolNonInteractive()**: Handles individual tool installation with mode switching
- **RegenerateCertificatesOnly()**: Specialized method for certificate refresh during deployments

## Usage Example

```go
// Basic installation check and install
installer := prerequisites.NewInstaller()
if err := installer.CheckAndInstall(); err != nil {
    log.Fatal("Failed to install prerequisites:", err)
}

// Non-interactive mode for CI/CD
if err := installer.CheckAndInstallNonInteractive(true); err != nil {
    log.Fatal("Automated installation failed:", err)
}

// Just regenerate certificates
if err := installer.RegenerateCertificatesOnly(); err != nil {
    log.Warn("Certificate refresh failed:", err)
}

// Install specific missing tools
missing := []string{"git", "helm"}
if err := installer.InstallMissingPrerequisites(); err != nil {
    log.Fatal("Installation failed:", err)
}
```

The installer gracefully handles memory warnings (non-blocking), provides detailed progress feedback with spinners, and can continue deployment even when some non-critical prerequisites fail in non-interactive mode.
