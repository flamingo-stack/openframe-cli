Test suite for the git package's `CloneResult` struct, validating default initialization and value assignment behavior.

## Key Components

- **TestCloneResult_DefaultValues**: Tests that a newly created `CloneResult` struct has empty default values for all fields
- **TestCloneResult_WithValues**: Tests that `CloneResult` struct properly stores assigned values for `TempDir` and `ChartPath` fields

## Usage Example

```go
// Run the tests
go test -v ./git

// The tests validate CloneResult behavior:
func TestCloneResult_DefaultValues(t *testing.T) {
    result := &CloneResult{}
    
    assert.Empty(t, result.TempDir)    // Should be ""
    assert.Empty(t, result.ChartPath)  // Should be ""
}

func TestCloneResult_WithValues(t *testing.T) {
    result := &CloneResult{
        TempDir:   "/tmp/clone-12345",
        ChartPath: "/tmp/clone-12345/manifests/app-of-apps",
    }
    
    assert.Equal(t, "/tmp/clone-12345", result.TempDir)
    assert.Equal(t, "/tmp/clone-12345/manifests/app-of-apps", result.ChartPath)
}
```

These tests ensure the `CloneResult` struct correctly handles both zero-value initialization and explicit field assignment, which is essential for git repository cloning operations that need to track temporary directories and chart paths.
