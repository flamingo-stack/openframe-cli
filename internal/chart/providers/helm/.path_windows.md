This file provides Windows-specific path expansion functionality to convert 8.3 short filenames to their full long path names, which is essential for WSL compatibility in Helm operations.

## Key Components

- **expandShortPath()** - Core function that expands Windows short paths (e.g., `RUNNER~1`) to full names (e.g., `runneradmin`)
- **kernel32 DLL bindings** - Lazy-loaded Windows API access for `GetLongPathNameW` system call
- **Build constraint** - `//go:build windows` ensures this code only compiles on Windows systems

## Technical Implementation

The function uses Windows' `GetLongPathNameW` API through syscalls:
1. Converts Go string to UTF-16 for Windows API compatibility
2. Makes initial API call to determine required buffer size
3. Allocates appropriate buffer and retrieves the expanded path
4. Converts UTF-16 result back to Go string

## Usage Example

```go
// Expand a Windows 8.3 short path
shortPath := `C:\Users\RUNNER~1\AppData\Local\Temp`
longPath, err := expandShortPath(shortPath)
if err != nil {
    // Handle error - function returns original path as fallback
    fmt.Printf("Using original path: %s\n", shortPath)
} else {
    // longPath might be: C:\Users\runneradmin\AppData\Local\Temp
    fmt.Printf("Expanded path: %s\n", longPath)
}
```

This is critical for Helm's Windows support since WSL (Windows Subsystem for Linux) cannot interpret Windows short filenames, requiring full path expansion for proper cross-platform operation.
