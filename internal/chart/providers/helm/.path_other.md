Platform-specific implementation for non-Windows systems that provides a no-op version of short path expansion functionality.

## Key Components

- **`expandShortPath`** - A no-operation function that returns the input path unchanged on non-Windows platforms

## Usage Example

```go
// On non-Windows systems, paths are returned as-is
originalPath := "/usr/local/bin/helm"
expandedPath, err := expandShortPath(originalPath)
if err != nil {
    log.Fatal(err)
}
// expandedPath equals "/usr/local/bin/helm" (unchanged)
fmt.Println(expandedPath)
```

This file uses Go's build constraint `//go:build !windows` to ensure it's only compiled on non-Windows platforms. Since Windows short filenames (8.3 format like `PROGRA~1`) don't exist on Unix-like systems, the function simply returns the original path without any transformation. This provides a consistent API across platforms while avoiding unnecessary processing on systems where short path expansion isn't needed.
