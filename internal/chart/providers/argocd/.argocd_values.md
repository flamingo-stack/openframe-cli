Provides pre-configured Helm chart values for deploying ArgoCD with optimized resource allocation and monitoring integration. This function returns a YAML configuration string that includes resource limits, custom health checks, and Loki logging annotations.

## Key Components

- **GetArgoCDValues()** - Returns a complete ArgoCD Helm values configuration as a YAML string
- **Resource Configuration** - Defines CPU and memory requests/limits for all ArgoCD components
- **Health Check Customization** - Includes Lua script for Application resource health status
- **Monitoring Integration** - Configures Loki scraping annotations for log collection

## Usage Example

```go
package main

import (
    "fmt"
    "github.com/your-org/your-project/argocd"
)

func main() {
    // Get ArgoCD Helm values
    values := argocd.GetArgoCDValues()
    
    // Use with Helm installation
    fmt.Printf("ArgoCD Values:\n%s", values)
    
    // Example: Save to file for helm install
    // helm install argocd argo/argo-cd --values argocd-values.yaml
}
```

The configuration includes optimized resource allocations ranging from 50m CPU/64Mi memory for lightweight components (redis, dex) up to 400m CPU/1Gi memory for the controller, plus extended sync timeouts and exec timeouts for handling large applications.
