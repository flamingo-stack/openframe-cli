This file provides default Helm chart configuration values for ArgoCD deployment, returning them as a YAML string for use in Kubernetes installations.

## Key Components

- **GetArgoCDValues()** - Returns a pre-configured YAML string containing ArgoCD Helm chart values with custom health check logic and timeout settings

## Configuration Details

The returned YAML includes:

- **Custom Health Checks**: Lua script for Application resource health monitoring
- **Repository Server Settings**: Extended exec timeout (180s) for long-running operations
- **Naming Override**: Sets fullname to "argocd" for consistent resource naming
- **Commented Sections**: Optional image pull secrets configuration for private registries

## Usage Example

```go
package main

import (
    "fmt"
    "github.com/your-org/argocd"
)

func main() {
    // Get ArgoCD Helm values for deployment
    values := argocd.GetArgoCDValues()
    
    // Use with Helm installation
    fmt.Printf("ArgoCD Helm values:\n%s", values)
    
    // Could be written to values.yaml file
    // or used with Helm Go SDK for programmatic deployment
}
```

The configuration is optimized for production deployments with extended timeouts for repository operations and robust health monitoring for ArgoCD Applications.
