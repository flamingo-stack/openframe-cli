This file provides pre-configured Helm chart values for ArgoCD deployment with custom health check logic and timeout settings.

## Key Components

- **GetArgoCDValues()** - Returns a YAML configuration string containing ArgoCD Helm chart values with custom resource health checks and repository server environment variables

## Usage Example

```go
package main

import (
    "fmt"
    "github.com/your-org/your-project/argocd"
)

func main() {
    // Get ArgoCD Helm values
    values := argocd.GetArgoCDValues()
    
    // Use with Helm deployment
    fmt.Println("ArgoCD Values:")
    fmt.Println(values)
    
    // Or write to file for helm install
    // helm install argocd argo/argo-cd -f values.yaml
}
```

The returned YAML includes:
- Custom health check logic for ArgoCD Application resources
- Extended execution timeout (180s) for the repository server
- Fullname override set to "argocd"
- Commented placeholder for image pull secrets

This configuration is typically used when deploying ArgoCD via Helm charts in Kubernetes environments.
