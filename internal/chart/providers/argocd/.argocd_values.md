This file provides Helm chart values configuration for deploying ArgoCD with customized settings for timeout and health checks.

## Key Components

- **`GetArgoCDValues()`** - Returns a YAML string containing ArgoCD Helm chart values with custom configurations

## Configuration Details

The returned YAML includes:

- **Custom Health Check**: Lua script for `argoproj.io/Application` resources that properly handles application health status
- **Timeout Configuration**: Sets ArgoCD exec timeout to 180 seconds via `ARGOCD_EXEC_TIMEOUT` environment variable
- **Naming Override**: Sets fullname override to "argocd" for consistent resource naming
- **Optional Image Pull Secrets**: Commented configuration for private registry authentication

## Usage Example

```go
package main

import (
    "fmt"
    "github.com/your-org/argocd"
)

func main() {
    // Get ArgoCD Helm values
    values := argocd.GetArgoCDValues()
    
    // Use with Helm deployment
    fmt.Println("ArgoCD Helm Values:")
    fmt.Println(values)
    
    // Example: Write to file for helm install
    // helm install argocd argo/argo-cd -f argocd-values.yaml
}
```

The health check script ensures ArgoCD applications report proper status information, while the increased timeout accommodates longer-running operations during application synchronization.
