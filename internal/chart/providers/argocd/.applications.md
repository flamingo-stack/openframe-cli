Parses raw JSON output from kubectl into a slice of `Application` structs, extracting health, sync, condition, and operation state details for each ArgoCD application.

## Key Components

**Types**
- `Manager` — Core struct managing ArgoCD operations; holds Kubernetes/ArgoCD native clients, executor, cluster context, and stabilization settings
- `Application` — Represents the status snapshot of a single ArgoCD application (health, sync, conditions, source info)
- `argoAppList` / `argoApp` — Internal JSON deserialization structs for kubectl output

**Constructors**
- `NewManager(exec)` — Basic manager with command executor only
- `NewManagerWithCluster(exec, clusterName)` — Manager with explicit k3d cluster context
- `NewManagerWithConfig(exec, config)` — Preferred constructor when a `*rest.Config` is already available; initializes all native clients immediately

**Methods**
- `initKubernetesClients()` — Lazily initializes kube, apiext, and ArgoCD native clients from kubeconfig
- `getTotalExpectedApplications(ctx, config)` — Counts expected apps via native ArgoCD client (falls back to kubectl on Windows)
- `getTotalExpectedApplicationsViaKubectl(ctx, config)` — kubectl-based fallback for app discovery
- `getKubeconfigPath()` — Resolves kubeconfig path from `$KUBECONFIG` env or `~/.kube/config`
- `getKubectlArgs(args...)` — Prepends `--context k3d-<name>` to kubectl args when cluster name is set

## Usage Example

```go
// Preferred: inject existing rest.Config (e.g., post-cluster creation)
mgr, err := argocd.NewManagerWithConfig(executor, restConfig)
if err != nil {
    log.Fatal(err)
}

// Or: lazily resolved from kubeconfig
mgr := argocd.NewManagerWithCluster(executor, "openframe")
mgr.StabilizationChecks = 3 // speed up in tests

total := mgr.getTotalExpectedApplications(ctx, installConfig)
fmt.Printf("Expecting %d ArgoCD applications\n", total)
```
