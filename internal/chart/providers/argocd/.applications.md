This file implements an ArgoCD application manager that monitors and analyzes ArgoCD application deployments. It provides functionality to discover applications and estimate total expected deployments during cluster setup.

## Key Components

- **Manager**: Core struct that handles ArgoCD operations using a command executor
- **Application**: Represents an ArgoCD application with name, health, and sync status
- **getTotalExpectedApplications()**: Attempts multiple strategies to determine total applications that will be deployed
- **parseApplications()**: Retrieves current ArgoCD application status using kubectl commands

## Usage Example

```go
import (
    "context"
    "github.com/flamingo-stack/openframe-cli/internal/shared/executor"
    "github.com/flamingo-stack/openframe-cli/internal/chart/utils/config"
)

// Create a new ArgoCD manager
exec := executor.NewCommandExecutor()
manager := NewManager(exec)

// Get current applications and their status
ctx := context.Background()
apps, err := manager.parseApplications(ctx, true)
if err != nil {
    log.Fatal(err)
}

for _, app := range apps {
    fmt.Printf("App: %s, Health: %s, Sync: %s\n", 
        app.Name, app.Health, app.Sync)
}

// Estimate total expected applications
config := config.ChartInstallConfig{Verbose: true}
totalExpected := manager.getTotalExpectedApplications(ctx, config)
fmt.Printf("Expected applications: %d\n", totalExpected)
```

The manager uses multiple fallback strategies to gather application information, making it resilient to different cluster states and ArgoCD configurations.
