This file provides an ArgoCD manager for handling application lifecycle operations and status monitoring through kubectl and helm commands.

## Key Components

- **Manager**: Main struct that wraps a command executor for ArgoCD operations
- **Application**: Represents an ArgoCD application with name, health, and sync status
- **NewManager()**: Factory function to create a new Manager instance
- **getTotalExpectedApplications()**: Attempts to determine the total number of applications using multiple fallback methods
- **parseApplications()**: Retrieves current ArgoCD application statuses via kubectl

## Usage Example

```go
package main

import (
    "context"
    "github.com/flamingo-stack/openframe-cli/internal/shared/executor"
)

func main() {
    // Create manager with command executor
    exec := executor.NewCommandExecutor()
    manager := NewManager(exec)
    
    // Get current application statuses
    ctx := context.Background()
    apps, err := manager.parseApplications(ctx, true)
    if err != nil {
        panic(err)
    }
    
    // Process applications
    for _, app := range apps {
        fmt.Printf("App: %s, Health: %s, Sync: %s\n", 
            app.Name, app.Health, app.Sync)
    }
    
    // Get expected application count
    config := config.ChartInstallConfig{Verbose: true}
    total := manager.getTotalExpectedApplications(ctx, config)
    fmt.Printf("Expected applications: %d\n", total)
}
```

The manager uses multiple detection strategies to handle various ArgoCD deployment scenarios and provides robust error handling when applications are still initializing.
