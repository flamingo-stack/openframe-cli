This ArgoCD management package provides a Go client interface for handling ArgoCD applications with both native Kubernetes API access and kubectl fallback methods, featuring optimized Windows/WSL2 support and comprehensive application status monitoring.

## Key Components

- **Manager**: Main ArgoCD operations handler with multiple client interfaces
- **Application**: Struct representing ArgoCD application status with health, sync, and error details
- **Constructor Functions**: Multiple initialization methods for different use cases
- **Client Management**: Native Kubernetes clients with kubectl fallback for reliability

## Usage Example

```go
// Create manager with command executor
exec := executor.NewCommandExecutor()
manager := argocd.NewManager(exec)

// Or with explicit cluster context
manager = argocd.NewManagerWithCluster(exec, "openframe")

// Or with pre-configured Kubernetes client
config := getKubernetesConfig()
manager, err := argocd.NewManagerWithConfig(exec, config)
if err != nil {
    log.Fatal(err)
}

// Get expected application count
ctx := context.Background()
config := config.ChartInstallConfig{
    ClusterName: "openframe",
    Verbose: true,
}
totalApps := manager.getTotalExpectedApplications(ctx, config)

// Parse application status
applications, err := manager.parseApplications(ctx, true)
if err != nil {
    log.Fatal(err)
}

for _, app := range applications {
    fmt.Printf("App: %s, Health: %s, Sync: %s\n", 
               app.Name, app.Health, app.Sync)
}
```

The manager automatically handles Windows/WSL2 networking issues by falling back to kubectl commands when native clients cannot connect, and applies insecure TLS configuration for local k3d clusters.
