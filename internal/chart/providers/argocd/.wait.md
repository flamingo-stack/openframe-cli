This file provides functionality for waiting on ArgoCD applications to reach a healthy and synced state during deployment operations.

## Key Components

- **WaitForApplications()** - Main function that monitors ArgoCD applications until they reach Healthy+Synced status
- **Context handling** - Supports cancellation via context or interrupt signals (Ctrl+C)
- **Progress monitoring** - Shows spinner with progress updates and verbose logging options
- **Timeout management** - 60-minute timeout with early exits for dry-run or cancelled contexts
- **Application tracking** - Monitors application health/sync status with detailed debugging for stuck applications

## Usage Example

```go
// Wait for ArgoCD applications to be ready
manager := &Manager{executor: executor}
config := config.ChartInstallConfig{
    Verbose: true,
    Silent:  false,
    DryRun:  false,
}

ctx, cancel := context.WithTimeout(context.Background(), 30*time.Minute)
defer cancel()

err := manager.WaitForApplications(ctx, config)
if err != nil {
    log.Fatalf("Failed to wait for applications: %v", err)
}

// For dry-run mode (skips waiting)
config.DryRun = true
err = manager.WaitForApplications(ctx, config)
// Returns immediately without waiting
```

The function includes a 30-second bootstrap period, checks applications every 2 seconds, provides verbose progress updates every 10 seconds, and shows detailed debugging information for stuck applications after 7 minutes.
