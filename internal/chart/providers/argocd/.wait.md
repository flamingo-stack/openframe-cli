Manages ArgoCD application synchronization and health monitoring with robust error handling, cluster connectivity checks, and recovery mechanisms for Windows/WSL environments.

## Key Components

- **WaitForApplications()** - Main function that waits for all ArgoCD applications to reach Healthy and Synced status
- **Context Management** - Handles cancellation, timeouts, and interrupt signals with graceful shutdown
- **Health Monitoring** - Periodic checks for cluster connectivity, repo-server health, and system resources
- **Recovery Logic** - Automatic WSL recovery on Windows and repo-server restart mechanisms
- **Progress Tracking** - Visual spinner with timer and verbose logging options

## Usage Example

```go
import (
    "context"
    "time"
    "github.com/flamingo-stack/openframe-cli/internal/chart/utils/config"
)

func deployApplications(manager *argocd.Manager) error {
    ctx, cancel := context.WithTimeout(context.Background(), 60*time.Minute)
    defer cancel()

    config := config.ChartInstallConfig{
        ClusterName: "my-cluster",
        Verbose:     true,
        Silent:      false,
        DryRun:      false,
        SkipCRDs:    false,
    }

    // Wait for all ArgoCD applications to be ready
    if err := manager.WaitForApplications(ctx, config); err != nil {
        return fmt.Errorf("failed to wait for applications: %w", err)
    }

    return nil
}
```

The function includes sophisticated error handling with exponential backoff, cluster diagnostics, and automatic recovery attempts for common failure scenarios in CI/CD environments.
